懒加载是一种"用时再取"的性能优化策略，其本质是将代码拆分（`Code Splitting`）与按需加载相结合。在 Next.js 应用中，合理使用懒加载可以显著减少初始 JavaScript bundle 的体积，从而加快首屏加载速度。

## 为什么需要懒加载？

Web 应用通常包含大量功能模块，如果将所有代码都打包到一个 bundle 中，会导致：

- 初始加载时间长，影响首屏渲染性能
- 浏览器需要下载并解析不必要的代码
- 移动端用户流量消耗增加

懒加载通过将非关键代码延迟到真正需要时再加载，让用户能够更快看到页面内容。在 Next.js 中，服务端组件默认已经实现了自动代码分割，但客户端组件需要手动使用懒加载技术。

Next.js 支持两种懒加载实现方式：`React.lazy()` + `Suspense`（React 提供的原生方案）和 `next/dynamic`（Next.js 封装的便捷 API，推荐使用）。通常建议使用 `next/dynamic`，因为它针对 Next.js 环境做了优化，使用更加便捷。

## React.lazy 基础

`React.lazy()` 允许你将组件定义为一个动态导入（dynamic import）。这个函数接收一个返回 `import()` 的函数作为参数，返回一个特殊的 React 组件。基本用法如下：

```tsx
import { lazy } from "react";

// 延迟加载组件
const HeavyComponent = lazy(() => import("./HeavyComponent"));
```

注意：`lazy()` 必须在组件外部、模块顶层调用，不能在函数或条件语句中使用。

懒加载的组件必须包裹在 `<Suspense>` 中，用于处理加载状态：

```tsx
import { Suspense, lazy } from "react";

const HeavyComponent = lazy(() => import("./HeavyComponent"));

function App() {
  return (
    <Suspense fallback={<div>加载中...</div>}>
      <HeavyComponent />
    </Suspense>
  );
}
```

下面是一个更贴近实际场景的例子：图片上传组件只有在用户点击上传按钮时才加载。

```tsx
"use client";

import { useState, Suspense, lazy } from "react";

// 懒加载图片上传组件
const ImageUploader = lazy(() => import("./ImageUploader"));

export default function PostEditor() {
  const [showUploader, setShowUploader] = useState(false);

  return (
    <div>
      <textarea placeholder="输入内容..." />

      <button onClick={() => setShowUploader(true)}>上传图片</button>

      {showUploader && (
        <Suspense fallback={<div>正在加载上传组件...</div>}>
          <ImageUploader onClose={() => setShowUploader(false)} />
        </Suspense>
      )}
    </div>
  );
}
```

`ImageUploader` 组件的完整实现：

```tsx
"use client";

import { useState } from "react";

interface ImageUploaderProps {
  onClose: () => void;
}

export default function ImageUploader({ onClose }: ImageUploaderProps) {
  const [selectedFile, setSelectedFile] = useState<File | null>(null);
  const [preview, setPreview] = useState<string | null>(null);

  const handleFileChange = (e: React.ChangeEvent<HTMLInputElement>) => {
    const file = e.target.files?.[0];
    if (file) {
      setSelectedFile(file);
      const reader = new FileReader();
      reader.onloadend = () => {
        setPreview(reader.result as string);
      };
      reader.readAsDataURL(file);
    }
  };

  const handleUpload = async () => {
    if (!selectedFile) return;
    // 上传逻辑
    console.log("上传文件:", selectedFile.name);
    onClose();
  };

  return (
    <div className="border border-gray-300 p-5 mt-2.5 rounded-lg bg-white">
      <h3 className="text-lg font-semibold mb-4">上传图片</h3>
      <input
        type="file"
        accept="image/*"
        onChange={handleFileChange}
        className="mb-4"
      />
      {preview && (
        <div className="mt-2.5 mb-2.5">
          <img src={preview} alt="预览" className="max-w-[200px] rounded" />
        </div>
      )}
      <div className="mt-2.5 flex gap-2.5">
        <button
          onClick={handleUpload}
          disabled={!selectedFile}
          className="px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600 disabled:bg-gray-300 disabled:cursor-not-allowed"
        >
          确认上传
        </button>
        <button
          onClick={onClose}
          className="px-4 py-2 bg-gray-200 text-gray-800 rounded hover:bg-gray-300"
        >
          取消
        </button>
      </div>
    </div>
  );
}
```

## next/dynamic 深入解析

`next/dynamic` 是 Next.js 提供的懒加载 API，它内部使用了 `React.lazy` 和 `Suspense`，但提供了更友好的配置选项。基本用法如下：

```tsx
import dynamic from "next/dynamic";

interface ChartData {
  name: string;
  value: number;
}

const Chart = dynamic(() => import("../components/Chart"), {
  loading: () => <p className="text-gray-500">图表加载中...</p>,
});

export default function Dashboard() {
  const data: ChartData[] = [
    { name: "一月", value: 100 },
    { name: "二月", value: 200 },
    { name: "三月", value: 150 },
  ];

  return (
    <div className="p-6">
      <h1 className="text-2xl font-bold mb-4">数据面板</h1>
      <Chart data={data} />
    </div>
  );
}
```

`Chart` 组件的完整实现：

```tsx
"use client";

interface ChartProps {
  data: { name: string; value: number }[];
}

export default function Chart({ data }: ChartProps) {
  return (
    <div className="p-5 border border-gray-300 rounded-lg bg-white">
      <h3 className="text-lg font-semibold mb-4">数据图表</h3>
      <div className="space-y-2.5">
        {data.map((item, index) => (
          <div key={index} className="flex justify-between items-center">
            <span className="text-gray-700">{item.name}:</span>
            <span className="font-bold text-gray-900">{item.value}</span>
          </div>
        ))}
      </div>
    </div>
  );
}
```

`dynamic()` 接收两个参数：

- 第一个参数：返回 `import()` 的函数
- 第二个参数：配置对象（可选）

### 重要约束条件

使用 `dynamic()` 时需要遵守以下规则：

1. 路径必须是静态字符串

`import()` 中的路径必须是静态字符串，不能使用变量或模板字符串，因为 Next.js 需要在构建时分析这些导入路径。

```tsx
// 正确：使用静态字符串路径
const Component = dynamic(() => import("./Component"));

// 错误：不能使用变量
const path = "./Component";
const Component = dynamic(() => import(path)); // 编译错误

// 错误：不能使用模板字符串
const componentName = "Component";
const Component = dynamic(() => import(`./${componentName}`)); // 编译错误
```

2. 必须在模块顶层调用

`dynamic()` 函数必须在模块顶层调用，不能在函数内部、条件语句中或其他函数内调用。

```tsx
// 正确：在模块顶层调用
const Component = dynamic(() => import("./Component"));

export default function Page() {
  return <Component />;
}

// 错误：不能在函数内部调用
export default function Page() {
  const Component = dynamic(() => import("./Component")); // 运行时错误
  return <Component />;
}

// 错误：不能在条件语句中调用
const shouldLoad = true;
const Component = shouldLoad
  ? dynamic(() => import("./Component")) // 运行时错误
  : null;
```

### SSR 控制

默认情况下，使用 `dynamic()` 的组件会参与服务端渲染。但某些情况下必须禁用服务端渲染，通过设置 `ssr: false` 来实现。

何时真正需要 `ssr: false`：

1. 第三方库在模块顶层或初始化时直接访问浏览器 API（无法修改库代码）
2. 组件依赖 DOM 测量（如元素宽高）才能正确渲染
3. 使用 `window.innerWidth` 等浏览器 API 进行响应式布局

需要 `ssr: false` 的真实场景：

```tsx
"use client";

import dynamic from "next/dynamic";
import { useState } from "react";

const ResponsiveChart = dynamic(() => import("./ResponsiveChart"), {
  ssr: false,
  loading: () => (
    <div className="flex items-center justify-center h-96 bg-gray-100 border border-gray-300 rounded">
      <p className="text-gray-500">图表加载中...</p>
    </div>
  ),
});

export default function DashboardPage() {
  return (
    <div className="p-6">
      <h1 className="text-2xl font-bold mb-4">数据看板</h1>
      <ResponsiveChart />
    </div>
  );
}
```

`ResponsiveChart` 组件的完整实现（必须在客户端才能正确计算尺寸）：

```tsx
"use client";

import { useEffect, useRef, useState } from "react";

export default function ResponsiveChart() {
  const containerRef = useRef<HTMLDivElement>(null);
  const canvasRef = useRef<HTMLCanvasElement>(null);

  // 根据容器宽度计算图表尺寸（服务端无法获取实际宽度）
  const [dimensions, setDimensions] = useState({
    width: window.innerWidth > 768 ? 800 : window.innerWidth - 48,
    height: 400,
  });

  useEffect(() => {
    const updateDimensions = () => {
      if (containerRef.current) {
        const width = containerRef.current.offsetWidth;
        setDimensions({
          width: width,
          height: Math.min(400, width * 0.5),
        });
      }
    };

    updateDimensions();
    window.addEventListener("resize", updateDimensions);
    return () => window.removeEventListener("resize", updateDimensions);
  }, []);

  useEffect(() => {
    const canvas = canvasRef.current;
    if (!canvas) return;

    const ctx = canvas.getContext("2d");
    if (!ctx) return;

    // 绘制响应式图表
    const data = [120, 200, 150, 300, 250];
    const maxValue = Math.max(...data);
    const barWidth = (dimensions.width - 100) / data.length - 10;
    const barSpacing = 10;

    ctx.clearRect(0, 0, canvas.width, canvas.height);
    ctx.fillStyle = "#3B82F6";

    data.forEach((value, index) => {
      const barHeight = (value / maxValue) * (dimensions.height - 60);
      const x = 50 + index * (barWidth + barSpacing);
      const y = dimensions.height - barHeight - 30;

      ctx.fillRect(x, y, barWidth, barHeight);

      // 标签
      ctx.fillStyle = "#1F2937";
      ctx.font = "14px Arial";
      ctx.textAlign = "center";
      ctx.fillText(value.toString(), x + barWidth / 2, y - 10);
      ctx.fillText(
        `项目${index + 1}`,
        x + barWidth / 2,
        dimensions.height - 10
      );
      ctx.fillStyle = "#3B82F6";
    });
  }, [dimensions]);

  return (
    <div ref={containerRef} className="w-full">
      <div className="mb-4 p-4 bg-blue-50 border border-blue-200 rounded">
        <p className="text-sm text-blue-800">
          图表尺寸：{dimensions.width} x {dimensions.height}px
        </p>
        <p className="text-sm text-blue-600 mt-1">
          根据屏幕宽度自动调整（{window.innerWidth}px）
        </p>
      </div>
      <canvas
        ref={canvasRef}
        width={dimensions.width}
        height={dimensions.height}
        className="border border-gray-300 rounded bg-white"
      />
    </div>
  );
}
```

这个组件必须设置 `ssr: false`，原因是：

1. 初始化时直接使用 `window.innerWidth` 计算尺寸（第 9 行），服务端没有 `window` 对象会直接报错
2. 组件依赖真实的 DOM 尺寸（`offsetWidth`）才能正确渲染
3. 服务端无法获取用户的实际屏幕宽度

如果不设置 `ssr: false`，会出现以下错误：

![20251216112120](https://raw.githubusercontent.com/xun082/md/main/blogs.images20251216112120.png)

即使添加 `typeof window !== "undefined"` 检查，也会导致服务端渲染的尺寸与客户端不一致，造成内容闪烁（hydration mismatch）。因此必须使用 `ssr: false` 确保组件只在客户端渲染。

### SSR 配置对比

通过实际例子理解不同 `SSR` 配置的影响：

```tsx
"use client";

import { useState } from "react";
import dynamic from "next/dynamic";

const ComponentA = dynamic(() => import("./components/A"));
const ComponentB = dynamic(() => import("./components/B"), {
  ssr: false,
});
const ComponentC = dynamic(() => import("./components/C"));

export default function ComparisonPage() {
  const [showC, setShowC] = useState(false);

  return (
    <div className="p-6 space-y-4">
      {/* 立即渲染，HTML 中包含完整内容 */}
      <ComponentA />

      {/* 延迟渲染，HTML 中只有占位符 */}
      <ComponentB />

      {/* 只有点击按钮后才开始加载 */}
      <button
        onClick={() => setShowC(true)}
        className="px-4 py-2 bg-blue-500 text-white rounded hover:bg-blue-600"
      >
        加载 C
      </button>
      {showC && <ComponentC />}
    </div>
  );
}
```

组件 A 实现（默认 SSR）：

```tsx
// components/A.tsx
"use client";

export default function ComponentA() {
  return (
    <div className="p-5 border-2 border-green-500 mb-2.5 rounded-lg bg-green-50">
      <h3 className="text-lg font-semibold mb-2 text-green-700">
        组件 A（默认 SSR）
      </h3>
      <p className="text-gray-700">
        这个组件会被服务端预渲染，HTML 中包含完整内容。
      </p>
    </div>
  );
}
```

组件 B 实现（禁用 SSR）：

```tsx
// components/B.tsx
"use client";

export default function ComponentB() {
  return (
    <div className="p-5 border-2 border-orange-500 mb-2.5 rounded-lg bg-orange-50">
      <h3 className="text-lg font-semibold mb-2 text-orange-700">
        组件 B（禁用 SSR）
      </h3>
      <p className="text-gray-700">
        这个组件不会在服务端渲染，HTML 中只有占位符，等待客户端 JavaScript
        执行。
      </p>
    </div>
  );
}
```

组件 C 实现（按需加载）：

```tsx
// components/C.tsx
"use client";

export default function ComponentC() {
  return (
    <div className="p-5 border-2 border-blue-500 mb-2.5 rounded-lg bg-blue-50">
      <h3 className="text-lg font-semibold mb-2 text-blue-700">
        组件 C（按需加载）
      </h3>
      <p className="text-gray-700">这个组件只有在点击按钮后才会开始加载。</p>
    </div>
  );
}
```

渲染行为对比：

| 组件       | 服务端渲染 | 初始 HTML | Bundle 加载时机          |
| ---------- | ---------- | --------- | ------------------------ |
| ComponentA | 是         | 完整内容  | 页面加载时               |
| ComponentB | 否         | 占位符    | 页面加载时（需等待水合） |
| ComponentC | 是         | 无        | 点击按钮后               |

即使组件在服务端渲染，客户端仍需加载 JavaScript 实现交互功能（事件处理等），这个过程称为"水合"（`Hydration`）。因此所有客户端组件最终都需要在浏览器中加载对应的 bundle。

## 懒加载第三方库

除了组件，还可以懒加载大型第三方库。以下是使用 `recharts` 图表库的真实案例：

```bash
pnpm add recharts
```

在用户点击时才加载 recharts 库：

```tsx
"use client";

import { useState } from "react";

interface ChartData {
  name: string;
  sales: number;
  revenue: number;
}

export default function DashboardPage() {
  const [showChart, setShowChart] = useState(false);
  const [ChartComponent, setChartComponent] = useState<React.ComponentType<{
    data: ChartData[];
  }> | null>(null);

  const handleLoadChart = async () => {
    const {
      BarChart,
      Bar,
      XAxis,
      YAxis,
      CartesianGrid,
      Tooltip,
      Legend,
      ResponsiveContainer,
    } = await import("recharts");

    // 创建图表组件
    const Chart = ({ data }: { data: ChartData[] }) => (
      <ResponsiveContainer width="100%" height={400}>
        <BarChart data={data}>
          <CartesianGrid strokeDasharray="3 3" />
          <XAxis dataKey="name" />
          <YAxis />
          <Tooltip />
          <Legend />
          <Bar dataKey="sales" fill="#3B82F6" />
          <Bar dataKey="revenue" fill="#10B981" />
        </BarChart>
      </ResponsiveContainer>
    );

    setChartComponent(() => Chart);
    setShowChart(true);
  };

  const chartData: ChartData[] = [
    { name: "1月", sales: 4000, revenue: 2400 },
    { name: "2月", sales: 3000, revenue: 1398 },
    { name: "3月", sales: 2000, revenue: 9800 },
    { name: "4月", sales: 2780, revenue: 3908 },
    { name: "5月", sales: 1890, revenue: 4800 },
    { name: "6月", sales: 2390, revenue: 3800 },
  ];

  return (
    <div className="p-6 max-w-4xl mx-auto">
      <h1 className="text-3xl font-bold mb-6">销售数据分析</h1>

      {!showChart ? (
        <div className="border-2 border-dashed border-gray-300 rounded-lg p-12 text-center">
          <p className="text-gray-500 mb-4">点击按钮加载图表（约 300KB）</p>
          <button
            onClick={handleLoadChart}
            className="px-6 py-3 bg-blue-500 text-white rounded-lg hover:bg-blue-600 transition-colors"
          >
            加载图表
          </button>
        </div>
      ) : (
        <div className="bg-white p-6 rounded-lg shadow-lg">
          {ChartComponent && <ChartComponent data={chartData} />}
        </div>
      )}
    </div>
  );
}
```

适用场景：

对于包体积较大的第三方库，懒加载能显著减少初始加载时间：

- 图表库：recharts、chart.js、d3.js 等可视化库
- 代码编辑器：Monaco Editor、CodeMirror 等重量级编辑器
- 富文本编辑器：Quill、Draft.js、Slate 等编辑器
- 3D 渲染库：Three.js、Babylon.js 等图形库
- PDF 预览：react-pdf、pdf.js 等文档处理库
- 复杂组件库：日期选择器、地图组件、视频播放器等

判断标准：当第三方库或组件体积较大，且不是首屏必需时，应该使用懒加载。

## 总结

懒加载是 Next.js 中强大的性能优化技术，需要根据实际情况判断使用。适合使用的场景包括：条件渲染的组件（模态框、抽屉、标签页内容）、低优先级功能（用户设置、帮助文档）、大型第三方库（数据可视化库、编辑器，通常 > 50KB）以及折叠内容（下拉列表、手风琴面板）。判断标准是：如果组件在初始渲染时不会显示，或需要用户交互后才显示，可以考虑懒加载。

避免对首屏关键组件（导航栏、主要内容区域）、小体积组件（几 KB，额外网络请求的成本可能超过收益）以及需要 `SEO` 的内容使用懒加载，因为额外的网络请求反而会降低性能，且可能影响搜索引擎抓取。

使用时注意两个常见误区：不要对所有组件都使用懒加载，也不要过度拆分小型组件。可以通过预加载策略（鼠标悬停时提前加载）和视口检测（`Intersection Observer`）来优化用户体验，减少等待时间。使用 `@next/bundle-analyzer` 分析打包体积，结合 `loading` 配置提供加载反馈，监控 Core Web Vitals 指标，确保优化真正提升了应用性能。性能优化需要在用户体验、代码复杂度和加载速度之间找到平衡点，合理使用懒加载可以显著提升 Next.js 应用的首屏加载速度和整体性能。
