## 前言

Next.js 内置了 `next/font` 组件，相比于传统使用字体的方式，使用 font 组件会更加灵活便捷。font 组件的使用主要分为两块，一块是 Google 字体，一块是本地字体，都是通过 font 组件实现，但具体配置上会略有不同。

下图展示了传统字体加载方式与 `next/font` 的核心差异：

![20260206093413](https://raw.githubusercontent.com/xun082/md/main/blogs.images20260206093413.png)

从图中可以看出，传统方式需要手动引入外部资源，每次访问都会向 Google 发送请求，还可能导致布局偏移。而 `next/font` 会在构建时自动下载字体文件，作为静态资源托管，既保证了性能，也保护了用户隐私。

本篇我们会先从传统使用字体的方式开始讲起，然后讲解 font 组件带来的便利和优化，最后深入细节，讲解 font 函数的具体参数。

## 传统使用字体

传统使用字体的方式主要有三种，通过 `@font-face` 自定义字体、使用 `link` 标签引入、使用 `@import` 导入。下图展示了这三种方式的工作流程：

![20260206093723](https://raw.githubusercontent.com/xun082/md/main/blogs.images20260206093723.png)

从流程图可以看出，无论使用哪种方式，最终都需要浏览器在运行时下载字体文件，这会增加页面加载时间，也可能导致字体闪烁或布局偏移。

最基本的方法是通过 `@font-face` 指定一个自定义字体，字体文件可以来自远程文件，也可以来自本地文件。然后在 `font-family` 中使用该字体。

```css
// global.css
@font-face {
  font-family: "Bitstream Vera Serif Bold";
  src: url("https://mdn.github.io/css-examples/web-fonts/VeraSeBd.ttf");
}

body {
  font-family: "Bitstream Vera Serif Bold", serif;
}
```

借助 Google Fonts 这样的字体网站，我们可以快速生成样式文件，再通过 `link` 标签或者 `@import` 的方式直接使用。

使用 `link` 标签

```javascript
// layout.js
export default function Layout({ children }) {
  return (
    <html>
      <head>
        <link
          href="https://fonts.googleapis.com/css2?family=Ma+Shan+Zheng&display=swap"
          rel="stylesheet"
        />
      </head>
      <body>{children}</body>
    </html>
  );
}
```

```css
// globals.css
body {
  font-family: "Ma Shan Zheng", serif;
}
```

使用 `@import`

```css
// globals.css
@import url('https://fonts.googleapis.com/css2?family=Ma+Shan+Zheng&display=swap');

body {
  font-family: "Ma Shan Zheng", serif;
}
```

## next/font 与布局偏移

Next.js 内置了 `next/font` 组件，帮助你更好地管理和使用字体。`next/font` 会自动优化字体（包括自定义字体），借助 CSS 的 `size-adjust` 属性实现零布局偏移。

布局偏移我们在 Image 组件篇讲过，除了图片不设置宽高会导致布局偏移，网页字体加载时也容易出现同样的问题：不同字体的字形设计差异很大，即使设置相同的 `font-size`，实际占用的垂直空间也可能不一致，从而引发布局跳动。

下图直观展示了字体加载与布局偏移的流程：

![20260206195207](https://raw.githubusercontent.com/xun082/md/main/blogs.images20260206195207.png)

下图直观展示了 size-adjust 的校正效果：

![20260206195309](https://raw.githubusercontent.com/xun082/md/main/blogs.images20260206195309.png)

左侧（Unadjusted）：三行 Player 1 使用不同字体，`font-size` 均为 64px，但实际渲染高度分别为 74px、64px、78px，导致各行垂直空间参差不齐，产生布局偏移。

右侧（Size Adjusted）：通过 `size-adjust` 对字体进行缩放校正后，三行文字的实际高度统一为 64px，布局稳定一致。Next.js 的 `next/font` 会在构建时自动完成这一优化。

除了防止布局偏移，`next/font` 还可以帮助你快捷使用 Google 字体，而且 CSS 和字体文件会在构建时下载，与其他静态资源一并保存，浏览器不会向 Google 发送任何请求，兼顾性能和隐私。

`next/font` 具体分为 `next/font/google` 和 `next/font/local`，分别对应使用 Google 字体和本地字体。我们逐一讲解。

## next/font/google

`next/font/google` 用于从 Google Fonts 引入字体。它会在构建时自动下载字体文件并托管为静态资源，浏览器无需向 Google 发起请求，既保证加载速度，又兼顾隐私。本节介绍其基础用法、可变字体以及子集配置。

### 3.1. 基础使用

借助 `next/font/google`，无需再到 Google Fonts 复制样式链接或通过 link、import 引入，可直接在代码中导入所需字体。

下图展示了 `next/font/google` 的完整使用流程：

![20260206204355](https://raw.githubusercontent.com/xun082/md/main/blogs.images20260206204355.png)

整个流程非常简洁，导入字体后实例化配置，然后将生成的 className 应用到元素上即可。Next.js 会在构建时自动下载字体文件并优化，无需手动管理。

使用示例如下：

```javascript
// app/layout.js
// 1. 导入想要使用的字体
import { Inter } from "next/font/google";

// 2. 实例化字体对象，设置使用子集等
const inter = Inter({
  subsets: ["latin"],
});

// 3. 应用，inter.className 会返回一个只读的 CSS 类名用于加载字体
export default function RootLayout({ children }) {
  return (
    <html lang="en" className={inter.className}>
      <body>{children}</body>
    </html>
  );
}
```

上述代码在 IDE 中的实际效果如下，字体样式会正确应用到页面：

![image.png](https://p3-juejin.byteimg.com/tos-cn-i-k3u1fbpfcp/a21325f8edb24dd5b891f2fcd5207b00~tplv-k3u1fbpfcp-jj-mark:0:0:0:0:q75.image#?w=1624&h=474&s=211319&e=png&b=2a2a2a)

Next.js 推荐使用可变字体以获得最佳性能和灵活性。若字体不支持可变，则需显式声明 `weight`（字重，即字体粗细）。

```javascript
// app/layout.js
import { Roboto } from "next/font/google";

const roboto = Roboto({
  weight: "400",
  subsets: ["latin"],
});

export default function RootLayout({ children }) {
  return (
    <html lang="en" className={roboto.className}>
      <body>{children}</body>
    </html>
  );
}
```

### 3.2. 可变字体

可变字体（Variable Fonts）是一种可在单个文件中包含多种字重、字宽、倾斜等变化的字体格式。引用维基百科：

> OpenType 可变字体（英语：OpenType variable fonts）是字体格式 OpenType 在 1.8 版规范中引入的扩展规范，由苹果、微软、谷歌和 Adobe 联合开发，于 2016 年 9 月 14 日正式发布。支持这一规范的计算机字体可以储存轮廓变化数据，在初始字形轮廓的基础上自动生成丰富的变化造型，使用户可以自由调整文字的外观。

下图对比了可变字体与传统字体的差异：

![20260206204425](https://raw.githubusercontent.com/xun082/md/main/blogs.images20260206204425.png)

简而言之，可变字体可在字宽、字重、倾斜等维度自由调节，一个文件即可呈现多种样式。相比传统字体需为每种样式单独提供文件，可变字体文件更少、体积更小，Next.js 推荐优先使用。

若需多种字重或样式，可使用数组指定：

```javascript
// app/layout.js
const roboto = Roboto({
  weight: ["400", "700"],
  style: ["normal", "italic"],
  subsets: ["latin"],
  display: "swap",
});
```

若字体名由多个单词组成（如 Roboto Mono），导入时需用下划线 `_` 连接，写成 `Roboto_Mono`：

```javascript
// app/layout.js
import { Ma_Shan_Zheng } from "next/font/google";

const font = Ma_Shan_Zheng({
  subsets: ["latin"],
  weight: "400",
});

export default function RootLayout({ children }) {
  return (
    <html lang="en" className={font.className}>
      <body>{children}</body>
    </html>
  );
}
```

### 3.3. 指定子集

Google 字体通常支持多种字符子集（subset），如 Roboto Mono 支持拉丁文、西里尔文、希腊文等。若只需部分字符，可通过 `subsets` 指定，从而减小字体体积、提升加载性能。

```javascript
// app/layout.js
const inter = Inter({ subsets: ["latin"] });
```

子集默认会被预加载（由 preload 控制）。若 preload 为 true 却未指定子集，会触发警告；部分字体仅有默认子集（如 latin），也需显式声明。

latin 即常见的 26 个英文字母，源自古罗马字母，是多数欧美语言的标准字符集，因此大多数字体都提供 latin 子集。

若不确定某字体支持哪些子集，可先随意指定一个；若无效，Next.js 会在构建时提示可用子集，例如：

![20260206200442](https://raw.githubusercontent.com/xun082/md/main/blogs.images20260206200442.png)

## next/font/local

使用本地字体时，通过 next/font/local 并使用 src 声明本地文件的地址。Next.js 依然推荐使用可变字体。使用示例如下

```javascript
// app/layout.js
import localFont from "next/font/local";

const myFont = localFont({
  src: "./my-font.woff2",
  display: "swap",
});

export default function RootLayout({ children }) {
  return (
    <html lang="en" className={myFont.className}>
      <body>{children}</body>
    </html>
  );
}
```

src 也可以是数组形式，用于一个字体使用多个本地文件的情况

```javascript
const roboto = localFont({
  src: [
    {
      path: "./Roboto-Regular.woff2",
      weight: "400",
      style: "normal",
    },
    {
      path: "./Roboto-Italic.woff2",
      weight: "400",
      style: "italic",
    },
    {
      path: "./Roboto-Bold.woff2",
      weight: "700",
      style: "normal",
    },
    {
      path: "./Roboto-BoldItalic.woff2",
      weight: "700",
      style: "italic",
    },
  ],
});
```

## 5. Font 函数参数

介绍完 `next/font` 的两种用法后，本节说明 Font 函数支持的参数。`next/font/google` 与 `next/font/local` 在参数上略有差异，下表为完整对比：

```javascript
import { Inter } from "next/font/google";

const inter = Inter({
  subsets: ["latin"],
  // 更多参数见下表
});
```

| 参数               | font/google | font/local | 类型                       | 必传   |
| ------------------ | ----------- | ---------- | -------------------------- | ------ |
| src                | ❌          | ✅         | String or Array of Objects | 是     |
| weight             | ✅          | ✅         | String or Array            | 看情况 |
| style              | ✅          | ✅         | String or Array            | -      |
| subsets            | ✅          | ❌         | Array of Strings           | -      |
| axes               | ✅          | ❌         | Array of Strings           | -      |
| display            | ✅          | ✅         | String                     | -      |
| preload            | ✅          | ✅         | Boolean                    | -      |
| fallback           | ✅          | ✅         | Array of Strings           | -      |
| adjustFontFallback | ✅          | ✅         | Boolean or String          | -      |
| variable           | ✅          | ✅         | String                     | -      |
| declarations       | ❌          | ✅         | Array of Objects           | -      |

### 5.1. src

在 `next/font/local` 中必传，可以是字符串，也可以是对象数组（类型为 `Array<{path: string, weight?: string, style?: string}>`），路径地址相对于字体加载函数调用的位置。

比如 `app/page.js` 中使用 `src:'./fonts/my-font.woff2'` 调用字体加载函数，`my-font.woff2` 就放置在 `app/fonts/` 下。

### 5.2. weight

字重，概念参考 `font-weight`。如果是可变字体则非必传，如果不是可变字体则必传。

值可以是字符串，如 `weight: '400'`、`weight: '100 900'`（可变字体，从 100 到 900 之间的范围），也可以是字符串数组，如 `weight: ['100','400','900']`（不可变字体的 3 个可能值）。

### 5.3. style

概念参考 `font-style`，默认值为 `normal`，其他值还有 `italic`、`oblique` 等。如果使用 `next/font/google` 的非可变字体，也可以传入一组样式值，如 `style: ['italic','normal']`。

### 5.4. subsets

子集的概念上节已经介绍过，就不多说了。

### 5.5. axes

axes（轴）与 `subsets` 一样，仅用于 `next/font/google`。可变字体可调节字宽、字重、倾斜等，这些维度即变形轴。以 [Inter 字体](https://fonts.google.com/variablefonts?vfquery=inter#font-families) 为例，其包含 `slnt`（slant，倾斜）和 `wght`（weight，字重）两个轴。可在 [wakamaifondue.com](https://wakamaifondue.com/) 上传字体文件，在线预览各轴调整效果。

`wght` 对应 CSS 的 `font-weight`，`slnt` 则控制倾斜程度。font-style 常用值有 `italic`（专门设计的斜体）和 `oblique`（原字形倾斜，可被模拟）。若指定其一但字体无对应样式会回退到另一；若都不可用，浏览器会模拟仿 oblique。`slnt` 轴即在 oblique 样式中控制倾斜角度的轴。常见变形轴与 CSS 对应关系如下：

| 轴         | 名称          | 对应 CSS            |
| ---------- | ------------- | ------------------- |
| wght       | Weight        | font-weight         |
| wdth       | Width         | font-stretch        |
| ital, slnt | Italic, Slant | font-style          |
| opsz       | Optical Size  | font-optical-sizing |

axes 是一个字符串数组形式，比如 `axes: ['slnt']`，你可以在[Google 可变字体页面](https://fonts.google.com/variablefonts#font-families)查询字体的 Axes 有哪些。之所以需要声明，是因为默认情况下，只有 weight 轴会被留下以减少字体文件大小，如果需要其他的轴就需要单独声明。

### 5.6. display

概念参考 `font-display`。默认我们加载字体的时候，使用该字体的地方会先显示空白，然后直到字体下载完成之后才会显示。CSS `font-display` 控制的就是这个过程。

下图展示了不同 display 值的字体加载策略：

![20260206204706](https://raw.githubusercontent.com/xun082/md/main/blogs.images20260206204706.png)

从流程图可以看出，不同 display 值在字体加载过程中的行为差异很大。swap 会立即显示备用字体，用户体验最好。block 会先显示空白，可能导致文字闪烁。fallback 和 optional 则在加载时间和用户体验之间做了权衡。

CSS `font-display` 的默认值为 `auto`，`next/font` 组件的默认值为 `swap`。

### 5.7. preload

布尔值，指定字体是否应该被预加载，默认值为 `true`。

### 5.8. fallback

字体无法被加载时的备用字体，没有默认值，字符串数组形式，如 `fallback: ['system-ui', 'arial']`。

### 5.9. adjustFontFallback

对于 `next/font/google`，`adjustFontFallback` 是一个布尔值，设置是否应该使用自动备用字体以减少累积布局偏移。默认值为 `true`。

对于 `next/font/local`，`adjustFontFallback` 可以是字符串，也可以是 `false`。可能的值有 `Arial`、`Times New Roman`、`false`。默认值是 `Arial`。

`Arial` 是经典的无衬线字体，`Times New Roman` 是经典的衬线字体，其实就是让你选用衬线还是无衬线字体作为备用字体进行调整，当然你也可以选择不调整，那就是 `false`。谷歌字体之所以不用选，是因为 Next.js 自动帮你判断了。

### 5.10. variable

通过 `variable` 可将字体声明为 CSS 变量（如 `--font-inter`），便于在样式表中用 `var(--font-inter)` 引用，并在多处复用。变量会随 `className` 挂载到 DOM，子元素通过继承获取。

```javascript
import { Inter } from "next/font/google";

const inter = Inter({
  subsets: ["latin"],
  variable: "--font-inter",
});
```

添加到 HTML 后，Next.js 会生成对应的 `@font-face` 规则，`--font-inter` 的值为 `'__Inter_a64ecd', '__Inter_Fallback_a64ecd'`，即主字体与调整过 metrics 的备用字体（Arial）组合：

```css
@font-face {
  font-family: "__Inter_a64ecd";
  font-style: normal;
  font-weight: 100 900;
  font-display: swap;
  src: url(/_next/static/media/c9a5bc6a7c948fb0-s.p.woff2) format("woff2");
}

@font-face {
  font-family: "__Inter_Fallback_a64ecd";
  src: local("Arial");
  ascent-override: 90.2%;
  descent-override: 22.48%;
  line-gap-override: 0%;
  size-adjust: 107.4%;
}
```

`ascent-override`、`descent-override`、`size-adjust` 用于让备用字体与主字体的垂直度量一致，减少布局偏移。若不需要备用字体，可设置 `adjustFontFallback: false`，此时变量仅包含主字体名。

使用方式：在 CSS 中通过 `var(--font-inter)` 引用，并确保变量已挂到父元素（子元素通过继承获取）：

```css
/* styles/component.module.css */
.text {
  font-family: var(--font-inter);
  font-weight: 200;
  font-style: italic;
}
```

```jsx
// app/page.js
<main className={inter.variable}>
  <p className={styles.text}>Hello World</p>
</main>
```

也可在根布局的 `html` 上挂载，供全局使用：

```jsx
// app/layout.js
import "./globals.css";
import { Inter } from "next/font/google";

const inter = Inter({
  subsets: ["latin"],
  variable: "--font-inter",
});

export default function RootLayout({ children }) {
  return (
    <html className={inter.variable}>
      <body>{children}</body>
    </html>
  );
}
```

### 5.11. declarations

`@font-face` 还支持 `ascent-override`、`descent-override`、`font-feature-settings`、`font-variation-settings`、`line-gap-override`、`unicode-range` 等属性。`declarations` 用于在 `next/font/local` 中自定义这些属性的生成：

```javascript
import localFont from "next/font/local";

const myFont = localFont({
  src: "./fonts/my-font.woff2",
  declarations: [{ prop: "ascent-override", value: "90%" }],
});
```

该属性仅支持 `next/font/local`。

## 6. 应用样式的三种方法

`next/font` 提供了三种方式来应用字体样式，`className`、`style` 和 CSS 变量。下图展示了三种方式的特点和使用场景：

![20260206204309](https://raw.githubusercontent.com/xun082/md/main/blogs.images20260206204309.png)

从图中可以看出，三种方式各有特点。`className` 最简单直接，适合全局应用。`style` 不需要额外的 CSS 文件，适合局部使用。CSS 变量最灵活，可以与 Tailwind CSS 等方案结合使用。

### 6.1. className

目前我们讲到的添加样式的方法都是通过 `className` 属性：

```tsx
import { Inter } from "next/font/google";
import type { ReactNode } from "react";

const inter = Inter({
  subsets: ["latin"],
});

export default function RootLayout({ children }: { children: ReactNode }) {
  return (
    <html lang="en" className={inter.className}>
      <body>{children}</body>
    </html>
  );
}
```

字体对象的 `className` 属性返回一个只读的 CSS 类名，由 Next.js 在构建时生成并注入到页面中。将该类名挂到 `html` 或任意父元素上，其子元素即可继承该字体。

下图展示了在 Chrome DevTools 中，`html` 元素上的生成类名及其对应样式：

![20260206203417](https://raw.githubusercontent.com/xun082/md/main/blogs.images20260206203417.png)

该类的样式大致如下，其中类名和字体族名均为 Next.js 自动生成的哈希值：

```css
.__className_a64ecd {
  font-family: "__Inter_a64ecd", "__Inter_Fallback_a64ecd";
  font-style: normal;
}

@font-face {
  font-family: "__Inter_a64ecd";
  /* font-weight、font-display、src、unicode-range 等 */
}

@font-face {
  font-family: "__Inter_Fallback_a64ecd";
  /* ascent-override、descent-override、size-adjust 等 */
}
```

### 6.2. style

`style` 返回一个只读的内联样式对象，包含 `font-family`、`font-style` 等属性。将其传给元素的 `style` 后，会直接渲染为 HTML 的 `style` 属性，无需额外 CSS 类：

```tsx
// app/layout.tsx
import { Inter } from "next/font/google";
import type { ReactNode } from "react";

const inter = Inter({
  subsets: ["latin"],
});

export default function RootLayout({ children }: { children: ReactNode }) {
  return (
    <html style={inter.style}>
      <body>{children}</body>
    </html>
  );
}
```

在 DevTools 中可看到，`html` 元素上会生成类似 `font-family: 'Inter', 'Inter Fallback'; font-style: normal` 的内联样式，对应的 `@font-face` 规则由 Next.js 注入到页面中：

## 总结

`next/font` 的核心优势有三点：

1. Google 字体直接导入，无需手动配置 CDN，构建时自动下载并托管为静态资源；
2. 优先使用可变字体，单文件即可覆盖多种字重与样式，减小体积；
3. 务必指定 `subsets`，按需加载字符集，提升加载性能。

使用建议：同一字体在多个组件中使用时，应在单一文件中定义并导出，按需导入，避免重复实例化；多种字体并存时，可通过 `variable` 声明 CSS 变量，在全局样式或 Tailwind 的 `fontFamily` 中引用，便于统一管理。
