Metadata（元数据）是描述数据的数据。在 HTML 中，`<head>` 元素中的各种标签都是元数据，如标题、描述、Open Graph 标签等。丰富的元数据可以改善 SEO 和 web 可共享性。

Next.js 16 提供了 Metadata API，让我们可以通过以下方式定义元数据：

1. 基于配置：在 `layout.tsx` 或 `page.tsx` 中导出静态 `metadata` 对象或动态 `generateMetadata` 函数
2. 基于文件：添加特殊文件（如 `favicon.ico`、`opengraph-image.jpg`、`robots.txt`、`sitemap.xml` 等）

通过这些选项，Next.js 会自动为页面生成相关的 `<head>` 标签。

## 1. 基于配置（Config-based）

### 1.1. 静态元数据

要定义静态元数据，从 `layout.tsx` 或 `page.tsx` 文件中导出一个 `Metadata` 对象：

```tsx
// app/blog/layout.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: "DocFlow",
  description: "...",
};

export default function Layout() {}
```

Metadata 对象包含多个字段（如 `title`、`description`、`openGraph`、`twitter` 等），我们会在后面详细介绍各个字段的用法。

### 1.2. 动态元数据

当元数据依赖于动态信息（如路由参数、外部数据、父级路由段的 metadata）时，我们可以使用 `generateMetadata` 函数来动态生成元数据。

```tsx
// app/blog/[slug]/page.tsx
import type { Metadata, ResolvingMetadata } from "next";

type Props = {
  params: Promise<{ slug: string }>;
  searchParams: Promise<{ [key: string]: string | string[] | undefined }>;
};

export async function generateMetadata(
  { params, searchParams }: Props,
  parent: ResolvingMetadata
): Promise<Metadata> {
  const slug = (await params).slug;

  // 获取数据
  const post = await fetch(`https://www.codecrack.cn/blog/${slug}`).then(
    (res) => res.json()
  );

  // 获取父路由段的 metadata
  const previousImages = (await parent).openGraph?.images || [];

  return {
    title: post.title,
    description: post.description,
    openGraph: {
      images: ["/docFlow.jpg", ...previousImages],
    },
  };
}

export default function Page({ params, searchParams }: Props) {}
```

`generateMetadata` 函数接收两个参数：

- `props`：包含 `params`（动态路由参数，Promise 类型）和 `searchParams`（URL 搜索参数，Promise 类型，仅在 `page.tsx` 中可用）

- `parent`：包含父路由段 metadata 的 Promise 对象

| 路由示例                         | URL         | `params`                           |
| -------------------------------- | ----------- | ---------------------------------- |
| `app/shop/[slug]/page.tsx`       | `/shop/1`   | `Promise<{ slug: '1' }>`           |
| `app/shop/[tag]/[item]/page.tsx` | `/shop/1/2` | `Promise<{ tag: '1', item: '2' }>` |
| `app/shop/[...slug]/page.tsx`    | `/shop/1/2` | `Promise<{ slug: ['1', '2'] }>`    |

> `metadata` 对象和 `generateMetadata` 函数仅在 Server Components 中支持。如果不依赖运行时信息，应尽可能使用静态元数据。

### 1.3. 流式元数据

对于动态渲染的页面，Next.js 会单独流式传输元数据，在 `generateMetadata` 解析后将其注入到 HTML 中，而不会阻塞 UI 渲染。这可以提升感知性能，让视觉内容先流式传输。

流式元数据对期望元数据在 `<head>` 标签中的爬虫（如 `Twitterbot`、`Slackbot`、`Bingbot`）会被禁用。Next.js 会通过请求的 User Agent 头来检测这些爬虫。

静态渲染的页面不使用流式传输，因为元数据在构建时已解析。

### 1.4. 数据请求记忆化

如果我们需要在 `generateMetadata` 和页面组件中获取相同的数据，可以使用 React 的 [`cache`](https://react.dev/reference/react/cache) 函数来记忆化返回值，避免重复请求：

```ts
// app/lib/data.ts
import { cache } from "react";
import { db } from "@/app/lib/db";

// getPost 会被调用两次，但只执行一次
export const getPost = cache(async (slug: string) => {
  const res = await db.query.posts.findFirst({ where: eq(posts.slug, slug) });
  return res;
});
```

```tsx
// app/blog/[slug]/page.tsx
import { getPost } from "@/app/lib/data";

export async function generateMetadata({
  params,
}: {
  params: Promise<{ slug: string }>;
}) {
  const { slug } = await params;
  const post = await getPost(slug);
  return {
    title: post.title,
    description: post.description,
  };
}

export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>;
}) {
  const { slug } = await params;
  const post = await getPost(slug);
  return <div>{post.title}</div>;
}
```

## 2. 基于文件（File-based）

Next.js 支持以下特殊文件用于元数据：

- `favicon.ico`、`apple-icon.jpg` 和 `icon.jpg`：用于设置网站图标
- `opengraph-image.jpg` 和 `twitter-image.jpg`：用于设置 Open Graph 和 Twitter 图片
- `robots.txt`：用于设置搜索引擎爬虫规则
- `sitemap.xml`：用于设置网站地图

这些文件可以是静态文件，也可以通过代码动态生成。例如，添加一个 `icon.(ico|jpg|jpeg|png|svg)` 文件，Next.js 会自动生成对应的图标元数据标签。

我们也可以使用 `icon.ts` 或 `icon.tsx` 文件来动态生成图标。其他文件同理，使用这种方式快捷直观。

> 基于文件的元数据具有更高的优先级，会覆盖任何基于配置的元数据。

## 3. 行为

### 3.1. 默认字段

有两个默认的 `meta` 标签，即使路由未定义元数据也会添加，一个是设置网站字符编码的 `charset`，一个是设置网站视口宽度和比例的 `viewport`：

```html
<meta charset="utf-8" />
<meta name="viewport" content="width=device-width, initial-scale=1" />
```

当然这两个元数据标签也都是可以覆盖的。

### 3.2. 元数据的合并规则

具体页面的设置优先级更高。Next.js 按照从根路由到 `page.tsx` 的顺序处理，例如访问 `/blog/1` 时的处理顺序为：

1. `app/layout.tsx`（根布局）

2. `app/blog/layout.tsx`（嵌套布局）

3. `app/blog/[slug]/page.tsx`（具体页面）

Next.js 会按照这个顺序，将各层元数据对象**浅合并**，计算得到最终的元数据。重复的字段会根据顺序进行替换。

例如，根布局的设置为：

```tsx
// app/layout.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: "DocFlow",
  openGraph: {
    title: "DocFlow",
    description: "DocFlow 是一个协同文档...",
  },
};
```

页面的设置为：

```tsx
// app/blog/page.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: "Blog",
  openGraph: {
    title: "Blog",
  },
};
```

输出结果：

![20251209180240](https://raw.githubusercontent.com/xun082/md/main/blogs.images20251209180240.png)

在这个例子中，`app/layout.tsx` 中的 `title` 被 `app/blog/page.tsx` 中的 `title` 替换，`openGraph` 字段也被替换。注意因为是浅合并，所以 `description` 字段会丢失。

如果页面没有设置某个字段，会继承父布局的该字段。例如：

如果需要在多个路由之间共享字段，可以将它们提取到单独的变量中，例如在 `app` 下建立 `shared-metadata.ts` 文件：

```ts
// app/shared-metadata.ts
import type { Metadata } from "next";

export const openGraphImage: Metadata["openGraph"] = {
  images: ["http://..."],
};
```

使用时：

```tsx
// app/page.tsx
import type { Metadata } from "next";
import { openGraphImage } from "./shared-metadata";

export const metadata: Metadata = {
  openGraph: {
    ...openGraphImage,
    title: "Home",
  },
};
```

```tsx
// app/about/page.tsx
import type { Metadata } from "next";
import { openGraphImage } from "../shared-metadata";

export const metadata: Metadata = {
  openGraph: {
    ...openGraphImage,
    title: "About",
  },
};
```

## 4. JSON-LD

JSON-LD（JSON for Linking Data）是一种基于 JSON 的数据格式，用于向 Google 和其他搜索引擎描述网站上的内容，帮助搜索引擎更好地理解页面内容并展示富媒体搜索结果。官方文档地址：[json-ld.org/](https://json-ld.org/)。

通过添加 JSON-LD 结构化数据，我们可以让搜索结果展示更丰富的信息。例如，当我们在 Google 搜索"番茄鸡蛋面"时，带有 JSON-LD 的页面会在搜索结果中显示评分（如 7.6/10，基于 6,153 条评价）、制作时间、食谱图片等富媒体信息，而没有 JSON-LD 的页面通常只显示标题、描述和链接。这些额外的富媒体信息能够显著提升搜索结果的可读性和吸引力，从而提高点击率和用户体验。

![20251209185805](https://raw.githubusercontent.com/xun082/md/main/blogs.images20251209185805.png)

在页面中添加结构化数据的方法很简单，只需添加一段 `<script type="application/ld+json">` 标签。具体内容需要参考 Google 搜索中心提供的[《结构化数据常规指南》](https://developers.google.com/search/docs/advanced/structured-data/sd-policies)。如果写的是文章，参考 [Article 章节](https://developers.google.com/search/docs/advanced/structured-data/article)后，可以写入以下属性：

```html
<script type="application/ld+json">
  {
    "@context": "https://schema.org",
    "@type": "Article",
    "headline": "文章标题",
    "description": "文章描述",
    "image": ["https://example.com/article-image.jpg"],
    "datePublished": "2024-01-15T10:00:00.000Z",
    "dateModified": "2024-01-20T15:30:00.000Z",
    "author": {
      "@type": "Person",
      "name": "作者姓名",
      "url": "https://example.com/author"
    },
    "publisher": {
      "@type": "Organization",
      "name": "网站名称",
      "logo": {
        "@type": "ImageObject",
        "url": "https://example.com/logo.png"
      }
    }
  }
</script>
```

在 Next.js 中添加 JSON-LD，推荐在 `layout.tsx` 或 `page.tsx` 组件中使用 `<script>` 标签：

```tsx
// app/products/[id]/page.tsx
export default async function Page({
  params,
}: {
  params: Promise<{ id: string }>;
}) {
  const { id } = await params;
  const product = await getProduct(id);

  const jsonLd = {
    "@context": "https://schema.org",
    "@type": "Product",
    name: product.name,
    image: product.image,
    description: product.description,
  };

  return (
    <section>
      <script
        type="application/ld+json"
        dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }}
      />
      {/* ... */}
    </section>
  );
}
```

如果使用 TypeScript，可以配合使用 [schema-dts](https://www.npmjs.com/package/schema-dts) 包来获得类型支持。

下面是一个更完整的示例，展示如何为食谱页面添加 JSON-LD，这样在搜索结果中就能显示评分、制作时间、卡路里等富媒体信息：

```tsx
// app/recipes/[slug]/page.tsx
export default async function Page({
  params,
}: {
  params: Promise<{ slug: string }>;
}) {
  const { slug } = await params;
  const recipe = await getRecipe(slug);

  const jsonLd = {
    "@context": "https://schema.org",
    "@type": "Recipe",
    name: recipe.name,
    description: recipe.description,
    image: recipe.image,
    author: {
      "@type": "Person",
      name: recipe.author,
    },
    datePublished: recipe.datePublished,
    prepTime: recipe.prepTime,
    cookTime: recipe.cookTime,
    totalTime: recipe.totalTime,
    recipeYield: recipe.servings,
    recipeIngredient: recipe.ingredients,
    recipeInstructions: recipe.instructions.map(
      (step: string, index: number) => ({
        "@type": "HowToStep",
        position: index + 1,
        text: step,
      })
    ),
    nutrition: {
      "@type": "NutritionInformation",
      calories: recipe.calories,
    },
    aggregateRating: {
      "@type": "AggregateRating",
      ratingValue: recipe.rating,
      reviewCount: recipe.reviewCount,
    },
  };

  return (
    <article>
      <script
        type="application/ld+json"
        dangerouslySetInnerHTML={{ __html: JSON.stringify(jsonLd) }}
      />
      {/* 页面内容 */}
    </article>
  );
}
```

添加 JSON-LD 并部署后，可以使用 [Google Rich Results Test](https://search.google.com/test/rich-results) 测试网页是否支持富媒体搜索结果，使用 [Schema.org Validator](https://validator.schema.org/) 测试页面的结构化数据。

## 5. Metadata 字段

Metadata 对象包含多个字段，用于定义页面的各种元数据。由于字段较多，我们可以先了解有哪些字段，需要时再查找使用。下面介绍主要字段及其用法。

### 5.1. title

`title` 用于设置 HTML 文档的 `<title>` 标签，这是浏览器标签页显示的标题，也是搜索引擎结果中显示的主要标题。该值可以是一个简单的字符串，也可以是一个包含 `default`、`template` 和 `absolute` 的模板对象，用于实现标题的继承和模板化。

#### 5.1.1. 字符串

```tsx
// layout.tsx | page.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: "DocFlow",
};
```

输出结果如下图所示：

![20251209190131](https://raw.githubusercontent.com/xun082/md/main/blogs.images20251209190131.png)

#### 5.1.2. 模板对象

```tsx
// app/layout.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: {
    default: "...",
    template: "...",
    absolute: "...",
  },
};
```

其中 `default` 为没有定义 `title` 的子路由提供一个备用 title：

```tsx
// app/layout.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: {
    default: "Moment",
  },
};
```

```tsx
// app/about/page.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {};

// Output: <title>Moment</title>
```

`template` 可用于为子路由的 `title` 添加一个前缀或者后缀，比如：

```tsx
// app/layout.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: {
    template: "%s | Moment",
    default: "Moment", // 设置 template 的时候，default 必传
  },
};
```

```tsx
// app/about/page.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: "About",
};
```

最终输出结果如下图所示：

![20251209193310](https://raw.githubusercontent.com/xun082/md/main/blogs.images20251209193310.png)

使用 `template` 时需要注意：

1. 使用 `title.template` 时，`title.default` 是必要的

2. 定义在 `layout.js` 中的 `title.template` 不会应用于同级的 `page.js` 中的 `title`

3. 定义在 `page.js` 中的 `title.template` 不会有效果，因为页面没有子路由段

4. 如果路由没有定义 `title` 或 `title.default`，`title.template` 不会生效

`absolute` 用于设置标题，但会忽略父路由段中设置的 `title.template`：

```tsx
// app/layout.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: {
    template: "%s | Moment",
  },
};
```

```tsx
// app/about/page.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  title: {
    absolute: "About",
  },
};
```

最终输出结果如下图所示：

![20251209193448](https://raw.githubusercontent.com/xun082/md/main/blogs.images20251209193448.png)

这三个值在 `layout.tsx` 和 `page.tsx` 中的表现会有不同，让我们细致梳理一下：

在 `layout.tsx` 中：

- `title`（字符串）和 `title.default` 定义未设置标题的子路由段的默认标题，会根据最近的父路由段中的 `title.template` 进行扩展
- `title.absolute` 定义子路由段的默认标题，会忽略来自父路由段的 `title.template`
- `title.template` 为子路由段定义标题模板，即使每层 layout 都设置了 `title.template`，也不会出现多层嵌套的标题

在 `page.tsx` 中：

- 如果页面没有定义标题，会使用最近的父级解析标题
- 如果定义了 `title`（字符串），会根据最近的父路由段中的 `title.template` 进行扩展
- 使用 `title.absolute` 定义标题，会忽略父路由段中的 `title.template`
- `page.tsx` 中设置 `title.template` 没有效果

> 建议尽可能保证每个页面都设置 `title`，避免遇到复杂的继承逻辑。

### 5.2. description

`description` 用于设置页面的元描述，对应 HTML 中的 `<meta name="description">` 标签。这个描述会出现在搜索引擎结果页面的摘要中，帮助用户了解页面内容。良好的描述可以提高点击率，建议控制在 120-160 个字符之间。

```tsx
// layout.tsx | page.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  description: "疯狂星期四",
};
```

### 5.3. 基础字段

基础字段包含多个常用的元数据属性，用于提供页面的基本信息：

- `generator`：标识生成页面的工具或框架
- `applicationName`：指定 Web 应用的名称
- `referrer`：控制 HTTP Referer 头的发送策略
- `keywords`：页面的关键词数组（注意：现代搜索引擎已不太依赖此字段）
- `authors`：页面作者信息，可以是字符串数组或包含 `name` 和 `url` 的对象数组
- `creator`：内容创建者
- `publisher`：内容发布者
- `formatDetection`：控制移动浏览器自动检测和格式化电话号码、邮箱地址等功能

```tsx
// layout.tsx | page.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  generator: "Next.js",
  applicationName: "Next.js",
  referrer: "origin-when-cross-origin",
  keywords: ["Next.js", "React", "JavaScript"],
  authors: [{ name: "Seb" }, { name: "Josh", url: "https://nextjs.org" }],
  creator: "DocFlow",
  publisher: "Moment",
  formatDetection: {
    email: false,
    address: false,
    telephone: false,
  },
};

export default function About() {
  return <div>1</div>;
}
```

### 5.4. metadataBase

`metadataBase` 用于设置所有相对 URL 的基础地址。当我们在 metadata 中使用相对路径（如 `/og-image.png`）时，Next.js 会自动将其与 `metadataBase` 组合成完整的绝对 URL。这避免了在每个页面都重复写完整的域名，特别适合多环境部署的场景。通常设置在根 `layout.tsx` 中，应用于所有子路由。

```tsx
// layout.tsx | page.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  metadataBase: new URL("https://acme.com"),
  alternates: {
    canonical: "/",
    languages: {
      "en-US": "/en-US",
      "de-DE": "/de-DE",
    },
  },
  openGraph: {
    images: "/og-image.png",
  },
};
```

输出的 HTML 为：

![20251209193947](https://raw.githubusercontent.com/xun082/md/main/blogs.images20251209193947.png)

关于 `metadataBase` 和 `metadata`，Next.js 会自动处理，并不是简单的字符串拼接。例如设置 `metadataBase` 为：

```tsx
// app/layout.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  metadataBase: new URL("https://acme.com"),
};
```

传入 `metadata` 的值时：

| **metadata 字段**                | **解析 URL**                     |
| -------------------------------- | -------------------------------- |
| /                                | <https://acme.com>               |
| ./                               | <https://acme.com>               |
| payments                         | <https://acme.com/payments>      |
| /payments                        | <https://acme.com/payments>      |
| ./payments                       | <https://acme.com/payments>      |
| ../payments                      | <https://acme.com/payments>      |
| <https://beta.acme.com/payments> | <https://beta.acme.com/payments> |

关于 `metadataBase`：

- 通常设置在 `app/layout.tsx`，应用于所有路由
- 可以设置更具体的地址，如 `https://app.acme.com`、`https://acme.com/start/from/here` 等
- 如果 `metadata` 提供了绝对地址，`metadataBase` 会被忽略
- 未配置 `metadataBase` 时，`metadata` 使用相对地址会报错
- Next.js 会自动处理 `metadataBase` 和相对地址间的多余斜线

### 5.5. openGraph

### 5.6. robots

`robots` 用于控制搜索引擎爬虫的索引和抓取行为。通过设置 `index`、`follow`、`nocache` 等选项，可以控制页面是否被索引、链接是否被跟踪、内容是否被缓存等。还可以为特定的搜索引擎（如 `googleBot`）设置不同的规则，实现精细化的 SEO 控制。

```tsx
      index: true,
      follow: false,
      noimageindex: true,
      "max-video-preview": -1,
      "max-image-preview": "large",
      "max-snippet": -1,
    },
  },
};
```

### 5.12. alternates

`alternates` 用于指定页面的替代版本和关联资源，包括规范 URL（canonical URL）、多语言版本、不同媒体类型的版本等。这对于多语言网站、移动端适配、RSS 订阅等场景非常重要，可以帮助搜索引擎理解页面之间的关系，避免重复内容问题。

```tsx
      "de-DE": "https://nextjs.org/de-DE",

    },
    types: {
      "application/rss+xml": "https://nextjs.org/rss",
    },
  },
};
```

### 5.13. appLinks

`appLinks` 用于配置应用链接（App Links），实现从网页到原生应用的深度链接。当用户点击链接时，如果设备上安装了对应的应用，可以直接打开应用并跳转到指定页面；如果没有安装，则打开网页版本。这对于提升用户体验和促进应用下载非常有用。

```tsx
    android: {
      package: "com.example.android/package",
      app_name: "app_name_android",
    },
    web: {
      url: "https://nextjs.org/web",
      should_fallback: true,
    },
  },
};
```

### 5.17. category

`category` 用于指定页面的分类或主题。这个字段可以帮助搜索引擎和内容管理系统对页面进行分类和组织，虽然现代搜索引擎不太依赖此字段，但在某些内容平台和 RSS 阅读器中仍然有用。

```tsx
// layout.tsx | page.tsx
import type { Metadata } from "next";
```

```html
<meta name="category" content="technology" />
```

### 5.18. other

`other` 用于定义自定义的元数据标签。当我们需要添加一些 Next.js Metadata API 不直接支持的元数据标签时，可以使用这个字段。它会将对象中的键值对转换为对应的 `<meta name="key" content="value">` 标签，提供了极大的灵活性来满足特殊需求。

```tsx
// layout.tsx | page.tsx
import type { Metadata } from "next";

export const metadata: Metadata = {
  other: {
    custom: "meta",
  },
};
```

输出的 HTML 为：

```html
<meta name="custom" content="meta" />
```

## 总结

Next.js 16 的 Metadata API 为我们提供了强大而灵活的元数据配置方案，通过静态 `metadata` 对象或动态 `generateMetadata` 函数，我们可以轻松控制页面在搜索引擎和社交媒体上的展示效果。合理配置 `title`、`description`、`openGraph`、`twitter` 等关键字段，结合 JSON-LD 结构化数据，能够显著提升网站的 SEO 排名和用户点击率。建议在项目初期就建立完善的元数据配置策略，让每个页面都能在搜索结果中脱颖而出。
