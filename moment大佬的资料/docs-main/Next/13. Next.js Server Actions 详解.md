在 Next.js `App Router` 中，`Server Actions` 是一种强大的服务器端功能，允许我们在服务器上执行异步函数来处理表单提交和数据变更。`Server Actions` 可以在 `Server Components` 和 `Client Components` 中使用，为我们提供了一种简单、安全的方式来处理服务器端逻辑。

`Server Actions` 是 React 18 和 Next.js 13+ 引入的重要特性，它们让我们可以直接在组件中定义服务器端函数，无需创建单独的 `API Route`。这不仅简化了代码结构，还提供了更好的类型安全性和开发体验。

本文将详细介绍 `Server Actions` 在 Next.js 中的工作原理、使用方式，以及如何在实际应用中有效使用它们。

## 什么是 Server Actions？

`Server Actions` 是在服务器上执行的异步函数。它们的主要特点包括：

1. 服务器端执行：所有逻辑在服务器端运行，不会暴露给客户端
2. 类型安全：与 TypeScript 完美集成，提供完整的类型检查
3. 渐进增强：支持在没有 JavaScript 的情况下工作
4. 自动优化：与 Next.js 缓存和重新验证机制集成
5. 安全性：内置 CSRF 保护和其他安全措施
6. HTTP 方法：在后台，`Server Actions` 使用 `POST` 方法，只有这个 HTTP 方法可以调用它们
7. 序列化要求：`Server Actions` 的参数和返回值必须可以被 React 序列化
8. 可重用性：`Server Actions` 是函数，可以在应用程序的任何地方重复使用
9. 运行时继承：`Server Actions` 继承它们所在页面或布局的运行时

`Server Actions` 使用 React 的 `"use server"` 指令来标记。这个指令可以放在函数内部（函数级）或文件顶部（模块级），用于标识哪些函数应该在服务器端执行。

## Server Actions 的声明方式

### 在 Server Components 中

在 `Server Components` 中，我们可以使用函数级或模块级的 `"use server"` 指令。要在函数内部内联定义 `Server Action`，在函数体顶部添加 `"use server"`：

```tsx
// app/page.tsx
// Server Component
export default function Page() {
  // Server Action
  async function create() {
    "use server";

    // 服务器端逻辑
    // ...
  }

  return <form action={create}>{/* 表单内容 */}</form>;
}
```

### 在 Client Components 中

`Client Components` 只能导入使用模块级 `"use server"` 指令的 `Server Actions`。要在 `Client Component` 中调用 `Server Action`，创建一个新文件，在文件顶部添加 `"use server"` 指令。该文件中的所有函数都会被标记为 `Server Actions`，可在 `Client Components` 和 `Server Components` 中重复使用：

```tsx
// app/actions.ts
"use server";

export async function create() {
  // 服务器端逻辑
  // ...
}
```

```tsx
// app/ui/button.tsx
"use client";

import { create } from "@/app/actions";

export function Button() {
  return <form action={create}>{/* 表单内容 */}</form>;
}
```

我们也可以将 `Server Action` 作为 `props` 传递给 `Client Component`：

```tsx
// app/page.tsx
import ClientComponent from "./client-component";
import { updateItem } from "./actions";

export default function Page() {
  return <ClientComponent updateItem={updateItem} />;
}
```

```tsx
// app/client-component.tsx
"use client";

export default function ClientComponent({ updateItem }) {
  return <form action={updateItem}>{/* ... */}</form>;
}
```

## 使用示例

### 表单处理

React 扩展了 HTML `<form>` 元素，允许使用 `action` 属性调用 `Server Actions`。在表单中调用时，`action` 会自动接收 `FormData` 对象。我们无需使用 React `useState` 管理字段，可直接使用原生的 `FormData` 方法提取数据。

在 `Server Components` 中，表单默认支持渐进增强，即使 JavaScript 未加载或禁用，表单也能正常提交。在 `Client Components` 中，如果 JavaScript 尚未加载，表单提交会被排队，优先进行客户端 hydration。hydration 完成后，表单提交时浏览器不会刷新页面。

```tsx
// app/page.tsx
import { revalidatePath } from "next/cache";

export default function Page() {
  async function createInvoice(formData: FormData) {
    "use server";

    const rawFormData = {
      customerId: formData.get("customerId") as string,
      amount: Number(formData.get("amount")),
      status: formData.get("status") as string,
    };

    // 保存到数据库
    // await saveInvoice(rawFormData);

    // 重新验证缓存
    revalidatePath("/invoices");
  }

  return (
    <form action={createInvoice}>
      <input type="text" name="customerId" placeholder="客户ID" required />
      <input type="number" name="amount" placeholder="金额" required />
      <select name="status">
        <option value="pending">待处理</option>
        <option value="paid">已支付</option>
      </select>
      <button type="submit">创建发票</button>
    </form>
  );
}
```

当表单字段较多时，可使用 `entries()` 方法和 JavaScript 的 `Object.fromEntries()`，例如：`const rawFormData = Object.fromEntries(formData.entries())`

#### 传递额外参数

我们可以使用 JavaScript 的 `bind` 方法向 `Server Action` 传递额外参数：

```tsx
// app/client-component.tsx
"use client";

import { updateUser } from "./actions";

export function UserProfile({ userId }: { userId: string }) {
  const updateUserWithId = updateUser.bind(null, userId);

  return (
    <form action={updateUserWithId}>
      <input type="text" name="name" />
      <button type="submit">Update User Name</button>
    </form>
  );
}
```

`Server Action` 将接收 `userId` 参数以及表单数据：

```ts
// app/actions.ts
"use server";

import { revalidatePath } from "next/cache";

export async function updateUser(userId: string, formData: FormData) {
  const name = formData.get("name") as string;

  // 更新用户信息
  // await db.user.update({ where: { id: userId }, data: { name } });

  revalidatePath(`/users/${userId}`);
}
```

另一种方法是在表单中传递参数作为隐藏输入字段（例如 `<input type="hidden" name="userId" value={userId} />`），但该值会成为渲染 HTML 的一部分，不会被编码。`.bind` 在 `Server Components` 和 `Client Components` 中都可以工作，并支持渐进增强。

#### 待处理状态

我们可以使用 React 的 `useFormStatus` Hook 显示表单提交时的待处理状态。`useFormStatus` 返回特定 `<form>` 的状态，必须定义为 `<form>` 元素的子元素。由于 `useFormStatus` 是 React Hook，必须在 `Client Component` 中使用：

```tsx
// app/submit-button.tsx
"use client";

import { useFormStatus } from "react-dom";

export function SubmitButton() {
  const { pending } = useFormStatus();

  return (
    <button type="submit" aria-disabled={pending}>
      {pending ? "提交中..." : "提交"}
    </button>
  );
}
```

```tsx
// app/form.tsx
import { SubmitButton } from "./submit-button";
import { revalidatePath } from "next/cache";

export default function Form() {
  async function create(formData: FormData) {
    "use server";

    const name = formData.get("name") as string;

    // 创建数据
    // await db.item.create({ data: { name } });

    revalidatePath("/items");
  }

  return (
    <form action={create}>
      <input type="text" name="name" placeholder="名称" required />
      <SubmitButton />
    </form>
  );
}
```

这个示例展示了如何使用 `useFormStatus` Hook 来显示表单提交的待处理状态。`SubmitButton` 组件是一个 `Client Component`，它使用 `useFormStatus` 获取当前表单的提交状态。当表单正在提交时，`pending` 为 `true`，按钮会显示"提交中..."并禁用，防止重复提交。当提交完成后，`pending` 变为 `false`，按钮恢复为"提交"状态。这样可以为用户提供清晰的反馈，提升用户体验。

#### 表单状态和错误处理

我们可以使用 React 的 `useFormState` Hook 处理表单状态和错误：

```tsx
// app/actions.ts
"use server";

import { revalidatePath } from "next/cache";

interface State {
  message?: string;
  errors?: {
    name?: string;
  };
}

export async function createUser(
  prevState: State | null,
  formData: FormData
): Promise<State> {
  const name = formData.get("name") as string;

  // 验证
  if (!name || name.length < 2) {
    return {
      errors: {
        name: "名称至少需要2个字符",
      },
    };
  }

  try {
    // 创建用户
    // await db.user.create({ data: { name } });

    revalidatePath("/users");
    return { message: "用户创建成功" };
  } catch (error) {
    return {
      message:
        "创建失败: " + (error instanceof Error ? error.message : "未知错误"),
    };
  }
}
```

```tsx
// app/form.tsx
"use client";

import { useFormState } from "react-dom";
import { createUser } from "./actions";

export default function Form() {
  const [state, formAction] = useFormState(createUser, null);

  return (
    <form action={formAction}>
      <div>
        <label htmlFor="name">用户名</label>
        <input type="text" id="name" name="name" required />
        {state?.errors?.name && (
          <p style={{ color: "red" }}>{state.errors.name}</p>
        )}
      </div>
      <button type="submit">创建用户</button>
      {state?.message && <p style={{ color: "green" }}>{state.message}</p>}
    </form>
  );
}
```

这个示例展示了如何使用 `useFormState` Hook 处理表单状态和错误。`createUser` `Server Action` 接收 `prevState` 和 `formData` 作为参数，返回一个 `State` 对象，可以包含成功消息或错误信息。在 `Server Action` 中，我们首先进行输入验证，如果验证失败，返回包含 `errors` 的对象；如果验证通过，执行创建操作，成功时返回包含 `message` 的对象。在客户端，`useFormState` Hook 接收 `Server Action` 和初始状态，返回当前状态和增强后的 `formAction`。当表单提交后，如果 `state` 包含 `errors`，会在输入框下方显示红色错误提示；如果包含 `message`，会显示绿色成功提示。这样可以为用户提供清晰的反馈，无论是验证错误还是操作结果。

### 从事件处理器调用

`Server Actions` 不仅限于 `<form>`，还可以从事件处理器、`useEffect`、第三方库和其他表单元素（如 `<button>`）中调用：

```tsx
// app/ui/delete-button.tsx
"use client";

import { deleteItem } from "@/app/actions";
import { useTransition } from "react";

export function DeleteButton({ id }: { id: string }) {
  const [isPending, startTransition] = useTransition();

  return (
    <button
      onClick={() => {
        if (confirm("确定要删除吗？")) {
          startTransition(async () => {
            await deleteItem(id);
          });
        }
      }}
      disabled={isPending}
    >
      {isPending ? "删除中..." : "删除"}
    </button>
  );
}
```

```tsx
// app/actions.ts
"use server";

import { revalidatePath } from "next/cache";

export async function deleteItem(id: string) {
  // 删除数据
  // await db.item.delete({ where: { id } });

  revalidatePath("/items");
}
```

这个示例展示了如何从事件处理器中调用 `Server Action`。`DeleteButton` 组件在点击时调用 `deleteItem` `Server Action`，使用 `useTransition` 管理待处理状态。当用户点击删除按钮时，会先显示确认对话框，确认后执行删除操作，按钮会显示"删除中..."并禁用，防止重复操作。删除完成后，通过 `revalidatePath` 更新缓存，确保页面显示最新数据。

### 使用 useTransition

我们可以使用 `useTransition` Hook 管理 `Server Action` 的待处理状态：

```tsx
// app/ui/button.tsx
"use client";

import { useTransition } from "react";
import { updateItem } from "@/app/actions";

export function UpdateButton({ id }: { id: string }) {
  const [isPending, startTransition] = useTransition();

  const handleUpdate = () => {
    startTransition(async () => {
      try {
        await updateItem(id);
        // 可以添加成功提示
      } catch (error) {
        // 处理错误
        console.error("更新失败:", error);
      }
    });
  };

  return (
    <button onClick={handleUpdate} disabled={isPending}>
      {isPending ? "更新中..." : "更新"}
    </button>
  );
}
```

```tsx
// app/actions.ts
"use server";

import { revalidatePath } from "next/cache";

export async function updateItem(id: string) {
  // 更新数据
  // await db.item.update({ where: { id }, data: { updatedAt: new Date() } });

  revalidatePath(`/items/${id}`);
}
```

这个示例展示了如何使用 `useTransition` Hook 管理 `Server Action` 的待处理状态。`UpdateButton` 组件使用 `useTransition` 获取 `isPending` 状态和 `startTransition` 函数，在点击按钮时通过 `startTransition` 调用 `updateItem` `Server Action`。当操作进行时，`isPending` 为 `true`，按钮显示"更新中..."并禁用；操作完成后，按钮恢复为"更新"状态。这种方式可以让我们在非表单场景下也能很好地管理异步操作的待处理状态，并提供用户反馈。

### 乐观更新

我们可以使用 React 的 `useOptimistic` Hook 实现乐观更新，在服务器响应之前立即更新 UI：

```tsx
// app/ui/optimistic-form.tsx
"use client";

import { useOptimistic } from "react";
import { addComment } from "@/app/actions";

interface Comment {
  id: string;
  text: string;
  createdAt?: Date;
}

export function OptimisticForm({ comments }: { comments: Comment[] }) {
  const [optimisticComments, addOptimisticComment] = useOptimistic(
    comments,
    (state: Comment[], newComment: Comment) => [...state, newComment]
  );

  async function formAction(formData: FormData) {
    const text = formData.get("text") as string;
    const comment: Comment = {
      id: Date.now().toString(),
      text,
    };

    addOptimisticComment(comment);
    await addComment(comment);
  }

  return (
    <div>
      <form action={formAction}>
        <input type="text" name="text" placeholder="输入评论" required />
        <button type="submit">添加评论</button>
      </form>
      <ul>
        {optimisticComments.map((comment) => (
          <li key={comment.id}>{comment.text}</li>
        ))}
      </ul>
    </div>
  );
}
```

```tsx
// app/actions.ts
"use server";

import { revalidatePath } from "next/cache";

interface Comment {
  id: string;
  text: string;
}

export async function addComment(comment: Comment) {
  // 保存评论到数据库
  // await db.comment.create({ data: comment });

  revalidatePath("/comments");
}
```

这个示例展示了如何使用 `useOptimistic` Hook 实现乐观更新。`OptimisticForm` 组件使用 `useOptimistic` 创建一个乐观状态，当用户提交评论时，先通过 `addOptimisticComment` 立即将新评论添加到列表中，然后调用 `addComment` `Server Action` 保存到服务器。这样用户可以在服务器响应之前就看到新评论，提供更流畅的交互体验。如果服务器操作失败，React 会自动回滚到之前的状态。

## 数据变更和缓存重新验证

`Server Actions` 与 Next.js 的缓存和重新验证架构集成。当调用 `action` 时，Next.js 可以在单个服务器往返中返回更新的 UI 和新数据。

### 重新验证路径

我们可以在 `Server Actions` 中使用 `revalidatePath` API 重新验证 Next.js 缓存：

```ts
// app/actions.ts
"use server";

import { revalidatePath } from "next/cache";

export async function createPost() {
  try {
    // 创建帖子逻辑
    // ...
  } catch (error) {
    // 错误处理
    // ...
  }

  revalidatePath("/posts");
}
```

### 重新验证标签

或者，我们可以使用 `revalidateTag` 通过缓存标签使特定的数据获取失效：

```ts
// app/actions.ts
"use server";

import { revalidateTag } from "next/cache";

export async function createPost() {
  try {
    // 创建帖子逻辑
    // ...
  } catch (error) {
    // 错误处理
    // ...
  }

  revalidateTag("posts");
}
```

### 重定向

如果希望在 `Server Action` 完成后将用户重定向到不同路由，可使用 `redirect` API。`redirect` 需要在 `try/catch` 块外部调用：

```ts
// app/actions.ts
"use server";

import { redirect } from "next/navigation";
import { revalidateTag } from "next/cache";

export async function createPost(id: string) {
  try {
    // 创建帖子逻辑
    // ...
  } catch (error) {
    // 错误处理
    // ...
  }

  revalidateTag("posts"); // 更新缓存的帖子
  redirect(`/post/${id}`); // 导航到新帖子页面
}
```

### Cookies

我们可以在 `Server Action` 中使用 `cookies` API 获取、设置和删除 cookies：

```ts
// app/actions.ts
"use server";

import { cookies } from "next/headers";

export async function exampleAction() {
  // 获取 cookie
  const value = cookies().get("name")?.value;

  // 设置 cookie
  cookies().set("name", "Delba");

  // 删除 cookie
  cookies().delete("name");
}
```

## 总结

`Server Actions` 是 Next.js `App Router` 中处理服务器端逻辑的强大工具。它们让我们可以直接在组件中定义服务器端函数，无需创建单独的 `API Route`，从而简化了代码结构并提供了更好的开发体验。

通过合理使用 `Server Actions`，我们可以构建高性能、安全、用户友好的全栈应用。它们与 Next.js 的缓存和重新验证机制完美集成，支持渐进增强，并提供了内置的安全保护。在安全性方面，我们应该像对待 API 端点一样对待 `Server Actions`，确保进行身份验证和授权检查。Next.js 会自动加密闭包变量，使用 `POST` 方法防止 CSRF 攻击，并通过 Origin header 验证请求来源。对于使用反向代理的场景，可以通过 `serverActions.allowedOrigins` 配置允许的来源。在实际开发中，我们应该注意适当的错误处理、输入验证、权限检查和缓存管理，通过充分利用 `Server Actions` 的特性，构建更好的 Web 应用程序。
