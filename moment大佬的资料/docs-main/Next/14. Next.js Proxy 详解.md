在 Next.js `App Router` 中，`Proxy` 是一个强大的功能，允许我们在请求完成之前运行代码。基于传入的请求，我们可以通过重写、重定向、修改请求或响应头，或者直接响应来修改响应。

从 Next.js 16 开始，`Middleware` 改名为 `Proxy`，以更好地反映其用途，但功能保持不变。`Proxy` 在请求和响应之间提供了一个拦截层，让我们可以在请求到达页面之前对其进行处理。

本文将详细介绍 `Proxy` 在 Next.js 中的工作原理、使用方式、常见场景，以及如何在实际应用中有效使用它。

## 什么是 Proxy？

`Proxy` 是在请求完成之前运行的代码。它允许我们基于传入的请求修改响应，包括：

1. 修改请求或响应头
2. 重写 URL 到不同的页面
3. 基于请求属性进行程序化重定向
4. 直接响应请求

`Proxy` 运行在 Edge Runtime 中，这是服务端环境，在请求到达页面之前执行。这意味着：

- 运行在服务端，不是客户端
- 不能使用浏览器 API，如 `localStorage`、`window`、`document`、`sessionStorage` 等
- 可以使用大多数 Web API，如 `fetch`、`Headers`、`URL` 等
- 非常快速，可以在全球边缘网络上运行，为所有用户提供低延迟的体验
- 在请求到达页面组件之前执行，可以拦截和处理所有请求

## Proxy 的约定

### 文件位置

创建一个 `proxy.ts`（或 `.js`）文件在项目根目录，或者如果在 `src` 目录中，则放在 `src` 目录下，使其与 `pages` 或 `app` 处于同一级别。

> 注意：虽然每个项目只支持一个 `proxy.ts` 文件，但我们仍然可以将 `Proxy` 逻辑组织成模块。将 `Proxy` 功能拆分到单独的 `.ts` 或 `.js` 文件中，然后导入到主 `proxy.ts` 文件中。这样可以更清晰地管理特定路由的 `Proxy`，在 `proxy.ts` 中集中控制。通过强制使用单个 `Proxy` 文件，可以简化配置，避免潜在冲突，并通过避免多个 `Proxy` 层来优化性能。

### 导出方式

我们可以将 `Proxy` 函数导出为默认导出或命名 `proxy` 导出：

```ts
// proxy.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

// 命名导出
export function proxy(request: NextRequest) {
  return NextResponse.redirect(new URL("/home", request.url));
}

// 或者使用默认导出
// export default function proxy(request: NextRequest) { ... }
```

## 基本使用

### 路径匹配

使用 `config.matcher` 来指定 `Proxy` 应该运行在哪些路径上：

```ts
// proxy.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function proxy(request: NextRequest) {
  return NextResponse.next();
}

export const config = {
  // 匹配单个路径
  matcher: "/about",

  // 匹配多个路径
  // matcher: ['/about', '/contact'],

  // 匹配路径模式（匹配所有以 /dashboard 开头的路径）
  // matcher: '/dashboard/:path*',

  // 排除静态文件和 API 路由
  // matcher: ['/((?!api|_next/static|_next/image|favicon.ico|.*\\.).*)',],
};
```

### NextResponse API

`NextResponse` 提供了多种方法来处理响应。

使用 `NextResponse.next()` 继续处理请求，可以修改请求或响应：

```ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function proxy(request: NextRequest) {
  const response = NextResponse.next();
  response.headers.set("x-custom-header", "Moment");
  return response;
}
```

通过图片我们可以看到我们通过 `NextResponse.next()` 添加了一个自定义的请求头。

使用 `NextResponse.redirect()` 重定向到不同的 URL：

```ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function proxy(request: NextRequest) {
  return NextResponse.redirect(new URL("/login", request.url));
}
```

使用 `NextResponse.rewrite()` 重写 URL 而不改变浏览器地址栏：

```ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function proxy(request: NextRequest) {
  return NextResponse.rewrite(new URL("/internal-page", request.url));
}
```

使用 `NextResponse.json()` 直接返回 JSON 响应：

```ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function proxy(request: NextRequest) {
  return NextResponse.json({ message: "Hello from Proxy" });
}
```

## 使用示例

### 国际化重定向

基于用户的语言偏好或地理位置进行重定向：

```ts
// proxy.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function proxy(request: NextRequest) {
  const pathname = request.nextUrl.pathname;

  // 如果路径已经包含语言代码，跳过
  if (pathname.startsWith("/zh") || pathname.startsWith("/en")) {
    return NextResponse.next();
  }

  // 从 Accept-Language 头获取语言偏好
  const acceptLanguage = request.headers.get("accept-language") || "en";
  const preferredLang = acceptLanguage.split(",")[0].split("-")[0];

  // 重定向到对应的语言版本
  if (preferredLang === "zh") {
    return NextResponse.redirect(new URL("/zh" + pathname, request.url));
  }

  return NextResponse.next();
}

export const config = {
  matcher: ["/((?!api|_next/static|_next/image|favicon.ico).*)"],
};
```

### API 请求添加认证头

为所有 API 请求添加认证头：

```ts
// proxy.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function proxy(request: NextRequest) {
  // 只处理 API 路由
  if (request.nextUrl.pathname.startsWith("/api")) {
    const requestHeaders = new Headers(request.headers);

    // 从 cookie 获取 token
    const token = request.cookies.get("auth-token")?.value;

    if (token) {
      requestHeaders.set("Authorization", `Bearer ${token}`);
    }

    return NextResponse.next({
      request: {
        headers: requestHeaders,
      },
    });
  }

  return NextResponse.next();
}

export const config = {
  matcher: "/api/:path*",
};
```

### 权限检查与重定向

`Proxy` 运行在服务端，不能使用 `localStorage`，但可以使用 `cookies` 来存储和读取认证信息：

```ts
// proxy.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function proxy(request: NextRequest) {
  // 从 cookies 读取认证 token（服务端环境，不能使用 localStorage）
  const token = request.cookies.get("auth-token");

  // 如果未登录，重定向到登录页
  if (!token && request.nextUrl.pathname.startsWith("/dashboard")) {
    return NextResponse.redirect(new URL("/login", request.url));
  }

  // 如果已登录，访问登录页时重定向到首页
  if (token && request.nextUrl.pathname === "/login") {
    return NextResponse.redirect(new URL("/", request.url));
  }

  return NextResponse.next();
}
```

### 维护模式

```ts
// proxy.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

const isMaintenanceMode = process.env.MAINTENANCE_MODE === "true";

export function proxy(request: NextRequest) {
  // 允许访问维护页面和管理员路径
  if (
    request.nextUrl.pathname.startsWith("/maintenance") ||
    request.nextUrl.pathname.startsWith("/admin")
  ) {
    return NextResponse.next();
  }

  // 维护模式下重定向到维护页面
  if (isMaintenanceMode) {
    return NextResponse.redirect(new URL("/maintenance", request.url));
  }

  return NextResponse.next();
}
```

### A/B 测试

```ts
// proxy.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function proxy(request: NextRequest) {
  const pathname = request.nextUrl.pathname;

  // 只对特定路径进行 A/B 测试
  if (pathname === "/") {
    // 基于 cookie 或随机数决定版本
    const variant =
      request.cookies.get("ab-variant")?.value ||
      (Math.random() > 0.5 ? "a" : "b");

    // 设置 cookie
    const response = NextResponse.next();
    response.cookies.set("ab-variant", variant);

    // 重写到不同版本
    if (variant === "b") {
      return NextResponse.rewrite(new URL("/home-variant-b", request.url));
    }
  }

  return NextResponse.next();
}
```

## 总结

`Proxy` 是 Next.js `App Router` 中处理请求和响应的强大工具。它让我们可以在请求完成之前运行代码，基于传入的请求修改响应。从 Next.js 16 开始，`Middleware` 改名为 `Proxy`，以更好地反映其用途。

通过合理使用 `Proxy`，我们可以实现国际化重定向、权限检查、A/B 测试、请求头修改等功能。`Proxy` 运行在 Edge Runtime（服务端环境）中，提供快速、低延迟的体验。

在使用时需要注意以下几点：`Proxy` 运行在 Edge Runtime（服务端环境）中，不能使用 Node.js API（如 `fs`、`path`、`crypto` 等）和浏览器 API（如 `localStorage`、`window`、`document`、`sessionStorage` 等），如果需要访问客户端存储应该使用 `cookies` API。不要用于慢速数据获取，包大小限制在 1MB 以内，执行时间有限制。在 `Proxy` 中使用 `fetch` 时，缓存选项（`options.cache`、`options.next.revalidate`、`options.next.tags`）没有效果。对于简单的重定向，应该使用 `next.config.ts` 中的 `redirects` 配置，只有在需要访问请求数据或更复杂逻辑时才使用 `Proxy`。

通过充分利用 `Proxy` 的特性，可以构建更好的 Web 应用程序。
