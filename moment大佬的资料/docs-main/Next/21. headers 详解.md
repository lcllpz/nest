Next.js 提供的 `headers` 是一个异步函数，用于在 [Server Component](https://nextjs.org/docs/app/getting-started/server-and-client-components) 中读取 HTTP 请求头。常用于获取 User-Agent、Authorization、语言偏好等信息，或将请求头转发给下游 API。

本文基于 [Next.js 官方文档](https://nextjs.org/docs/app/api-reference/functions/headers) 详解 `headers` 的用法与注意事项。

## headers 基本用法

从 `next/headers` 导入 `headers`，在异步函数中调用并获取只读的 [Web Headers](https://developer.mozilla.org/docs/Web/API/Headers) 对象：

```tsx
// app/page.tsx
import { headers } from "next/headers";

export default async function Page() {
  const headersList = await headers();

  const allHeaders = Object.fromEntries(headersList.entries());
  console.log(allHeaders);

  return (
    <div>
      <h2>全部请求头</h2>
      <pre>{JSON.stringify(allHeaders, null, 2)}</pre>
    </div>
  );
}
```

`headers()` 无参数，返回一个 Promise，必须使用 `async/await` 或 React 的 [`use`](https://react.dev/reference/react/use) 获取。返回的 [Web Headers](https://developer.mozilla.org/docs/Web/API/Headers) 对象为只读，支持 `get`、`has`、`keys`、`values`、`entries`、`forEach` 等标准方法，无法调用 `set` 或 `delete` 修改响应头。

## 常见使用场景

在 `await headers()` 获取 `headersList` 后，按需调用 `get`、`has`、`entries` 等方法即可。

- 获取 User-Agent：`headersList.get("user-agent")`
- 转发 Authorization：`headersList.get("authorization")` 后传入 `fetch` 的 `headers` 选项
- 获取客户端 IP：反向代理场景下使用 `x-forwarded-for` 或 `x-real-ip`，如 `headersList.get("x-real-ip") ?? headersList.get("x-forwarded-for")?.split(",")[0]`
- 获取语言偏好：从 `accept-language` 解析，如 `headersList.get("accept-language")?.split(",")[0]?.split("-")[0]`
- 遍历所有请求头：基本用法已展示 `headersList.entries()` 或 `Object.fromEntries(headersList.entries())`
- 检查存在：`headersList.has("x-mobile-app")` 等

## 总结

`headers()` 是 Next.js 在服务端读取 HTTP 请求头的异步 API，返回只读的 [Web Headers](https://developer.mozilla.org/docs/Web/API/Headers) 对象，支持 `get`、`has`、`keys`、`values`、`entries`、`forEach` 等标准方法。Next.js 15 起须用 `async/await` 或 `use()` 调用，v14 及更早为同步，v15 兼容但将废弃。`headers` 属于 [Dynamic API](https://nextjs.org/docs/app/guides/caching#dynamic-rendering)，使用会使路由进入动态渲染；只能在 Server Component、Server Action、Route Handler 等服务端环境使用，不能在 Client Component 中调用。返回的 Headers 对象只读，无法 `set` 或 `delete`，若需设置响应头应在 [Route Handler](https://nextjs.org/docs/app/api-reference/file-conventions/route) 或 [Middleware](https://nextjs.org/docs/app/building-your-application/routing/middleware) 中使用 `Response` 的 `headers`。

与 `cookies` 的区别：`headers` 仅读取请求头，常用于 User-Agent、Authorization、IP、语言偏好等；`cookies` 可读写，适用于主题、会话等需客户端持久化的状态。
