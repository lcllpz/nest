在 Next.js `App Router` 中，默认情况下 `layout.tsx` 和 `page.tsx` 都是 `Server Components`，这让我们可以在服务器端获取数据、渲染 UI、缓存结果，并流式传输到客户端。当需要交互性或浏览器 API 时，可以使用 `Client Components` 来添加功能。

`Server Components` 是 React 18 引入的一项革命性特性，它允许我们在服务器端渲染组件，并将结果发送到客户端，从而显著减少客户端 JavaScript 包的大小，提升应用性能。在 Next.js 13+ 的 `App Router` 中，`Server Components` 是默认的渲染模式。

本文将详细介绍 `Server Components` 和 `Client Components` 在 Next.js 中的工作原理、使用场景、最佳实践，以及如何在应用中组合使用它们。

## 什么是 Server Components？

`Server Components` 是在服务器上运行的 React 组件。它们的主要特点包括：

1. 零客户端 JavaScript：`Server Components` 的代码不会被打包到客户端 JavaScript 中，从而减少包大小
2. 直接访问服务器资源：可以直接访问数据库、文件系统、环境变量等服务器资源
3. 自动代码分割：每个 `Server Component` 都是独立的代码块，自动进行代码分割
4. 流式渲染：支持流式传输，用户可以更快看到内容

`Server Components` 与传统的 `SSR`（Server-Side Rendering）不同。`SSR` 仍然需要将组件代码发送到客户端进行 `hydration`，而 `Server Components` 完全在服务器端运行，只将渲染结果发送到客户端。

## 什么是 Client Components？

`Client Components` 是在客户端（浏览器）中运行的 React 组件。它们的特点包括：

1. 完整的 React 功能：可以使用所有 React Hooks（`useState`、`useEffect`、`useContext` 等）
2. 浏览器 API 访问：可以访问 `window`、`document`、`localStorage` 等浏览器 API
3. 交互性：可以处理用户事件（点击、输入等）
4. 状态管理：可以管理组件内部状态

`Client Components` 需要在文件顶部添加 `"use client"` 指令来声明。

## 何时使用 Server Components 和 Client Components？

客户端和服务器环境具有不同的能力。`Server Components` 和 `Client Components` 允许我们根据使用场景在各自环境中运行逻辑。

### 使用 Client Components 的场景

当我们需要以下功能时，应使用 `Client Components`：

#### 1. State 和事件处理器

需要管理组件内部状态或处理用户交互时，使用 `useState` 和事件处理器：

```tsx
// app/ui/counter.tsx
"use client";

import { useState } from "react";

export default function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>{count} 次点击</p>
      <button onClick={() => setCount(count + 1)}>点击我</button>
    </div>
  );
}
```

#### 2. 生命周期逻辑

需要使用 `useEffect`、`useLayoutEffect` 等生命周期 Hooks 时：

```tsx
// app/ui/theme-toggle.tsx
"use client";

import { useEffect, useState } from "react";

export default function ThemeToggle() {
  const [theme, setTheme] = useState("light");

  useEffect(() => {
    const savedTheme = localStorage.getItem("theme");
    if (savedTheme) {
      setTheme(savedTheme);
    }
  }, []);

  useEffect(() => {
    document.documentElement.setAttribute("data-theme", theme);
    localStorage.setItem("theme", theme);
  }, [theme]);

  return (
    <button onClick={() => setTheme(theme === "light" ? "dark" : "light")}>
      切换主题
    </button>
  );
}
```

#### 3. 仅浏览器 API

需要访问浏览器特定的 API，如 `localStorage`、`window`、`navigator.geolocation` 等时：

```tsx
// app/ui/geolocation.tsx
"use client";

import { useState, useEffect } from "react";

export default function Geolocation() {
  const [location, setLocation] = useState<GeolocationCoordinates | null>(null);

  useEffect(() => {
    if ("geolocation" in navigator) {
      navigator.geolocation.getCurrentPosition((position) => {
        setLocation(position.coords);
      });
    }
  }, []);

  if (!location) return <p>获取位置中...</p>;

  return (
    <div>
      <p>纬度: {location.latitude}</p>
      <p>经度: {location.longitude}</p>
    </div>
  );
}
```

#### 4. 自定义 Hooks

使用自定义 Hooks 时，组件必须是 `Client Component`：

```tsx
// hooks/useWindowSize.ts
"use client";

import { useState, useEffect } from "react";

export function useWindowSize() {
  const [size, setSize] = useState({
    width: 0,
    height: 0,
  });

  useEffect(() => {
    const handleResize = () => {
      setSize({
        width: window.innerWidth,
        height: window.innerHeight,
      });
    };

    window.addEventListener("resize", handleResize);
    handleResize();

    return () => window.removeEventListener("resize", handleResize);
  }, []);

  return size;
}

// app/ui/responsive.tsx
("use client");

import { useWindowSize } from "@/hooks/useWindowSize";

export default function Responsive() {
  const { width, height } = useWindowSize();

  return (
    <div>
      <p>
        窗口大小: {width} x {height}
      </p>
    </div>
  );
}
```

### 使用 Server Components 的场景

当我们需要以下功能时，应使用 `Server Components`：

#### 1. 数据获取

从数据库或 API 获取数据，更接近数据源，减少客户端请求：

```tsx
// app/posts/page.tsx
import { db } from "@/lib/db";

export default async function PostsPage() {
  // 直接在服务器端获取数据
  const posts = await db.post.findMany({
    orderBy: { createdAt: "desc" },
    take: 10,
  });

  return (
    <div>
      <h1>文章列表</h1>
      {posts.map((post) => (
        <article key={post.id}>
          <h2>{post.title}</h2>
          <p>{post.content}</p>
        </article>
      ))}
    </div>
  );
}
```

#### 2. 访问服务器资源

使用 API 密钥、令牌和其他密钥，而不暴露给客户端：

```tsx
// app/api/data/route.ts
import { NextResponse } from "next/server";

export async function GET() {
  // 使用服务器端的环境变量，不会暴露给客户端
  const apiKey = process.env.SECRET_API_KEY;

  const response = await fetch("https://api.example.com/data", {
    headers: {
      Authorization: `Bearer ${apiKey}`,
    },
  });

  const data = await response.json();
  return NextResponse.json(data);
}

// app/data/page.tsx
export default async function DataPage() {
  const response = await fetch("http://localhost:3000/api/data", {
    cache: "no-store", // 或使用 revalidate
  });
  const data = await response.json();

  return (
    <div>
      <h1>数据页面</h1>
      <pre>{JSON.stringify(data, null, 2)}</pre>
    </div>
  );
}
```

#### 3. 减少客户端 JavaScript

保持大部分 UI 为 `Server Components`，只在需要交互的地方使用 `Client Components`：

```tsx
// app/layout.tsx
// 这是 Server Component，不需要 'use client'
import Header from "./header";
import Footer from "./footer";
import Search from "./search"; // 这是 Client Component

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <html>
      <body>
        <Header />
        <main>{children}</main>
        <Search />
        <Footer />
      </body>
    </html>
  );
}
```

#### 4. 改善性能指标

减少首次内容绘制（`FCP`）时间，支持流式传输：

```tsx
// app/products/page.tsx
import { Suspense } from "react";
import ProductList from "./product-list";
import ProductSkeleton from "./product-skeleton";

export default function ProductsPage() {
  return (
    <div>
      <h1>产品列表</h1>
      <Suspense fallback={<ProductSkeleton />}>
        <ProductList />
      </Suspense>
    </div>
  );
}

// app/products/product-list.tsx
import { db } from "@/lib/db";

export default async function ProductList() {
  // 这个数据获取会被流式传输
  const products = await db.product.findMany();

  return (
    <ul>
      {products.map((product) => (
        <li key={product.id}>{product.name}</li>
      ))}
    </ul>
  );
}
```

### 组合使用示例

在实际应用中，我们通常需要组合使用 `Server Components` 和 `Client Components`。例如，`<Page>` 组件是一个 `Server Component`，它获取文章数据，并将其作为 `props` 传递给处理客户端交互的 `<LikeButton>`：

```tsx
// app/[id]/page.tsx
import LikeButton from "@/app/ui/like-button";
import { getPost } from "@/lib/data";

export default async function Page({
  params,
}: {
  params: Promise<{ id: string }>;
}) {
  const { id } = await params;
  const post = await getPost(id);

  return (
    <div>
      <main>
        <h1>{post.title}</h1>
        <p>{post.content}</p>
        <p>发布时间: {new Date(post.createdAt).toLocaleDateString()}</p>
        {/* Server Component 传递数据给 Client Component */}
        <LikeButton likes={post.likes} postId={post.id} />
      </main>
    </div>
  );
}
```

```tsx
// app/ui/like-button.tsx
"use client";

import { useState, useTransition } from "react";

export default function LikeButton({
  likes,
  postId,
}: {
  likes: number;
  postId: string;
}) {
  const [likeCount, setLikeCount] = useState(likes);
  const [isPending, startTransition] = useTransition();

  const handleLike = () => {
    startTransition(async () => {
      // 调用 API 更新点赞数
      const response = await fetch(`/api/posts/${postId}/like`, {
        method: "POST",
      });
      const data = await response.json();
      setLikeCount(data.likes);
    });
  };

  return (
    <button onClick={handleLike} disabled={isPending}>
      {isPending ? "..." : "👍"} {likeCount}
    </button>
  );
}
```

## Server Components 和 Client Components 在 Next.js 中的工作原理

### 在服务器端

在服务器端，Next.js 使用 React 的 API 来编排渲染。渲染工作按单个路由段（`layouts` 和 `pages`）拆分为多个块：

- `Server Components` 被渲染成一种特殊的数据格式，称为 `RSC Payload`（React Server Component Payload）
- `Client Components` 和 `RSC Payload` 用于预渲染 HTML

渲染流程示意图：

![20251210141328](https://raw.githubusercontent.com/xun082/md/main/blogs.images20251210141328.png)

`RSC Payload`（React Server Component Payload）是渲染后的 `Server Components` 树的紧凑二进制表示。React 在客户端使用它来更新浏览器的 DOM。

`RSC Payload` 包含以下内容：

- `Server Components` 的渲染结果（序列化的 JSX）
- `Client Components` 应该渲染的位置占位符及其 JavaScript 文件的引用
- 从 `Server Component` 传递给 `Client Component` 的任何 `props`（必须是可序列化的）
- 组件树的引用信息，用于客户端协调

`RSC Payload` 的优势：

1. 体积小：二进制格式比 HTML 更紧凑
2. 增量更新：只传输变化的部分
3. 类型安全：保留了 React 的类型信息

### 在客户端（首次加载）

然后，在客户端：

1. `HTML` 用于立即向用户显示路由的快速非交互式预览（快速首屏渲染）
2. `RSC Payload` 用于协调 `Client Components` 和 `Server Components` 树，更新 DOM
3. `JavaScript` 用于水合 `Client Components` 并使应用具有交互性

客户端处理流程示意图：

![20251210141843](https://raw.githubusercontent.com/xun082/md/main/blogs.images20251210141843.png)

`Hydration`（水合）是 React 将事件处理器附加到 DOM 的过程，使静态 HTML 具有交互性。在 Next.js 中，只有 `Client Components` 需要 `hydration`，`Server Components` 不需要。

`Hydration` 过程包括以下步骤：

1. React 读取 `RSC Payload`
2. 识别哪些是 `Client Components`
3. 加载对应的 JavaScript 文件
4. 将事件处理器附加到 DOM 元素
5. 初始化组件状态（如果有）

### 后续导航

在用户完成首次加载后，当他们通过点击链接或其他方式进行页面跳转时，Next.js 使用客户端路由机制，这时的渲染流程与首次加载有很大不同。

在后续导航中，Next.js 采用了更高效的渲染策略。`RSC Payload` 会被提前预取并缓存，当用户点击链接时可以实现几乎即时的导航响应。此时 `Client Components` 完全在客户端渲染，不再需要服务器返回完整的 HTML 文档，只传输必要的组件树数据。这意味着只有发生变化的部分会被传输和更新，大大减少了网络传输的数据量。

这种优化机制为我们带来了显著的性能提升：导航速度更快，因为不需要重新加载整个页面；网络请求更少，只传输必要的数据；用户体验更流畅，页面切换几乎感觉不到延迟。

## 使用示例

### 创建 Client Components

通过在文件顶部添加 [`"use client"`](https://react.dev/reference/react/use-client) 指令（位于导入语句之上）来创建 Client Component：

```tsx
// app/ui/counter.tsx
"use client";

import { useState } from "react";

export default function Counter() {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>{count} 次点击</p>
      <button onClick={() => setCount(count + 1)}>点击我</button>
    </div>
  );
}
```

重要说明：

1. `"use client"` 必须放在文件的最顶部，在所有导入语句之前
2. `"use client"` 用于声明 `Server` 和 `Client` 模块图（树）之间的边界
3. 一旦文件被标记为 `"use client"`，它的所有导入和子组件都被视为客户端包的一部分。这意味着我们不需要为每个用于客户端的组件添加该指令

边界示例：

```tsx
// app/ui/button.tsx
"use client"; // 边界在这里

import { useState } from "react";
import Icon from "./icon"; // Icon 也会成为 Client Component

export default function Button() {
  const [clicked, setClicked] = useState(false);
  return (
    <button onClick={() => setClicked(!clicked)}>
      <Icon /> {/* Icon 不需要 'use client'，因为它在这个边界内 */}
      {clicked ? "已点击" : "点击我"}
    </button>
  );
}
```

### 减少 JavaScript 包大小

为了减少客户端 JavaScript 包的大小，将 `'use client'` 添加到特定的交互组件，而不是将大部分 UI 标记为 Client Components。

不好的做法（将整个布局标记为 `Client Component`）：

```tsx
// ❌ 不好的做法
// app/layout.tsx
"use client"; // 这会让整个布局成为 Client Component

import Header from "./header";
import Footer from "./footer";
import Search from "./search";

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <>
      <Header />
      <main>{children}</main>
      <Search />
      <Footer />
    </>
  );
}
```

好的做法（只将需要交互的部分标记为 `Client Component`）：

```tsx
// ✅ 好的做法
// app/layout.tsx
// Layout 是 Server Component，不需要 'use client'
import Search from "./search"; // 只有 Search 是 Client Component
import Logo from "./logo"; // Logo 是 Server Component
import Navigation from "./navigation"; // Navigation 是 Server Component

export default function Layout({ children }: { children: React.ReactNode }) {
  return (
    <>
      <nav>
        <Logo />
        <Navigation />
        <Search /> {/* 只有这个需要交互 */}
      </nav>
      <main>{children}</main>
    </>
  );
}
```

```tsx
// app/ui/search.tsx
"use client";

import { useState } from "react";

export default function Search() {
  const [query, setQuery] = useState("");

  return (
    <input
      type="search"
      value={query}
      onChange={(e) => setQuery(e.target.value)}
      placeholder="搜索..."
    />
  );
}
```

### 从 Server Components 传递数据到 Client Components

我们可以使用 `props` 将数据从 `Server Components` 传递到 `Client Components`。传递的数据必须是可序列化的（JSON 兼容）。

可序列化的数据类型：

- 字符串、数字、布尔值
- 数组、对象
- Date（会被序列化为字符串）
- null、undefined

不可序列化的类型：

- 函数
- 类实例
- Symbol
- 未定义的属性

示例：

```tsx
// app/[id]/page.tsx
import LikeButton from "@/app/ui/like-button";
import { getPost } from "@/lib/data";

export default async function Page({
  params,
}: {
  params: Promise<{ id: string }>;
}) {
  const { id } = await params;
  const post = await getPost(id);

  // ✅ 可以传递可序列化的数据
  return (
    <LikeButton
      likes={post.likes}
      postId={post.id}
      metadata={{
        title: post.title,
        author: post.author,
        createdAt: post.createdAt.toISOString(), // Date 需要转换为字符串
      }}
    />
  );
}
```

```tsx
// app/ui/like-button.tsx
"use client";

import { useState } from "react";

interface LikeButtonProps {
  likes: number;
  postId: string;
  metadata: {
    title: string;
    author: string;
    createdAt: string;
  };
}

export default function LikeButton({
  likes,
  postId,
  metadata,
}: LikeButtonProps) {
  const [likeCount, setLikeCount] = useState(likes);

  return (
    <div>
      <button onClick={() => setLikeCount(likeCount + 1)}>
        👍 {likeCount}
      </button>
      <p>文章: {metadata.title}</p>
      <p>作者: {metadata.author}</p>
    </div>
  );
}
```

### 不能将 Server Components 作为 props 传递给 Client Components

不能将 `Server Components` 作为 `props` 传递给 `Client Components`。这是因为 `Server Components` 需要在服务器端渲染，而 `props` 传递发生在客户端。

错误的做法：

```tsx
// app/ui/cart.tsx
"use client";

export default function Cart({ children }: { children: React.ReactNode }) {
  return <div>{children}</div>;
}
```

```tsx
// app/page.tsx
import Cart from "./ui/cart";
import Modal from "./ui/modal"; // Modal 是 Server Component

export default function Page() {
  return (
    <Cart>
      <Modal /> {/* ❌ 错误：不能将 Server Component 作为 props 传递 */}
    </Cart>
  );
}
```

正确的做法 1：使用 children 模式

```tsx
// app/page.tsx
import Modal from "./ui/modal"; // Modal 是 Server Component
import Cart from "./ui/cart"; // Cart 是 Client Component

export default function Page() {
  return (
    <Modal>
      <Cart /> {/* ✅ 正确：Cart 作为 children 传递 */}
    </Modal>
  );
}
```

正确的做法 2：使用组合模式

```tsx
// app/page.tsx
import Modal from "./ui/modal";
import CartContent from "./cart-content"; // Server Component

export default function Page() {
  return (
    <div>
      <Modal>
        <CartContent />
      </Modal>
    </div>
  );
}
```

在这种模式中，所有 Server Components 都会在服务器端提前渲染，包括作为 props 传递的组件。生成的 RSC Payload 将包含 Client Components 在组件树中应该渲染的位置的引用。

### Context Providers

`React Context` 通常用于共享全局状态，如当前主题、用户信息等。但是，React Context 在 Server Components 中不受支持，因为 Context 需要在客户端维护状态。

要使用 `Context`，创建一个接受 `children` 的 `Client Component`：

```tsx
// app/theme-provider.tsx
"use client";

import { createContext, useContext, useState, useEffect } from "react";

type Theme = "light" | "dark";

interface ThemeContextType {
  theme: Theme;
  toggleTheme: () => void;
}

const ThemeContext = createContext<ThemeContextType | undefined>(undefined);

export function useTheme() {
  const context = useContext(ThemeContext);
  if (context === undefined) {
    throw new Error("useTheme must be used within a ThemeProvider");
  }
  return context;
}

export default function ThemeProvider({
  children,
}: {
  children: React.ReactNode;
}) {
  const [theme, setTheme] = useState<Theme>("light");

  useEffect(() => {
    // 从 localStorage 读取主题
    const savedTheme = localStorage.getItem("theme") as Theme;
    if (savedTheme) {
      setTheme(savedTheme);
    }
  }, []);

  useEffect(() => {
    // 应用主题到 document
    document.documentElement.setAttribute("data-theme", theme);
    localStorage.setItem("theme", theme);
  }, [theme]);

  const toggleTheme = () => {
    setTheme((prev) => (prev === "light" ? "dark" : "light"));
  };

  return (
    <ThemeContext.Provider value={{ theme, toggleTheme }}>
      {children}
    </ThemeContext.Provider>
  );
}
```

然后，将其导入到 Server Component（例如 `layout`）：

```tsx
// app/layout.tsx
import ThemeProvider from "./theme-provider";

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html>
      <body>
        <ThemeProvider>{children}</ThemeProvider>
      </body>
    </html>
  );
}
```

现在，任何 `Client Component` 都可以使用这个 `Context`：

```tsx
// app/ui/theme-toggle.tsx
"use client";

import { useTheme } from "@/app/theme-provider";

export default function ThemeToggle() {
  const { theme, toggleTheme } = useTheme();

  return (
    <button onClick={toggleTheme}>
      当前主题: {theme === "light" ? "🌞" : "🌙"}
    </button>
  );
}
```

> 我们应该尽可能在组件树的深层位置渲染 `Provider`，而不是在根节点。例如 `ThemeProvider` 只包装 `{children}` 而不是整个 `<html>` 文档，这样可以让 Next.js 更容易优化 `Server Components` 的静态部分。

### 第三方组件

当使用依赖仅客户端功能的第三方组件时，可以将其包装在 `Client Component` 中以确保其正常工作。

场景 1：第三方组件没有 `"use client"` 指令

例如，`<Carousel />` 可以从 `acme-carousel` 包中导入。该组件使用 `useState`，但还没有 `"use client"` 指令。

如果我们在 `Client Component` 中使用 `<Carousel />`，它将正常工作：

```tsx
// app/gallery.tsx
"use client";

import { useState } from "react";
import { Carousel } from "acme-carousel";

export default function Gallery() {
  const [isOpen, setIsOpen] = useState(false);

  return (
    <div>
      <button onClick={() => setIsOpen(true)}>查看图片</button>
      {/* 可以工作，因为 Carousel 在 Client Component 中使用 */}
      {isOpen && <Carousel />}
    </div>
  );
}
```

但是，如果尝试在 `Server Component` 中直接使用它，会看到错误：

```tsx
// app/page.tsx
import { Carousel } from "acme-carousel"; // ❌ 错误！

export default function Page() {
  return (
    <div>
      <Carousel /> {/* 这会报错 */}
    </div>
  );
}
```

错误信息示例：

![20251210142425](https://raw.githubusercontent.com/xun082/md/main/blogs.images20251210142425.png)

错误提示：`createContext only works in Client Components. Add the "use client" directive at the top of the file to use it.`

这是因为 `acme-carousel` 内部使用了 `createContext`，而 `createContext` 只能在 `Client Components` 中使用。

解决方案：包装第三方组件

要修复此问题，可以将依赖仅客户端功能的第三方组件包装在我们自己的 `Client Component` 中：

```tsx
// app/carousel.tsx
"use client";

import { Carousel } from "acme-carousel";

export default Carousel;
```

现在，可以在 `Server Component` 中直接使用 `<Carousel />`：

```tsx
// app/page.tsx
import Carousel from "@/Components/carousel";

export default function Page() {
  const items = [
    { id: 1, content: "Item 1" },
    { id: 2, content: "Item 2" },
    { id: 3, content: "Item 3" },
  ];
  return (
    <div>
      <Carousel items={items} />
    </div>
  );
}
```

![20251210142850](https://raw.githubusercontent.com/xun082/md/main/blogs.images20251210142850.png)

场景 2：第三方组件已经有 `"use client"` 指令

如果第三方组件库已经正确添加了 `"use client"` 指令，可以直接使用：

```tsx
// app/page.tsx
import { Button } from "@radix-ui/react-button"; // 已经有 'use client'

export default function Page() {
  return (
    <div>
      <Button>点击我</Button>
    </div>
  );
}
```

如果我们正在构建组件库，应该将 `"use client"` 指令添加到依赖仅客户端功能的入口点。这样可以让用户将组件直接导入到 `Server Components` 中，而无需创建包装器。

示例：

```tsx
// packages/ui/button.tsx
"use client";

export function Button({ children, ...props }) {
  return <button {...props}>{children}</button>;
}
```

### 防止环境污染

JavaScript 模块可以在 `Server Components` 和 `Client Components` 模块之间共享。这意味着可能意外地将仅服务器代码导入到客户端，导致安全问题或运行时错误。

问题示例：

```ts
// lib/data.ts
export async function getData() {
  const res = await fetch("https://external-service.com/data", {
    headers: {
      authorization: process.env.API_KEY, // 敏感信息！
    },
  });

  return res.json();
}
```

此函数包含一个永远不应暴露给客户端的 `API_KEY`。

在 Next.js 中，只有前缀为 `NEXT_PUBLIC_` 的环境变量才会包含在客户端包中。如果变量没有前缀，Next.js 会用空字符串替换它们。

因此，即使 `getData()` 可以在客户端导入和执行，它也不会按预期工作（`API_KEY` 会是空字符串）。

解决方案：使用 `server-only` 包

为了防止在 Client Components 中意外使用，可以使用 [`server-only` 包](https://www.npmjs.com/package/server-only)：

```bash
pnpm install server-only
```

然后，将包导入到包含仅服务器代码的文件中：

```js
// lib/data.js
import "server-only";

export async function getData() {
  const res = await fetch("https://external-service.com/data", {
    headers: {
      authorization: process.env.API_KEY,
    },
  });

  return res.json();
}
```

现在，如果尝试将模块导入到 `Client Component` 中，会在构建时出现错误：

```tsx
// app/ui/data-display.tsx
"use client";

import { getData } from "@/lib/data"; // ❌ 构建错误！

export default function DataDisplay() {
  // ...
}
```

错误信息：`This module cannot be imported from a Client Component module.`

使用 `client-only` 包

相应的 [`client-only` 包](https://www.npmjs.com/package/client-only) 可用于标记包含仅客户端逻辑的模块，例如访问 `window` 对象的代码：

```bash
npm install client-only
```

```ts
// lib/browser-utils.ts
import "client-only";

export function getWindowSize() {
  return {
    width: window.innerWidth,
    height: window.innerHeight,
  };
}
```

在 Next.js 中，安装 `server-only` 或 `client-only` 是可选的。但是，如果 linting 规则标记了多余的依赖项，可以安装它们以避免问题。

Next.js 在内部处理 `server-only` 和 `client-only` 导入，以在模块在错误环境中使用时提供更清晰的错误消息。Next.js 不使用来自 NPM 的这些包的内容。

Next.js 还为自己的 `server-only` 和 `client-only` 提供类型声明，适用于 [`noUncheckedSideEffectImports`](https://www.typescriptlang.org/tsconfig/#noUncheckedSideEffectImports) 处于活动状态的 TypeScript 配置。

## 总结

`Server Components` 和 `Client Components` 是 Next.js `App Router` 的核心概念。默认情况下，组件是 `Server Components`，可以在服务器端获取数据、访问服务器资源，而无需额外配置。当需要交互性、状态管理或浏览器 API 时，使用 `"use client"` 指令创建 `Client Components`。

通过合理组合 `Server Components` 和 `Client Components`，我们可以构建高性能、交互性强的全栈应用，同时减少客户端 JavaScript 包的大小，提升首屏加载性能。

在实际开发中，我们应该默认使用 `Server Components`，除非明确需要客户端功能。这样可以减少客户端 JavaScript 包大小、提升首屏加载性能，并更好地利用服务器资源。我们应该最小化 `Client Components` 边界，只在真正需要交互的地方使用 `Client Components`，避免将整个布局或大型组件标记为客户端组件。同时，合理组织组件结构，将 `Server Components` 和 `Client Components` 分开组织，便于维护和理解。利用 `Suspense` 进行流式渲染，可以实现更好的用户体验，让用户可以更快看到页面内容。如果需要在 `Server Component` 中使用浏览器 API，应该将其提取到 `Client Component` 中，而不是使用条件检查。

在开发过程中，我们可能会遇到一些常见问题。当在 `Server Component` 中使用了需要客户端的 Hook（如 `useState`）时，需要在文件顶部添加 `"use client"` 指令。不能将函数从 `Server Component` 传递给 `Client Component`，应该将函数逻辑移到 `Client Component` 内部，或者使用 Server Actions。在 `Server Component` 中使用了 `React Context` 会报错，应该将 `Context Provider` 包装在 `Client Component` 中。在 `Client Component` 中访问 `process.env.API_KEY` 会得到 `undefined`，只有 `NEXT_PUBLIC_` 前缀的环境变量会暴露给客户端，对于客户端需要的变量应该使用 `NEXT_PUBLIC_` 前缀，对于服务器端变量应该在 `Server Component` 或 `API Route` 中使用。

在性能优化方面，`Server Components` 自动进行代码分割，每个路由段都是独立的代码块，可以按需加载，减少初始包大小。对于大型 `Client Components`，可以使用 `next/dynamic` 进行动态导入，实现按需加载，提升首屏性能。在 `Server Components` 中，可以使用 Next.js 的缓存策略（如 `unstable_cache`）来缓存数据获取结果，减少重复请求，提升性能。
