## 流式渲染（Streaming HTML）

流式渲染是一种将 HTML 内容按块分段传输 给客户端的方式，通常基于 HTTP/1.1 的 Chunked Transfer Encoding 实现。浏览器可在接收到部分 HTML 后立即开始解析和渲染，从而 缩短首屏加载时间、提升用户体验。

在 React 18 之前，我们常使用 `renderToString` 来进行服务端渲染。它是一个同步阻塞函数，将组件树整体转换成 HTML 字符串后一次性返回。但它存在以下问题：

- 阻塞渲染：必须等待整个组件树渲染完成后才能返回，响应延迟较高；

- 占用事件循环：同步操作会阻塞线程，影响 Node.js 对其他请求的处理；

- 延迟资源加载：浏览器需等待完整 HTML 响应后，才能发起 CSS/JS 等资源加载；

- 整体 hydration：客户端必须等所有 JS bundle 加载完，才能整体执行 hydration，导致交互延迟。

React 18 推出的 `renderToPipeableStream` 实现了真正的 Streaming SSR，主要带来两大核心能力：

1.  Streaming HTML

    - 服务端可以边渲染边将 HTML 发送到客户端；

    - 浏览器可立刻解析 HTML、加载资源，提升 FP/FCP 首屏渲染速度；

    - 更适合大页面、慢组件、数据密集型内容。

2.  Selective Hydration

    - 客户端不必等待全部 JS bundle 加载完毕；

    - 已完成渲染的区域可以先行 hydration（事件绑定等），提升页面可交互性；

    - 搭配 `<Suspense>` 可优雅控制各区域的加载优先级。

### 基本使用

### 选择性注水

在 React 18 之前，SSR 不支持代码分割，异步组件只能在客户端加载，或提前把 JS 写进 HTML，等全部加载完再执行 hydration。这种方式响应慢，交互延迟明显。

React 18 引入了流式渲染和选择性水合，允许服务端边渲染边发送 HTML，浏览器收到后可立即开始展示。而注水过程也变得“懒惰”：只优先激活用户正在交互的区域，其它部分延后处理。

React 还内置了事件重放机制：如果用户在未水合的区域点击了按钮，事件会被记录，在该区域完成水合后自动重放，确保交互不丢失、体验更顺滑。

通过支持 `lazy` 和 `Suspense`，React 现在可以像客户端一样组织异步组件，同时在服务端实现更快、更流畅的 SSR。

### 流式渲染会对 SEO 有影响吗

在服务端渲染（SSR）场景中，输出正确的 TDK（即 Title、Description、Keywords）是 SEO 的关键。React 通常配合 [`react-helmet`](https://github.com/nfl/react-helmet) 来设置这些标签，它允许你在组件中动态定义 `<title>` 和 `<meta>`，并在服务端渲染时收集这些信息。

在传统 SSR（如 `renderToString`）中，TDK 渲染是同步完成的。React 渲染完组件后，通过 `Helmet.renderStatic()` 拿到标签内容，再插入 HTML 模板中即可。

而在 React 18 中引入的 Streaming SSR 模式下，虽然渲染过程是流式分块的，但只要在主内容流（shell）启动前完成 TDK 的注入逻辑，SEO 效果不会受到影响。React 仍然可以在服务端完整输出页面结构和 `<head>` 元信息。只需要确保在 `onShellReady` 触发前拿到 `Helmet.renderStatic()` 的内容，将其插入模板即可。

如下代码所示：

```jsx
import { Helmet } from "react-helmet";

export default function App({ initialData }) {
  const { title, description, keywords } = initialData?.tdk || {};

  return (
    <>
      <Helmet>
        <title>{title}</title>
        <meta name="description" content={description} />
        <meta name="keywords" content={keywords} />
      </Helmet>
      <main>...</main>
    </>
  );
}
```

```js
// server.js 中，stream 渲染前插入 TDK 信息
const { pipe } = renderToPipeableStream(<App initialData={data} />, {
  onShellReady() {
    const helmet = Helmet.renderStatic();
    const headHtml = `
      ${helmet.title.toString()}
      ${helmet.meta.toString()}
    `;

    res.write(templateHead.replace("<!-- TDK -->", headHtml));
    pipe(res);
  },
});
```

只要我们确保在服务端输出的 HTML 中包含 `<title>` 和 `<meta>` 标签，流式渲染不会对 SEO 产生任何影响，反而因为加载速度提升，对爬虫抓取更友好。

TDK 推荐与服务端数据预取绑定，保持内容与标题的一致性。对于需要动态内容的页面，也可以将 TDK 信息作为接口响应的一部分提前获取。
