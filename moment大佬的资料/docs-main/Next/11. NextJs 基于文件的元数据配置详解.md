在上一篇中，我们介绍了 Next.js 添加元数据的两种方式：

1. 基于配置的元数据：在 `layout.tsx` 或 `page.tsx` 中导出一个静态 `metadata` 对象或动态 `generateMetadata` 函数
2. 基于文件的元数据：添加静态或动态生成的特殊文件

基于文件的元数据可以用来添加网站的应用图标、OG 图片、`robots.txt`、`sitemap.xml`、`manifest.json` 等。本篇我们将详细介绍这些文件的用法和配置方式。

## 1. favicon、icon 和 apple-icon

Next.js 约定 `favicon`、`icon`、`apple-icon` 文件都可以用来设置应用的图标。这些图标会显示在浏览器标签页、收藏夹（书签）、手机主屏幕、搜索引擎结果等多个场景中。

有两种方式可以设置图标：

1. 使用静态文件
2. 使用代码生成

### 1.1. 使用静态文件

在 `/app` 目录下放置 `favicon.ico`、`icon` 或 `apple-icon` 图片文件即可设置图标。其中 `favicon` 只能位于 `app/` 根目录，`icon` 可以放在更深层的目录以实现精细控制，`apple-icon` 专门用于苹果设备。

#### 1.1.1. 多图标配置

可以通过在文件名中添加数字后缀设置多个图标，如 `icon1.png`、`icon2.png`。不同设备和场景可能需要不同尺寸的图标，生成的 `<link>` 标签属性会根据图标类型和文件内容自动生成。

推荐使用 [RealFaviconGenerator](https://realfavicongenerator.net/) 或 [Favicon.cc](https://www.favicon.cc/) 帮助生成图标。

### 1.2. 使用代码生成

除了直接使用图片文件，我们也可以用代码生成图标。需要新建一个名为 `icon` 或 `apple-icon` 的文件，不过这次的后缀是 `.js`、`.ts`、`.tsx`。

最简单的生成一个图标的方式是通过 `next/og` 的 `ImageResponse` API，使用示例如下：

```tsx
// app/icon.tsx
import { ImageResponse } from "next/og";

// 路由段配置
export const runtime = "edge";

// 图片 metadata
export const size = {
  width: 32,
  height: 32,
};
export const contentType = "image/png";

// 图片生成
export default function Icon(): ImageResponse {
  return new ImageResponse(
    (
      <div
        style={{
          width: "100%",
          height: "100%",
          display: "flex",
          justifyContent: "center",
          alignItems: "center",
          backgroundColor: "black",
        }}
      >
        <div
          style={{
            fontSize: "16px",
            fontWeight: "bold",
            color: "white",
          }}
        >
          M
        </div>
      </div>
    ),
    {
      ...size,
    }
  );
}
```

使用代码生成图标时需要注意以下几点：

1. 从 Next.js v14.0.0 起，`ImageResponse` 从 `next/server` 转移到 `next/og`，如果出现导入错误，请升级版本或使用 `next/server`
2. 默认情况下，生成的图标是静态优化的，会在构建时生成并缓存，除非使用了动态函数（如 `cookies()`、`headers()`）或未缓存数据
3. 不能使用代码生成 `favicon` 图标，需要使用 `icon` 或 `favicon.ico` 静态文件代替
4. 可以使用 `generateImageMetadata` API 在一个文件中生成多个图标

默认导出函数可接收 `params` 参数，用于获取动态路由参数（与 `generateMetadata` 中的 `params` 用法相同）。例如：

```tsx
// app/shop/[slug]/icon.tsx
import { ImageResponse } from "next/og";

type Props = {
  params: Promise<{ slug: string }>;
};

export default async function Icon({ params }: Props): Promise<ImageResponse> {
  const { slug } = await params;

  console.log("接收到的参数", slug);

  return new ImageResponse(
    (
      <div
        style={{
          width: "100%",
          height: "100%",
          display: "flex",
          justifyContent: "center",
          alignItems: "center",
          backgroundColor: "black",
          color: "white",
          fontSize: "18px",
          fontWeight: "bold",
        }}
      >
        {slug}
      </div>
    ),
    {
      width: 32, // 图标宽度
      height: 32, // 图标高度
    }
  );
}
```

![20251210120108](https://raw.githubusercontent.com/xun082/md/main/blogs.images20251210120108.png)

函数返回值可以是 `Blob`、`ArrayBuffer`、`TypedArray`、`DataView`、`ReadableStream` 或 `Response` 类型。

除了导出图片文件本身，还可以选择性导出 `size`、`contentType` 变量来设置图片元数据。例如 `export const size = { width: 32, height: 32 }` 会生成 `<link rel="icon" sizes="32x32" />`，`export const contentType = "image/png"` 会生成 `<link rel="icon" type="image/png" />`。

`icon` 和 `apple-icon` 本质上是特殊的路由处理程序，因此支持路由段配置，如 `dynamic`、`revalidate`、`runtime`、`preferredRegion` 等选项。例如可以设置 `export const runtime = "edge"` 来指定运行时环境。

### 1.3. ImageResponse

`ImageResponse` 可以使用 JSX 和 CSS 生成动态图片，特别适合生成依赖动态内容的社交媒体图片（Open Graph 图像、Twitter 卡片等）。

类型定义：

```typescript
import { ImageResponse } from 'next/og'

new ImageResponse(
  element: ReactElement,
  options: {
    width?: number = 1200
    height?: number = 630
    emoji?: 'twemoji' | 'blobmoji' | 'noto' | 'openmoji' = 'twemoji',
    fonts?: {
      name: string,
      data: ArrayBuffer,
      weight: number,
      style: 'normal' | 'italic'
    }[]
    debug?: boolean = false

    // Options that will be passed to the HTTP response
    status?: number = 200
    statusText?: string
    headers?: Record<string, string>
  },
)
```

底层使用 Vercel 的 [satori](https://github.com/vercel/satori) 库，可将 JSX、HTML 和 CSS 转为 SVG。具体用法和支持的 HTML、CSS 特性请参考 [satori 文档](https://github.com/vercel/satori#html-elements)。

还有一个 `generateImageMetadata` API，搭配默认导出函数，可以生成多张图片，我们将在第 3 节详细介绍。

## 2. opengraph-image 和 twitter-image

用于设置页面在社交媒体平台分享时显示的图片。社交网站（如 Facebook、LinkedIn、微信等）会根据 Open Graph 标签呈现内容，`twitter-image` 专门用于 Twitter（X）平台。

使用方式：静态图片文件或代码生成。

### 2.1. 静态文件

在路由文件夹下添加 `opengraph-image.(jpg|jpeg|png|gif)` 或 `twitter-image.(jpg|jpeg|png|gif)` 图片文件，系统会自动生成相应的 meta 标签。可以添加对应的 `.alt.txt` 文件来设置图片的 alt 文字，例如 `opengraph-image.alt.txt` 和 `twitter-image.alt.txt`。

### 2.2. 代码生成

创建 `opengraph-image.(js|ts|tsx)` 或 `twitter-image.(js|ts|tsx)` 文件。代码生成方式与图标类似，但图片元数据配置多了 `alt` 字段：

```tsx
// app/about/opengraph-image.tsx
import { ImageResponse } from "next/og";

export const runtime = "edge";

export const alt = "About Acme";
export const size = {
  width: 1200,
  height: 630,
};

export const contentType = "image/png";

export default async function Image(): Promise<ImageResponse> {
  // 加载字体
  const interSemiBold = await fetch(
    new URL("./Inter-SemiBold.ttf", import.meta.url)
  ).then((res) => res.arrayBuffer());

  return new ImageResponse(
    (
      <div
        style={{
          fontSize: 128,
          background: "white",
          width: "100%",
          height: "100%",
          display: "flex",
          alignItems: "center",
          justifyContent: "center",
        }}
      >
        About Acme
      </div>
    ),
    {
      ...size,
      fonts: [
        {
          name: "Inter",
          data: interSemiBold,
          style: "normal",
          weight: 400,
        },
      ],
    }
  );
}
```

与图标类似，也可以导出 `alt`、`size`、`contentType` 变量来设置元数据，还可以根据页面内容动态生成图片：

```tsx
// app/posts/[slug]/opengraph-image.tsx
import { ImageResponse } from "next/og";

type Props = {
  params: Promise<{ slug: string }>;
};

export const runtime = "edge";

export const size = {
  width: 1200,
  height: 630,
};
export const contentType = "image/png";

export default async function Image({ params }: Props): Promise<ImageResponse> {
  const { slug } = await params;
  const post = await fetch(`https://www.codecrack.cn/posts/${slug}`).then(
    (res) => res.json()
  );

  return new ImageResponse(
    (
      <div
        style={{
          fontSize: 48,
          background: "white",
          width: "100%",
          height: "100%",
          display: "flex",
          alignItems: "center",
          justifyContent: "center",
        }}
      >
        {post.title}
      </div>
    ),
    {
      ...size,
    }
  );
}
```

## 3. generateImageMetadata 生成多张图片

使用 `generateImageMetadata` 可以生成一张图片的不同版本或返回多张图片。该函数返回包含图片元数据的对象数组，每个对象必须包含 `id` 字段（必传），可选的还有 `alt`、`size`、`contentType`。

示例：

```tsx
// icon.tsx
import { ImageResponse } from "next/og";
import type { ImageMetadata } from "next";

export function generateImageMetadata(): ImageMetadata[] {
  return [
    {
      contentType: "image/png",
      size: { width: 48, height: 48 },
      id: "small",
    },
    {
      contentType: "image/png",
      size: { width: 72, height: 72 },
      id: "medium",
    },
  ];
}

type IconProps = {
  id: string;
};

export default function Icon({ id }: IconProps): ImageResponse {
  return new ImageResponse(
    (
      <div
        style={{
          width: "100%",
          height: "100%",
          display: "flex",
          alignItems: "center",
          justifyContent: "center",
          fontSize: 88,
          background: "#000",
          color: "#fafafa",
        }}
      >
        Icon {id}
      </div>
    )
  );
}
```

使用 `generateImageMetadata` 时，默认导出函数会接收额外的 `id` 参数，Next.js 遍历数组将 `id` 传入，生成多张图片。

实际应用示例：

```tsx
// app/products/[id]/opengraph-image.tsx
import { ImageResponse } from "next/og";
import type { ImageMetadata } from "next";
import { getCaptionForImage, getOGImages } from "@/app/utils/images";

type Props = {
  params: Promise<{ id: string }>;
};

export async function generateImageMetadata({
  params,
}: Props): Promise<ImageMetadata[]> {
  const { id } = await params;
  const images = await getOGImages(id);

  return images.map((image, idx) => ({
    id: String(idx),
    size: { width: 1200, height: 600 },
    alt: image.text,
    contentType: "image/png",
  }));
}

type ImageProps = {
  params: Promise<{ id: string }>;
  id: string;
};

export default async function Image({
  params,
  id,
}: ImageProps): Promise<ImageResponse> {
  const { id: productId } = await params;
  const imageId = id;
  // 获取图片说明
  const text = await getCaptionForImage(productId, imageId);

  return new ImageResponse(
    (
      <div
        style={{
          fontSize: 48,
          background: "white",
          width: "100%",
          height: "100%",
          display: "flex",
          alignItems: "center",
          justifyContent: "center",
        }}
      >
        {text}
      </div>
    )
  );
}
```

## 4. robots.txt

用于告诉搜索引擎可以爬取哪些 URL。可以直接在 `app/` 目录下添加 `robots.txt` 静态文件，或者创建 `robots.ts` 文件动态生成。

使用代码生成时，导出的 `Robots` 对象支持 `rules`（userAgent、allow、disallow、crawlDelay）、`sitemap`、`host` 等配置，可以是单个规则对象或规则数组。

代码生成示例：

```tsx
// app/robots.ts
import type { MetadataRoute } from "next";

export default function robots(): MetadataRoute.Robots {
  return {
    rules: {
      userAgent: "*",
      allow: "/",
      disallow: "/private/",
    },
    sitemap: "https://www.codecrack.cn/sitemap.xml",
  };
}
```

## 5. sitemap.xml

站点地图，帮助搜索引擎更高效地爬取网站。可以直接在 `app/` 目录下创建 `sitemap.xml` 静态文件，或者创建 `sitemap.ts` 文件动态生成。

使用代码生成时，返回包含 URL 对象的数组，每个对象包含 `url`（必填）、`lastModified`、`changeFrequency`、`priority`、`alternates` 等字段。

代码生成示例：

```tsx
// app/sitemap.ts
import type { MetadataRoute } from "next";

export default function sitemap(): MetadataRoute.Sitemap {
  return [
    {
      url: "https://www.codecrack.cn",
      lastModified: new Date(),
      changeFrequency: "yearly",
      priority: 1,
    },
    {
      url: "https://www.codecrack.cn/about",
      lastModified: new Date(),
      changeFrequency: "monthly",
      priority: 0.8,
    },
    {
      url: "https://www.codecrack.cn/blog",
      lastModified: new Date(),
      changeFrequency: "weekly",
      priority: 0.5,
    },
  ];
}
```

## 6. manifest.json

PWA 应用清单文件，设置网站的相关信息。可以直接在 `app/` 目录下创建 `manifest.json` 或 `manifest.webmanifest` 静态文件，或者创建 `manifest.ts` 文件动态生成。

使用代码生成时，返回包含 `name`、`short_name`、`description`、`start_url`、`display`、`background_color`、`theme_color`、`icons` 等字段的对象，具体字段请参考 [MDN 文档](https://developer.mozilla.org/zh-CN/docs/Web/Manifest)。

代码生成示例：

```tsx
// app/manifest.ts
import type { MetadataRoute } from "next";

export default function manifest(): MetadataRoute.Manifest {
  return {
    name: "Next.js App",
    short_name: "Next.js App",
    description: "Next.js App",
    start_url: "/",
    display: "standalone",
    background_color: "#fff",
    theme_color: "#fff",
    icons: [
      {
        src: "/favicon.ico",
        sizes: "any",
        type: "image/x-icon",
      },
    ],
  };
}
```

## 总结

Next.js 基于文件的元数据配置提供了直观灵活的方式来管理网站图标、社交媒体图片、SEO 相关文件等。通过静态文件或代码生成，可以为不同路由设置不同的元数据，实现精细化的 SEO 控制。`ImageResponse` API 和 `generateImageMetadata` 函数支持根据动态内容生成个性化图片，提升网站展示效果和用户体验。
