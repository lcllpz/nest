Next.js 提供的 `cookies` 是一个异步函数，用于在服务端读取和写入 HTTP Cookie。在 [Server Components](/docs/app/getting-started/server-and-client-components) 中可读取 Cookie。

在 [Server Functions](/docs/app/getting-started/updating-data) 或 [Route Handlers](/docs/app/api-reference/file-conventions/route) 中可读写 Cookie。

Cookie 常用于主题偏好、用户会话、语言设置等场景。本文基于 [Next.js 官方文档](https://nextjs.org/docs/app/api-reference/functions/cookies) 详解 `cookies` 的用法与注意事项。

## cookies 基本用法

从 `next/headers` 导入 `cookies`，在异步函数中调用并获取 cookie 存储对象：

```tsx
// app/page.tsx
import { cookies } from "next/headers";

export default async function Page() {
  const cookieStore = await cookies();
  const theme = cookieStore.get("theme");
  return <div>主题: {theme?.value ?? "未设置"}</div>;
}
```

`cookies()` 返回一个 Promise，必须使用 `async/await` 或 React 的 [`use`](https://react.dev/reference/react/use) 获取。

## 读取 Cookie

在 Server Component 中调用 `await cookies()` 获取 cookie 存储对象后，可通过以下方法读取请求中的 Cookie：

### 获取单个 Cookie

使用 `get('name')` 按名称获取单个 cookie，返回 `{ name, value }` 或 `undefined`：

```tsx
// app/dashboard/page.tsx
import { cookies } from "next/headers";

export default async function DashboardPage() {
  const cookieStore = await cookies();
  const theme = cookieStore.get("theme");

  return (
    <div>
      <p>当前主题: {theme?.value ?? "light"}</p>
    </div>
  );
}
```

### 获取所有 Cookie

使用 `getAll()` 获取所有 cookie，返回 `{ name, value }[]` 数组。若需字符串形式，可调用 `toString()`：

```tsx
// app/admin/page.tsx
import { cookies } from "next/headers";

export default async function AdminPage() {
  const cookieStore = await cookies();
  const allCookies = cookieStore.getAll();

  return (
    <ul>
      {allCookies.map((cookie) => (
        <li key={cookie.name}>
          <strong>{cookie.name}:</strong> {cookie.value}
        </li>
      ))}
    </ul>
  );
}
```

### 检查 Cookie 是否存在

使用 `has('name')` 判断指定名称的 cookie 是否存在，返回 `boolean`：

```tsx
// app/page.tsx
import { cookies } from "next/headers";

export default async function Page() {
  const cookieStore = await cookies();
  const hasAuth = cookieStore.has("auth-token");

  return <div>{hasAuth ? <p>已登录</p> : <p>请先登录</p>}</div>;
}
```

## 写入 Cookie

使用 `set(name, value, options)` 设置 cookie、`delete(name)` 删除 cookie。注意：`set` 和 `delete` 只能在 [Server Function](https://nextjs.org/docs/app/getting-started/updating-data) 或 [Route Handler](https://nextjs.org/docs/app/api-reference/file-conventions/route) 中调用。HTTP 协议不允许在流式响应开始后设置 Cookie，因此必须在可设置响应头的上下文中执行。

### 在 Server Action 中设置

```tsx
// app/actions.ts
"use server";

import { cookies } from "next/headers";

export async function setThemeFromForm(formData: FormData) {
  const theme = (formData.get("theme") as "light" | "dark") ?? "light";
  const cookieStore = await cookies();
  cookieStore.set("theme", theme, {
    path: "/",
    maxAge: 60 * 60 * 24 * 365, // 1 年
  });
}
```

```tsx
// app/page.tsx
import { setThemeFromForm } from "./actions";

export default function Page() {
  return (
    <form action={setThemeFromForm}>
      <input type="hidden" name="theme" value="dark" />
      <button type="submit">切换到深色模式</button>
    </form>
  );
}
```

### set 的三种调用方式

`set` 支持多种传参方式：

```tsx
"use server";

import { cookies } from "next/headers";

export async function createUser(data: FormData) {
  const cookieStore = await cookies();

  // 方式 1：仅 name 和 value
  cookieStore.set("name", "lee");

  // 方式 2：第三个参数传入 options
  cookieStore.set("name", "lee", { secure: true });

  // 方式 3：传入对象（name、value 放在对象内）
  cookieStore.set({
    name: "session",
    value: "abc123",
    httpOnly: true,
    path: "/",
    maxAge: 3600,
  });
}
```

`set` 的第三参数 options 支持：`expires`（过期时间）、`maxAge`（有效期，秒）、`domain`（生效域名）、`path`（生效路径，默认 `'/'`）、`secure`（仅 HTTPS 发送）、`httpOnly`（禁止 JS 访问，防 XSS）、`sameSite`（`'lax'`|`'strict'`|`'none'`）、`priority`、`partitioned`。详见 [MDN Cookie 文档](https://developer.mozilla.org/zh-CN/docs/Web/HTTP/Cookies)。

## 删除 Cookie

删除 Cookie 的本质是让浏览器移除该 Cookie。Next.js 中可通过以下三种方式实现：

### 方式 1：使用 delete

`delete(name)` 是语义最清晰的方式，直接按名称删除指定 Cookie。适用于 `auth-token`、`session` 等需显式登出的场景。

```tsx
// app/actions.ts
"use server";

import { cookies } from "next/headers";

export async function logout() {
  const cookieStore = await cookies();
  cookieStore.delete("auth-token");
}
```

### 方式 2：set 空值

将 Cookie 的值设为空字符串，浏览器会覆盖原值。但部分场景下可能不会立即删除，取决于浏览器实现，不如 `delete` 可靠。

```tsx
cookieStore.set("auth-token", "");
```

### 方式 3：maxAge 设为 0

通过 `set` 将 `maxAge` 设为 0，使 Cookie 立即过期，浏览器会在下次请求时移除。效果与 `delete` 类似，但需传入一个临时 value。

```tsx
cookieStore.set("auth-token", "value", { maxAge: 0 });
```

注意：上述操作只能在 Server Function 或 Route Handler 中调用，且需满足：

- Cookie 与调用 `set` 的域名一致（通配符域名需精确匹配子域）
- 使用与设置时相同的协议（HTTP 或 HTTPS）

## Server Components 与 Server Functions 的差异

Server Component 在渲染时只能读取请求中的 Cookie，无法调用 `set` 或 `delete`。原因是 HTTP 不允许在流式响应开始后设置响应头，而 Server Component 的渲染结果是流式输出的，无法插入 `Set-Cookie`。写入 Cookie 必须在 Server Action、Server Function 或 Route Handler 中完成，这些上下文可以在发送响应前设置响应头。

上图是 Server Action、Route Handler 的 Cookie 流程。在完整响应发送前可设置 `Set-Cookie`，浏览器据此更新存储。

- Server Component：可以读取请求头中的 Cookie，但不能在渲染阶段设置 Cookie。
- Server Function、Route Handler：可以设置响应头中的 `Set-Cookie`，从而让浏览器存储 Cookie。

Cookie 本质是客户端存储，服务器通过 `Set-Cookie` 指导浏览器存储、因此写操作必须在能设置响应头的上下文中执行。

## cookies()、NextRequest.cookies 与 NextResponse.cookies

Next.js 中有三套 Cookie API，适用场景与操作对象不同：

| 特性          | `cookies()` (next/headers)                       | `NextRequest.cookies`                                                                                 | `NextResponse.cookies`          |
| ------------- | ------------------------------------------------ | ----------------------------------------------------------------------------------------------------- | ------------------------------- |
| 使用场景      | Server Component、Server Function、Route Handler | [Middleware](https://nextjs.org/docs/app/building-your-application/routing/middleware)、Route Handler | Middleware、Route Handler       |
| 操作对象      | 请求 Cookie + 响应 Set-Cookie                    | 请求 Cookie（可读写）                                                                                 | 响应 Set-Cookie（可读写）       |
| `set` 作用    | 设置响应头，让浏览器存储 Cookie                  | 在请求中注入 Cookie，转发给下游                                                                       | 设置响应头，让浏览器存储 Cookie |
| `delete` 作用 | 让浏览器删除 Cookie                              | 从请求中移除 Cookie                                                                                   | 让浏览器删除 Cookie             |

- `cookies()`：在 Server Component、Server Function、Route Handler 中读取请求 Cookie，并设置响应的 `Set-Cookie`。
- `NextRequest.cookies`：在 Middleware 中读写请求的 Cookie，修改即将转发给路由的请求。
- `NextResponse.cookies`：在 Middleware 中读写响应的 Cookie，设置返回给浏览器的 `Set-Cookie`，相当于 Middleware 版的 `cookies().set()`。

### 在 Middleware 中操作 Cookie

Middleware 中需同时处理请求和响应：

```tsx
// middleware.ts
import { NextResponse } from "next/server";
import type { NextRequest } from "next/server";

export function middleware(request: NextRequest) {
  // 1. 读取请求中的 Cookie
  const showBanner = request.cookies.get("show-banner")?.value;

  // 2. 创建响应
  const response = NextResponse.next();

  // 3. 通过 NextResponse.cookies 设置响应 Cookie，让浏览器存储
  if (showBanner === undefined) {
    response.cookies.set("show-banner", "true");
  } else if (showBanner === "false") {
    response.cookies.set("show-banner", "false");
    // 响应会带上 Set-Cookie: show-banner=false;path=/home
  }

  // 4. 或通过 NextRequest.cookies 修改请求，再转发给下游
  if (request.cookies.has("experiments")) {
    request.cookies.delete("experiments");
  }

  return response;
}
```

NextRequest.cookies（操作请求）：

| 方法               | 说明                                    |
| ------------------ | --------------------------------------- |
| `get(name)`        | 获取单个 cookie                         |
| `getAll(name?)`    | 获取所有 cookie，可传 `name` 按名称过滤 |
| `set(name, value)` | 在请求中设置 cookie（仅 name、value）   |
| `delete(name)`     | 从请求中删除 cookie                     |
| `has(name)`        | 判断 cookie 是否存在                    |
| `clear()`          | 清除请求中所有 cookie                   |

NextResponse.cookies（操作响应）：

| 方法               | 说明                          |
| ------------------ | ----------------------------- |
| `get(name)`        | 获取响应中已设置的 cookie     |
| `getAll(name?)`    | 获取响应中所有 cookie         |
| `set(name, value)` | 设置响应 Cookie，让浏览器存储 |
| `delete(name)`     | 删除响应 Cookie，让浏览器删除 |
| `has(name)`        | 判断响应中是否已设置该 cookie |

详见 [NextRequest](https://nextjs.org/docs/app/api-reference/functions/next-request)、[NextResponse](https://nextjs.org/docs/app/api-reference/functions/next-response)。

## 在 Server Function 中同时更新 UI 与 Cookie

在 Server Function 中执行 `set` 或 `delete` 后，若将其作为 [Server Action](https://nextjs.org/docs/app/getting-started/updating-data#what-are-server-functions) 使用（例如传给表单的 `action`），Next.js 可在一次请求中同时返回更新后的 UI 和新 Cookie。

组件不会卸载，但依赖服务端数据的 effects 会重新执行。如需刷新缓存，可在 Server Function 内调用 [`revalidatePath`](https://nextjs.org/docs/app/api-reference/functions/revalidatePath) 或 [`revalidateTag`](https://nextjs.org/docs/app/api-reference/functions/revalidateTag)。

## 如何实现一个主题切换

```tsx
// app/actions.ts
"use server";

import { cookies } from "next/headers";

export async function setTheme(theme: "light" | "dark") {
  const cookieStore = await cookies();
  cookieStore.set("theme", theme, {
    path: "/",
    maxAge: 60 * 60 * 24 * 365,
    sameSite: "lax",
  });
}
```

```tsx
// app/page.tsx
import { cookies } from "next/headers";
import { setTheme } from "./actions";

export default async function Page() {
  const cookieStore = await cookies();
  const theme = cookieStore.get("theme")?.value ?? "light";

  return (
    <div>
      <p>当前主题: {theme}</p>
      <form>
        <button formAction={setTheme.bind(null, "light")}>浅色</button>
        <button formAction={setTheme.bind(null, "dark")}>深色</button>
      </form>
    </div>
  );
}
```

点击按钮时，`formAction` 会调用对应的 Server Action，在服务端设置 Cookie，Next.js 会重新获取数据并更新 UI。

## 总结

`cookies()` 是 Next.js 在服务端读写 HTTP Cookie 的异步 API，返回 cookie 存储对象，支持 `get`、`getAll`、`has`、`set`、`delete`。Next.js 15 起须用 `async/await` 或 `use()` 调用，v14 及更早为同步，v15 兼容但将废弃。`cookies` 属于 [Dynamic API](https://nextjs.org/docs/app/guides/caching#dynamic-rendering)，使用会使路由进入动态渲染；因 HTTP 流式响应无法在响应开始后设置 Cookie，`set`、`delete` 必须在 Server Function 或 Route Handler 中调用。

Server Component 只能读取 Cookie，Server Function 和 Route Handler 可读写。主题、会话、语言等偏好适合用 Cookie 存储。在 Middleware 中，`NextRequest.cookies` 操作请求，`NextResponse.cookies` 操作响应（设置返回给浏览器的 Cookie）。
