# use client 和 use server 详解

`use client` 与 `use server` 是 React 的指令，用于在 Next.js App Router 中划分客户端与服务端边界。前者标记在客户端渲染的组件，后者标记在服务端执行的函数。二者配合使用，可构建兼顾性能与交互的应用。

本文基于 [Next.js 官方文档](https://nextjs.org/docs/app/api-reference/directives/use-client) 与 [use server 文档](https://nextjs.org/docs/app/api-reference/directives/use-server) 详解用法与注意事项。

## use client

`use client` 声明客户端组件的入口，表示该文件中的组件将在浏览器中渲染，适用于需要状态管理、事件处理、浏览器 API 等客户端能力的交互式 UI。并非每个包含 Client Component 的文件都要加该指令，只需在会被 Server Component 直接引用的入口文件顶部添加；它定义客户端与服务端的[边界](https://nextjs.org/docs/app/building-your-application/rendering#network-boundary)，从该文件导出的组件即成为客户端入口。下面从概念、用法、约束与组合方式逐一说明。

### 基本用法

在文件最顶部（任意 import 之前）添加 `'use client'`：

```tsx
// app/components/counter.tsx
'use client'

import { useState } from 'react'

export default function Counter() {
  const [count, setCount] = useState(0)

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>Increment</button>
    </div>
  )
}
```

### Props 必须可序列化

使用 `use client` 的组件，其 props 必须是[可序列化](https://react.dev/reference/rsc/use-client#serializable-types)的，因为数据需要从服务端序列化后传给客户端。函数不可序列化，不能直接作为 props 传递。

以下写法会导致错误（函数不可序列化）：

```tsx
// app/components/counter.tsx
'use client'

interface CounterProps {
  onClick: () => void // 普通函数无法序列化，不能作为 props 传递
}

export default function Counter({ onClick }: CounterProps) {
  return <button onClick={onClick}>Increment</button>
}
```

正确做法：将逻辑定义为 Server Action。Server Action 是特殊引用，React 会序列化后传给客户端，客户端点击时再发起请求调用服务端。完整示例（计数器通过 Cookie 持久化）：

```tsx
// app/actions.ts
'use server'

import { cookies } from 'next/headers'

export async function incrementAction(): Promise<void> {
  const cookieStore = await cookies()
  const current = parseInt(cookieStore.get('count')?.value ?? '0', 10)
  cookieStore.set('count', String(current + 1), { path: '/', maxAge: 60 * 60 * 24 })
}
```

```tsx
// app/page.tsx
import { cookies } from 'next/headers'
import Counter from './components/counter'
import { incrementAction } from './actions'

export default async function Page(): Promise<JSX.Element> {
  const cookieStore = await cookies()
  const count = parseInt(cookieStore.get('count')?.value ?? '0', 10)

  return <Counter initialCount={count} onIncrement={incrementAction} />
}
```

```tsx
// app/components/counter.tsx
'use client'

import { useTransition } from 'react'
import { useRouter } from 'next/navigation'

interface CounterProps {
  initialCount: number
  onIncrement: () => Promise<void>
}

export default function Counter({ initialCount, onIncrement }: CounterProps): JSX.Element {
  const router = useRouter()
  const [isPending, startTransition] = useTransition()

  const handleClick = (): void => {
    startTransition(() => {
      onIncrement().then(() => router.refresh())
    })
  }

  return (
    <div>
      <p>Count: {initialCount}</p>
      <button disabled={isPending} onClick={handleClick}>
        Increment
      </button>
    </div>
  )
}
```

若不需要传参，也可直接在 Client Component 中导入 Server Action 使用，无需通过 props 传递。

### 在 Server Component 中嵌套 Client Component

推荐在 Server Component 中按需嵌套 Client Component，实现职责分离。Server Component 负责静态内容、数据获取和 SEO，Client Component 负责需要状态、effects 或浏览器 API 的交互逻辑。在 Server Component 中引入 Client Component 时，仅该入口及其依赖会打入客户端 bundle，其余保持服务端渲染。

```tsx
// app/page.tsx
import Header from './header'
import Counter from './counter' // Client Component

export default function Page() {
  return (
    <div>
      <Header />
      <Counter />
    </div>
  )
}
```

## use server

`use server` 标记函数在服务端执行，常用于数据库操作、鉴权、服务端逻辑等。有两种写法：写在文件顶部，该文件内所有导出的 async 函数均为 Server Function；写在函数体顶部，仅标记该函数。

### 文件顶部使用

在单独文件中顶部添加 `'use server'`，导出的 async 函数即成为 Server Function，可从 Client Component 或 Server Component 中导入调用。

```tsx
// app/actions.ts
'use server'

import { db } from '@/lib/db'

export async function fetchUsers() {
  const users = await db.user.findMany()
  return users
}
```

```tsx
// app/components/my-button.tsx
'use client'

import { fetchUsers } from '../actions'

export default function MyButton() {
  return <button onClick={() => fetchUsers()}>Fetch Users</button>
}
```

### 函数内联使用

在函数体顶部添加 `'use server'`，仅该函数成为 Server Function。适用于需访问闭包（如路由 `params`）的场景，可在 Server Component 内定义后传给 Client Component。

```tsx
// app/posts/[id]/page.tsx
import { EditPost } from './edit-post'
import { revalidatePath } from 'next/cache'

export default async function PostPage({ params }: { params: { id: string } }) {
  const post = await getPost(params.id)

  async function updatePost(formData: FormData) {
    'use server'
    await savePost(params.id, formData)
    revalidatePath(`/posts/${params.id}`)
  }

  return <EditPost updatePostAction={updatePost} post={post} />
}
```

### 安全与权限

Server Function 暴露为可调用 API，执行敏感操作前必须先做认证与鉴权，避免未授权访问。

```tsx
// app/actions.ts
'use server'

import { db } from '@/lib/db'
import { authenticate } from '@/lib/auth'

export async function createUser(
  data: { name: string; email: string },
  token: string
) {
  const user = authenticate(token)
  if (!user) {
    throw new Error('Unauthorized')
  }
  const newUser = await db.user.create({ data })
  return newUser
}
```

## use client 与 use server 对比

二者分别作用于组件与函数，Client Component 可调用 Server Function 完成服务端逻辑。对比如下：

| 特性       | use client                  | use server                        |
| ---------- | --------------------------- | --------------------------------- |
| 作用对象   | 组件（文件级）              | 函数（文件级或函数级）            |
| 执行环境   | 浏览器                      | 服务端                            |
| 典型用途   | 状态、事件、浏览器 API      | 数据库、鉴权、服务端逻辑          |
| 放置位置   | 文件顶部，且必须位于 import 之前 | 文件顶部或函数体顶部              |
| 与对方组合 | 可调用 Server Function      | 可被 Client Component 调用        |

## 总结

`use client` 定义客户端组件入口，用于需要交互、状态和浏览器能力的 UI；`use server` 定义服务端函数，可写在文件顶部或函数内，用于数据操作、鉴权等。二者配合时，Server Component 负责数据与布局，Client Component 负责交互，通过 Server Function 完成服务端逻辑调用。使用时需注意：`use client`、`use server` 均须写在文件或函数体最顶部、任何 import 之前；Client Component 的 props 必须可序列化，不能传递函数、Date、Symbol 等；`use client` 所在文件即客户端 bundle 边界，其导入的模块会一并打入；Server Function 暴露为可调用 API，务必在内部做认证和鉴权。
