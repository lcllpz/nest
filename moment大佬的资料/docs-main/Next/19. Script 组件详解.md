Next.js 内置的 `<Script>` 组件用于控制第三方脚本的加载和执行时机。相比原生 `<script>` 会阻塞渲染且难以控制执行顺序，`<Script>` 提供多种加载策略和生命周期回调，便于按需加载、优化性能。

下图对比传统 script 与 Next.js Script 的差异：

![20260206214742](https://raw.githubusercontent.com/xun082/md/main/blogs.images20260206214742.png)

传统 script 会阻塞渲染并影响性能，Script 则通过策略和回调实现可控加载。

## Script 组件介绍

`<Script>` 基于 HTML `<script>` 扩展，用于加载第三方脚本并控制执行时机。通过 `src` 指定脚本地址，不指定 `strategy` 时默认为 `afterInteractive`：

```tsx
"use client";
// app/page.tsx
import Script from "next/script";
import { useState } from "react";

export default function Dashboard() {
  const [random, setRandom] = useState<number | null>(null);
  return (
    <>
      <Script
        src="https://cdn.jsdelivr.net/npm/lodash@4.17.21/lodash.min.js"
        onLoad={() => {
          const _ = (
            window as unknown as { _: { sample: (arr: number[]) => number } }
          )._;
          if (_) setRandom(_.sample([1, 2, 3, 4]) ?? null);
        }}
      />
      {random !== null && <p>Lodash 随机采样结果: {random}</p>}
    </>
  );
}
```

### 在布局中使用

放在布局中可为该布局及其子路由统一加载脚本，Next.js 会去重，同一脚本只加载一次：

```tsx
// app/layout.tsx
import Script from "next/script";
import type { ReactNode } from "react";

export default function DashboardLayout({ children }: { children: ReactNode }) {
  return (
    <>
      <section>{children}</section>
      <Script src="https://cdn.jsdelivr.net/npm/lodash@4.17.21/lodash.min.js" />
    </>
  );
}
```

访问 `dashboard` 或 `dashboard/settings` 等子路由时都会加载该脚本，且不会重复请求。子页面可通过 `window._` 使用 lodash（需声明 `Window` 类型），例如 `_.debounce(fn, 300)` 或 `_.orderBy(data, 'date')`。

### 在根布局中使用

根布局中的 Script 会对全站生效，适用于全局分析、Cookie 等脚本。出于性能考虑，优先在具体页面或布局中按需加载：

```tsx
// app/layout.tsx
import Script from "next/script";
import type { ReactNode } from "react";

export default function RootLayout({ children }: { children: ReactNode }) {
  return (
    <html lang="zh-CN">
      <body>{children}</body>
      <Script src="https://cdn.jsdelivr.net/npm/dayjs@1.11.10/dayjs.min.js" />
    </html>
  );
}
```

全站任意页面可通过 `window.dayjs` 格式化时间（需声明 `Window` 类型），例如 `dayjs().format('YYYY-MM-DD HH:mm')`。

## Props 属性

Script 支持的常用属性如下，其他 HTML `<script>` 属性会透传到底层元素：

| Prop       | 示例                                                            | 类型     | 是否必传     |
| ---------- | --------------------------------------------------------------- | -------- | ------------ |
| `src`      | `src="https://cdn.jsdelivr.net/npm/dayjs@1.11.10/dayjs.min.js"` | String   | 外部脚本必传 |
| `strategy` | `strategy="lazyOnload"`                                         | String   | -            |
| `onLoad`   | `onLoad={() => {}}`                                             | Function | -            |
| `onReady`  | `onReady={() => {}}`                                            | Function | -            |
| `onError`  | `onError={(e) => {}}`                                           | Function | -            |

## src 属性

`src` 为外部脚本 URL，支持绝对路径或相对路径。使用内联脚本时可省略。

### 内联脚本

除 `src` 外，可直接在组件内写 JavaScript，需通过 `id` 供 Next.js 追踪和去重：

```tsx
<Script id="show-banner">
  {`document.getElementById('banner')?.classList.remove('hidden')`}
</Script>
```

也可用 `dangerouslySetInnerHTML` 传入字符串，同样必须提供 `id`：

```tsx
<Script
  id="show-banner"
  dangerouslySetInnerHTML={{
    __html: `document.getElementById('banner')?.classList.remove('hidden')`,
  }}
/>
```

## strategy 加载策略

`strategy` 决定脚本何时加载和执行，是 Script 的核心能力，共四种：

| 策略                | 时机                         | 适用场景                               |
| ------------------- | ---------------------------- | -------------------------------------- |
| `beforeInteractive` | 页面可交互前，注入到 `head`  | 机器人检测、Cookie 同意、关键 polyfill |
| `afterInteractive`  | 页面可交互后立即加载（默认） | 分析、聊天、地图等                     |
| `lazyOnload`        | 所有资源加载完、浏览器空闲时 | 低优先级统计、广告                     |
| `worker`            | Web Worker 中执行（实验性）  | 非主线程脚本，仅 Pages Router          |

下图展示各策略在页面加载流程中的执行时机：

![20260207095240](https://raw.githubusercontent.com/xun082/md/main/blogs.images20260207095240.png)

`beforeInteractive` 最早执行；`afterInteractive` 在可交互后立即加载；`lazyOnload` 在所有资源加载完且浏览器空闲时加载；`worker` 在独立线程中执行。

### beforeInteractive

在页面可交互前加载，脚本会注入到 `head`，无论写在组件何处。仅支持在根布局中使用，适用于全站必需、需尽早执行的脚本（如 Cookie 同意、机器人检测）：

```tsx
// app/layout.tsx
import Script from "next/script";
import type { ReactNode } from "react";

export default function RootLayout({ children }: { children: ReactNode }) {
  return (
    <html lang="zh-CN">
      <body>{children}</body>
      <Script
        src="https://cdn.jsdelivr.net/npm/dayjs@1.11.10/dayjs.min.js"
        strategy="beforeInteractive"
      />
    </html>
  );
}
```

脚本加载后，页面中可通过 `dayjs().format('YYYY-MM-DD')` 等 API 使用。

### afterInteractive

默认策略。页面可交互后立即加载，可写在任意页面或布局，仅在该路由被访问时加载。适用于分析、聊天、地图等需要尽快加载的脚本：

```tsx
"use client";
import Script from "next/script";

declare global {
  interface Window {
    dayjs?: () => { format: (format: string) => string };
  }
}

export default function Page() {
  return (
    <>
      <Script
        src="https://cdn.jsdelivr.net/npm/dayjs@1.11.10/dayjs.min.js"
        strategy="afterInteractive"
        onLoad={() => {
          const time = window.dayjs?.().format("HH:mm:ss");
          if (time) document.title = `当前时间 ${time}`;
        }}
      />
    </>
  );
}
```

### lazyOnload

在所有资源加载完成、浏览器进入空闲时加载，适合低优先级脚本（如广告、非核心统计）。同样可写在任意页面或布局，按路由按需加载：

```tsx
"use client";
import Script from "next/script";

declare global {
  interface Window {
    dayjs?: () => { format: (format: string) => string };
  }
}

export default function Page() {
  return (
    <Script
      src="https://cdn.jsdelivr.net/npm/dayjs@1.11.10/dayjs.min.js"
      strategy="lazyOnload"
      onLoad={() => {
        const now = window.dayjs?.().format("YYYY-MM-DD HH:mm");
        console.log("低优先级脚本已加载，当前时间:", now);
      }}
    />
  );
}
```

### worker（实验性）

在 Web Worker 中执行脚本，减轻主线程压力，基于 Partytown 实现。目前仅支持 Pages Router，且需在 `next.config.js` 中开启 `experimental.nextScriptWorkers`，并安装 `@builder.io/partytown`。兼容性有限，建议谨慎使用。

## 生命周期回调

Script 提供 `onLoad`、`onReady`、`onError` 三个回调，用于在加载不同阶段执行逻辑。三者均需在客户端组件中使用（`'use client'`），且不能与 `beforeInteractive` 搭配。

下图展示回调触发时机：

![20260207095613](https://raw.githubusercontent.com/xun082/md/main/blogs.images20260207095613.png)

`onLoad` 在脚本首次加载成功时触发；`onReady` 在每次组件挂载时触发（含首次）；`onError` 在加载失败时触发。

### onLoad

脚本加载完成后执行初始化逻辑，适用于 `afterInteractive` 和 `lazyOnload`。例如加载第三方库后调用其 API：

```tsx
"use client";
// app/page.tsx
import Script from "next/script";
import { useState } from "react";

export default function Page() {
  const [random, setRandom] = useState<number | null>(null);
  return (
    <>
      <Script
        src="https://cdnjs.cloudflare.com/ajax/libs/lodash.js/4.17.21/lodash.min.js"
        onLoad={() => {
          const _ = (
            window as unknown as { _: { sample: (arr: number[]) => number } }
          )._;
          if (_) setRandom(_.sample([1, 2, 3, 4]) ?? null);
        }}
      />
      {random !== null && <p>Lodash sample 结果: {random}</p>}
    </>
  );
}
```

### onReady

在脚本加载完成且每次组件挂载时都会执行，适合地图、图表等需要反复初始化的场景。与 `onLoad` 的区别是：`onLoad` 仅首次加载成功时触发，`onReady` 每次挂载都会触发：

```tsx
"use client";
// app/page.tsx
import { useRef } from "react";
import Script from "next/script";

declare global {
  interface Window {
    L?: {
      map: (el: HTMLElement | null) => {
        setView: (latlng: number[], zoom: number) => unknown;
      };
      tileLayer: (url: string) => { addTo: (map: unknown) => unknown };
    };
  }
}

export default function Page() {
  const mapRef = useRef<HTMLDivElement>(null);

  return (
    <>
      <div ref={mapRef} style={{ width: "100%", height: 400 }} />
      <Script
        id="leaflet-maps"
        src="https://unpkg.com/leaflet@1.9.4/dist/leaflet.js"
        onReady={() => {
          if (typeof window.L !== "undefined" && mapRef.current) {
            const map = window.L.map(mapRef.current).setView(
              [-34.397, 150.644],
              8,
            );
            window.L.tileLayer(
              "https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png",
            ).addTo(map);
          }
        }}
      />
    </>
  );
}
```

### onError

脚本加载失败时触发，用于错误上报或降级处理：

```tsx
"use client";
import Script from "next/script";
import { useState } from "react";

declare global {
  interface Window {
    dayjs?: () => { format: (format: string) => string };
  }
}

export default function Page() {
  const [time, setTime] = useState<string>("--");

  return (
    <>
      <p>当前时间: {time}</p>
      <Script
        src="https://cdn.jsdelivr.net/npm/dayjs@1.11.10/dayjs.min.js"
        onLoad={() => {
          const t = window.dayjs?.().format("HH:mm:ss");
          if (t) setTime(t);
        }}
        onError={(e: Error) => {
          console.error("Script failed to load", e);
          setTime("加载失败");
        }}
      />
    </>
  );
}
```

## 其他 props

`id`、`nonce`、`data-*` 等 HTML `<script>` 属性会透传到底层元素，便于 CSP、测试或第三方集成：

```tsx
"use client";
import Script from "next/script";

declare global {
  interface Window {
    dayjs?: () => { format: (format: string) => string };
  }
}

export default function Page() {
  return (
    <>
      <Script
        src="https://cdn.jsdelivr.net/npm/dayjs@1.11.10/dayjs.min.js"
        id="example-script"
        nonce="XUENAJFW"
        data-custom="value"
        onLoad={() => {
          const formatted = window.dayjs?.().format("YYYY年MM月DD日");
          console.log("dayjs 已加载，今日:", formatted);
        }}
      />
    </>
  );
}
```

## 总结

Script 的核心在于按需选择 `strategy` 并善用生命周期回调：

1. 全站必需、需尽早执行（Cookie、机器人检测）→ `beforeInteractive`，放在根布局；
2. 分析、地图、聊天等 → `afterInteractive`（默认）；
3. 广告、非核心统计 → `lazyOnload`；
4. 脚本加载后需初始化 → 使用 `onLoad` 或 `onReady`，组件需加 `'use client'`。
