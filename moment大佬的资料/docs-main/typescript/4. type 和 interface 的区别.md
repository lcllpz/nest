在 TypeScript 中，`type` 和 `interface` 都用于定义类型，并且在许多情况下它们是可以互换使用的。然而，它们有一些关键的区别和各自的适用场景。理解它们的差异对于编写类型安全和可维护的代码非常重要。

## 基本用法

`type` 和 `interface` 都可以用来定义对象类型，在使用上几乎没有区别：

```ts
// 使用 interface
interface User {
  name: string;
  age: number;
}

// 使用 type
type UserType = {
  name: string;
  age: number;
};

const user1: User = { name: "Alice", age: 25 };
const user2: UserType = { name: "Bob", age: 30 };
```

## 扩展方式

`interface` 和 `type` 在扩展方式上有所不同。

`interface` 使用 `extends` 关键字进行继承扩展：

```ts
interface User {
  name: string;
  age: number;
}

interface Admin extends User {
  role: string;
}

const admin: Admin = {
  name: "Alice",
  age: 30,
  role: "Administrator",
};
```

`type` 使用 `&` 交叉类型进行扩展：

```ts
type UserType = {
  name: string;
  age: number;
};

type AdminType = UserType & {
  role: string;
};

const adminType: AdminType = {
  name: "Bob",
  age: 35,
  role: "Admin",
};
```

## 声明合并

`interface` 和 `type` 在声明合并方面有重要区别。

`interface` 支持声明合并，允许多个同名的接口自动合并属性：

```ts
interface User {
  name: string;
}

interface User {
  age: number;
}

const user: User = { name: "Alice", age: 25 }; // 合并后的 User 接口包含了 name 和 age 属性
```

这个特性使得 `interface` 在扩展第三方库类型时非常有用，可以在不修改原始类型定义的情况下添加新的属性。

`type` 不支持声明合并，如果多次定义同一个类型名会报错：

```ts
type UserType = {
  name: string;
};

// 再次定义同名类型会报错
type UserType = {
  age: number; // Error: Duplicate identifier 'UserType'.
};
```

## 联合类型和交叉类型

`type` 支持定义联合类型和交叉类型，而 `interface` 不支持。

`type` 可以定义联合类型和交叉类型：

```ts
// 联合类型
type Status = "success" | "error" | "loading";

// 交叉类型
type Name = { name: string };
type Age = { age: number };
type Person = Name & Age;

const person: Person = { name: "Alice", age: 25 };
```

`interface` 无法直接用于定义联合类型：

```ts
// 不支持
interface Status = "success" | "error" | "loading"; // Error
```

## 类型别名

`type` 可以用来定义除对象之外的其他类型别名，例如基本类型、元组、函数类型等：

```ts
// 定义基本类型别名
type ID = number | string;

// 定义元组类型
type Point = [number, number];

// 定义函数类型
type Handler = (event: Event) => void;

// 定义映射类型
type Readonly<T> = {
  readonly [P in keyof T]: T[P];
};

const id: ID = 123;
const point: Point = [10, 20];
```

`interface` 则只能用于定义对象类型，无法定义基本类型别名、元组类型等。

## 类的实现

`interface` 和 `type` 都可以用来约束类的结构，但使用场景有所不同。

`interface` 在面向对象编程中更加常用，因为它允许声明合并并且有更强的类型扩展能力：

```ts
interface User {
  name: string;
  age: number;
}

class Person implements User {
  name: string;
  age: number;
  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }
}
```

`type` 也可以用于类的约束，但主要用于联合类型和其他复杂类型组合：

```ts
type UserType = {
  name: string;
  age: number;
};

class PersonType implements UserType {
  name: string;
  age: number;
  constructor(name: string, age: number) {
    this.name = name;
    this.age = age;
  }
}
```

## 互操作性

`interface` 和 `type` 在实际开发中可以互相组合使用，这为类型定义提供了更大的灵活性。

`interface` 可以扩展 `type`：

```ts
type Name = { name: string };

interface User extends Name {
  age: number;
}

const user: User = { name: "Alice", age: 25 };
```

`type` 也可以扩展 `interface`：

```ts
interface User {
  name: string;
  age: number;
}

type Admin = User & {
  role: string;
};

const admin: Admin = { name: "Bob", age: 35, role: "Admin" };
```

## 复杂类型的表达能力

`type` 的表达能力更强，因为 `type` 支持联合类型、交叉类型、基本类型别名等复杂类型的组合，因此在处理复杂类型表达时，`type` 更加灵活：

```ts
type ComplexType = string | number | { name: string };

const value1: ComplexType = "Hello";
const value2: ComplexType = 123;
const value3: ComplexType = { name: "Alice" };
```

`interface` 主要用于描述对象的结构，在联合类型和复杂组合类型场景下不如 `type` 灵活：

```ts
// interface 无法直接定义联合类型
// interface ComplexType = string | number | { name: string }; // Error
```

## 总结

| 特性             | `interface`                    | `type`                             |
| ---------------- | ------------------------------ | ---------------------------------- |
| 定义对象类型     | 支持                           | 支持                               |
| 定义基本类型别名 | 不支持                         | 支持                               |
| 声明合并         | 支持                           | 不支持                             |
| 扩展方式         | `extends` 继承                 | `&` 交叉类型                       |
| 支持联合类型     | 不支持                         | 支持                               |
| 支持交叉类型     | 不支持                         | 支持                               |
| 用于类的实现     | 支持（常用）                   | 支持（不常见）                     |
| 表达能力         | 适合对象结构定义               | 适合复杂类型组合                   |
| 常用场景         | 面向对象设计，类与接口组合使用 | 联合类型、复杂类型定义，类型别名等 |

总体来说，`interface` 更适合用于面向对象编程，特别是在类的设计和继承中。`interface` 的适用场景包括：定义对象类型，尤其是面向对象的设计模式；类的约束，接口扩展和实现（implements）；需要声明合并的场景（如第三方库中的类型定义）；需要清晰的继承关系的场景。

`type` 更加灵活，适用于更复杂的类型组合和类型别名的场景。`type` 的适用场景包括：联合类型、交叉类型等复杂类型定义；基本类型和元组类型别名；需要表达多个类型组合时更灵活；映射类型、条件类型等高级类型操作。

在大多数项目中，`interface` 和 `type` 是可以互补使用的，根据具体需求选择合适的工具可以编写出更加类型安全和可维护的代码。
