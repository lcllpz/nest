TypeScript 的类型保护（Type Guards）是一种机制，用于在代码中根据特定条件细化变量的类型，从而确保代码在运行时更安全和准确。类型保护可以通过几种不同的方式实现，包括 typeof 检查、instanceof 检查、in 关键字检查和用户定义的类型保护函数。

## typeof 检查

typeof 检查主要用于基本类型（如 string、number、boolean、symbol）的类型保护。

```ts
function padLeft(value: string, padding: string | number) {
  if (typeof padding === "number") {
    return Array(padding + 1).join(" ") + value;
  }
  if (typeof padding === "string") {
    return padding + value;
  }
  throw new Error(`Expected string or number, got '${typeof padding}'.`);
}
```

在上面的代码中，`typeof padding === "number"` 和 `typeof padding === "string"` 分别对 padding 进行类型保护，确保在每个条件分支中 padding 的类型是确定的。TypeScript 编译器会根据这些类型检查来缩小类型范围。

## instanceof 检查

instanceof 检查用于对象类型之间的类型保护。它通常用于检查一个对象是否是某个类的实例。

```ts
class Bird {
  fly() {
    console.log("Flying");
  }
}

class Fish {
  swim() {
    console.log("Swimming");
  }
}

type Pet = Bird | Fish;

function getPetAction(pet: Pet) {
  if (pet instanceof Bird) {
    pet.fly();
  } else if (pet instanceof Fish) {
    pet.swim();
  }
}

const bird = new Bird();
const fish = new Fish();

getPetAction(bird); // Flying
getPetAction(fish); // Swimming
```

在上面的代码中，`pet instanceof Bird` 和 `pet instanceof Fish` 分别对 pet 进行类型保护，确保在每个条件分支中 pet 的类型是确定的。instanceof 检查在运行时验证对象的构造函数，从而进行类型保护。

## in 关键字检查

在 TypeScript 中，in 关键字可以用于实现类型保护，帮助你在类型不明确时进行类型检查，并保证特定类型的属性在安全范围内使用。in 关键字可以检查对象中是否存在某个属性，从而进行类型保护。

```ts
interface Cat {
  meow: () => void;
}

interface Dog {
  bark: () => void;
}

type Pet = Cat | Dog;

function makeSound(pet: Pet) {
  if ("meow" in pet) {
    pet.meow(); // TypeScript 知道这里是 Cat 类型
  } else {
    pet.bark(); // TypeScript 知道这里是 Dog 类型
  }
}
```

在这个例子中，in 关键字检查对象 pet 中是否存在 meow 属性。如果存在，TypeScript 会将 pet 视为 Cat 类型，并允许你调用 meow 方法。否则，TypeScript 会将 pet 视为 Dog 类型，并允许你调用 bark 方法。

## 自定义类型保护

自定义类型保护是通过定义一个返回类型谓词（type predicate）的函数实现的。类型谓词的格式是 `parameterName is Type`，表示如果函数返回 true，则该参数属于指定的类型。

```ts
interface Bird {
  kind: "bird";
  fly(): void;
}

interface Fish {
  kind: "fish";
  swim(): void;
}

type Pet = Bird | Fish;

function isBird(pet: Pet): pet is Bird {
  return pet.kind === "bird";
}

function getPetAction(pet: Pet) {
  if (isBird(pet)) {
    pet.fly();
  } else {
    pet.swim();
  }
}

const bird: Bird = { kind: "bird", fly: () => console.log("Flying") };
const fish: Fish = { kind: "fish", swim: () => console.log("Swimming") };

getPetAction(bird); // Flying
getPetAction(fish); // Swimming
```

在上面的代码中，isBird 函数通过检查 pet.kind 是否为 "bird" 来进行类型保护，确保在 if (isBird(pet)) 分支中，pet 的类型为 Bird。自定义类型保护函数提供了更灵活的类型检查方式，特别适用于复杂的类型判断场景。

### 自定义类型保护的优势

自定义类型保护函数可以封装复杂的类型判断逻辑，使代码更加清晰和可维护：

```ts
function isString(value: unknown): value is string {
  return typeof value === "string";
}

function processValue(value: unknown) {
  if (isString(value)) {
    console.log(value.toUpperCase()); // value 被缩小为 string 类型
  }
}
```

## 联合类型和类型保护

联合类型（Union Types）可以结合类型保护来使用，以确保在不同的分支中变量的类型是确定的。

```ts
type StringOrNumber = string | number;

function processValue(value: StringOrNumber) {
  if (typeof value === "string") {
    console.log("String value:", value.toUpperCase());
  } else if (typeof value === "number") {
    console.log("Number value:", value.toFixed(2));
  } else {
    console.log("Unexpected type");
  }
}

processValue("hello"); // String value: HELLO
processValue(123.456); // Number value: 123.46
```

在上面的代码中，typeof value === "string" 和 typeof value === "number" 分别对 value 进行类型保护，确保在每个条件分支中 value 的类型是确定的。通过类型保护，TypeScript 可以在不同的分支中提供准确的类型推断。

## 类型保护的使用场景

类型保护在实际开发中非常有用，特别是在处理以下场景时：

- 处理联合类型：当变量可能是多种类型之一时，使用类型保护来缩小类型范围

- API 响应处理：处理来自 API 的响应数据时，使用类型保护来确保数据结构的正确性

- 错误处理：在处理可能为 null 或 undefined 的值时，使用类型保护来确保类型安全

- 多态对象：处理具有不同形状的对象时，使用类型保护来区分不同的对象类型

## 总结

TypeScript 的类型保护通过不同的机制（如 typeof 检查、instanceof 检查、in 关键字检查、自定义类型保护函数等）提供了一种在代码运行时细化变量类型的方法，从而提高了代码的类型安全性和可维护性。

这些机制使得开发者可以在编写代码时充分利用 TypeScript 的类型系统，避免类型错误，并使代码更加健壮。通过合理使用类型保护，可以编写出既灵活又类型安全的代码，提高代码的可读性和可维护性。
