`Extract` 是 TypeScript 提供的一个条件类型工具，用于从一个联合类型 `T` 中提取出与另一个类型 `U` 兼容的类型。简单来说，`Extract<T, U>` 返回一个新类型，它包含 `T` 中所有与 `U` 类型兼容的部分。

## 基本使用

假设我们有两个联合类型 `T` 和 `U`，我们希望从 `T` 中提取出与 `U` 兼容的类型。

首先我们先来看一个提取类型的例子：

```typescript
type A = string | number | boolean; // A 是 string | number | boolean
type B = number | boolean; // B 是 number | boolean

// Extract A 中与 B 兼容的部分，结果是 number | boolean
type Result = Extract<A, B>; // Result 是 number | boolean
```

在这个例子中，`A` 是一个联合类型，包含 `string | number | boolean`。`B` 是另一个联合类型，包含 `number | boolean`。使用 `Extract<A, B>` 后，`Result` 将是 `number | boolean`，因为这两个类型都同时出现在 `A` 和 `B` 中。

我们再来讲解一个提取字面量类型的例子：

```typescript
type Fruit = "apple" | "banana" | "orange";
type Citrus = "orange" | "lemon";

// 提取 Fruit 中与 Citrus 兼容的部分，结果是 'orange'
type ExtractResult = Extract<Fruit, Citrus>; // ExtractResult 是 'orange'
```

在这个例子中，`Fruit` 包含 `'apple' | 'banana' | 'orange'`，而 `Citrus` 包含 `'orange' | 'lemon'`。使用 `Extract<Fruit, Citrus>` 后，结果是 `'orange'`，因为只有 `'orange'` 同时出现在 `Fruit` 和 `Citrus` 中。

## 实现原理

`Extract` 使用条件类型来实现，它的内部机制是判断类型 `T` 中的元素是否也属于 `U`。它的实现可以用条件类型来表示：

```typescript
type Extract<T, U> = T extends U ? T : never;
```

在上面的代码中：

- `T extends U`：检查 `T` 中的元素是否也属于 `U` 类型

- 如果 `T` 中的元素与 `U` 兼容，则返回 `T`，即保留该类型

- 如果 `T` 中的元素与 `U` 不兼容，则返回 `never`，表示排除该类型

## Extract 与 Exclude 的区别

`Extract` 和 `Exclude` 是相反的操作：

- `Extract<T, U>`：从 `T` 中提取与 `U` 兼容的类型（交集）

- `Exclude<T, U>`：从 `T` 中排除与 `U` 兼容的类型（差集）

```typescript
type A = string | number | boolean;
type B = number | boolean;

type ExtractResult = Extract<A, B>; // number | boolean
type ExcludeResult = Exclude<A, B>; // string
```

## 使用场景

`Extract` 在实际开发中有以下常见应用场景：

1. 类型交集：从多个联合类型中提取共同的类型，创建类型交集

2. 类型过滤：根据特定条件从联合类型中筛选出符合条件的类型

3. 条件类型：在条件类型中使用 `Extract` 来实现复杂的类型判断和转换

4. 工具类型组合：与其他工具类型结合使用，实现更复杂的类型操作

5. API 类型处理：在处理 API 响应类型时，提取特定的字段类型

## 总结

`Extract<T, U>` 是 TypeScript 的一个条件类型工具，用于从联合类型 `T` 中提取出与类型 `U` 兼容的部分。它帮助我们筛选出符合特定条件的类型，常用于提取多个类型中的子集。

`Extract` 的实现基于条件类型机制，通过判断类型是否兼容，决定哪些类型应被保留。最终，它返回一个新的类型，只包含 `T` 中与 `U` 兼容的部分。通过合理使用 `Extract`，可以编写出更加类型安全和健壮的代码，提高代码的可维护性和可读性。
