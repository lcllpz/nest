TypeScript 提供了丰富的类型系统，帮助我们编写更安全、更可维护的代码。本文档将介绍 TypeScript 中的基本类型，包括基础类型、数组、元组、枚举等，以及一些高级类型特性如条件类型、映射类型等。

## 1. 基础类型

基础类型是 TypeScript 中最基本的类型，与 JavaScript 的原生类型相对应。主要包括：

- string：字符串类型

- number：数字类型（包括整数和浮点数）

- boolean：布尔类型（true/false）

- null：空值类型

- undefined：未定义类型

- void：表示没有任何返回值的函数返回类型

- symbol：唯一标识符类型

这些类型的使用方式与 JavaScript 基本一致，但 TypeScript 提供了类型检查的能力。

```ts
let name: string = "TypeScript";
let age: number = 10;
let isActive: boolean = true;
let nothing: null = null;
let notDefined: undefined = undefined;

function logMessage(): void {
  console.log("This function returns nothing");
}
```

## 2. 数组类型

数组类型用来表示元素类型一致的数组。TypeScript 提供了两种方式定义数组类型：

方式一：类型 + 方括号

```ts
const numbers: number[] = [1, 2, 3, 4, 5];
const strings: string[] = ["hello", "world"];
```

方式二：泛型数组

```ts
const numbers: Array<number> = [1, 2, 3, 4, 5];
const strings: Array<string> = ["hello", "world"];
```

两种方式功能相同，可以根据个人喜好选择。推荐使用第一种方式，因为它更简洁易读。

## 3. 元组类型

元组类型允许表示一个已知元素数量和类型的数组，各元素的类型不必相同。元组在需要固定长度和类型顺序的场景中非常有用。

```ts
// 定义一个元组类型
let x: [string, number];

// 合法赋值
x = ["hello", 10];

// 不合法赋值 - 类型顺序错误
// x = [10, "hello"]; // Error: Type 'number' is not assignable to type 'string'

// 不合法赋值 - 元素数量不匹配
// x = ["hello"]; // Error: Type '[string]' is not assignable to type '[string, number]'
```

可选元组元素：

```ts
let optionalTuple: [string, number?];
optionalTuple = ["hello"]; // 第二个元素可选
optionalTuple = ["hello", 10]; // 也可以提供
```

## 4. 枚举类型

枚举类型用于定义一组命名常量，可以是数字枚举或字符串枚举。枚举提供了更好的代码可读性和维护性。

数字枚举：

```ts
enum Color {
  Red, // 0
  Green, // 1
  Blue, // 2
}

const c: Color = Color.Green; // 1
console.log(Color[1]); // "Green"
```

自定义数字枚举：

```ts
enum Status {
  Pending = 1,
  Approved = 2,
  Rejected = 3,
}
```

字符串枚举：

```ts
enum Direction {
  Up = "UP",
  Down = "DOWN",
  Left = "LEFT",
  Right = "RIGHT",
}

const dir: Direction = Direction.Up; // "UP"
```

## 5. Any 类型

`any` 类型表示任意类型的值，使用 `any` 类型可以跳过 TypeScript 的类型检查。虽然提供了灵活性，但会失去类型安全的优势，应该谨慎使用。

```ts
let notSure: any = 4;
notSure = "maybe a string instead";
notSure = false;
notSure.toUpperCase(); // 编译时不会报错，但运行时可能出错
```

注意：在 TypeScript 中，应该尽量避免使用 `any`，优先考虑使用 `unknown` 类型。

## 6. Unknown 类型

`unknown` 类型表示未知类型的值，类似于 `any`，但更安全。对 `unknown` 类型的值进行操作之前必须进行类型检查或类型断言。

```ts
let notSure: unknown = 4;
notSure = "maybe a string instead";
notSure = false;

// 直接操作会报错
// console.log(notSure.toUpperCase()); // Error: Object is of type 'unknown'

// 需要类型检查
if (typeof notSure === "string") {
  console.log(notSure.toUpperCase()); // 安全
}

// 或使用类型断言
console.log((notSure as string).toUpperCase());
```

`unknown` 是类型安全的 `any`，推荐在需要表示未知类型时使用。

## 7. Never 类型

`never` 类型表示那些永不存在的值类型，通常用于：

- 总是会抛出错误的函数

- 无限循环的函数

- 类型守卫中不可能到达的分支

```ts
// 抛出错误的函数
function error(message: string): never {
  throw new Error(message);
}

// 无限循环的函数
function infiniteLoop(): never {
  while (true) {
    // ...
  }
}

// 类型守卫中的使用
function assertNever(x: never): never {
  throw new Error("Unexpected value: " + x);
}
```

## 8. 函数类型

函数类型用于定义函数的参数类型和返回值类型。TypeScript 提供了多种定义函数类型的方式。

函数声明：

```ts
function add(x: number, y: number): number {
  return x + y;
}
```

函数表达式：

```ts
const add: (x: number, y: number) => number = function (
  x: number,
  y: number
): number {
  return x + y;
};
```

箭头函数：

```ts
const add = (x: number, y: number): number => {
  return x + y;
};
```

可选参数和默认参数：

```ts
function greet(name: string, greeting?: string): string {
  return `${greeting || "Hello"}, ${name}!`;
}

function multiply(a: number, b: number = 1): number {
  return a * b;
}
```

## 9. 交叉类型

交叉类型（Intersection Types）用来组合多个类型成一个类型，使用 `&` 操作符。表示对象必须同时满足所有被组合类型的成员。

```ts
interface Person {
  name: string;
  age: number;
}

interface Contact {
  phone: string;
  email: string;
}

// 交叉类型：必须同时包含 Person 和 Contact 的所有属性
type PersonContact = Person & Contact;

const contact: PersonContact = {
  name: "moment",
  age: 25,
  phone: "1234567890",
  email: "moment@example.com",
};
```

交叉类型的实际应用：

```ts
type ReadonlyAndPartial<T> = Readonly<T> & Partial<T>;
```

## 10. 索引类型

索引类型允许使用动态属性名来访问类型中的属性。主要包括 `keyof` 操作符和索引访问类型。

`keyof` 操作符可以获取一个类型的所有键的联合类型：

```ts
interface Person {
  name: string;
  age: number;
  email: string;
}

type PersonKeys = keyof Person; // "name" | "age" | "email"

// 使用示例
function getProperty<T, K extends keyof T>(obj: T, key: K): T[K] {
  return obj[key];
}

const person: Person = {
  name: "Alice",
  age: 30,
  email: "alice@example.com",
};

const name = getProperty(person, "name"); // string
const age = getProperty(person, "age"); // number
```

可以通过索引访问类型来获取类型中某个属性的类型：

```ts
interface Person {
  name: string;
  age: number;
}

type NameType = Person["name"]; // string
type AgeType = Person["age"]; // number
type PersonValueTypes = Person[keyof Person]; // string | number
```

## 11. 条件类型

条件类型（Conditional Types）是一种基于类型的条件判断语法，可以根据条件返回不同的类型。语法如下：

```ts
T extends U ? X : Y
```

这表示如果类型 `T` 能够赋值给类型 `U`（即 `T` 是 `U` 的子类型），那么返回类型 `X`，否则返回类型 `Y`。

基础示例：

```ts
type IsString<T> = T extends string ? true : false;

type A = IsString<string>; // true
type B = IsString<number>; // false
type C = IsString<"hello">; // true（字面量类型是 string 的子类型）
```

实际应用：

```ts
// 提取数组元素类型
type ArrayElementType<T> = T extends (infer U)[] ? U : never;

type StringArray = string[];
type Element = ArrayElementType<StringArray>; // string

// 提取函数返回类型
type ReturnType<T> = T extends (...args: any[]) => infer R ? R : never;

type Func = () => number;
type Return = ReturnType<Func>; // number
```

## 12. 联合类型

联合类型（Union Types）允许一个变量可以是几种类型之一。使用竖线 `|` 分隔不同的类型。

```ts
type MyUnion = string | number | boolean;

let value: MyUnion;

value = "Hello"; // 合法
value = 123; // 合法
value = true; // 合法
// value = {}; // 不合法：Error: Type '{}' is not assignable to type 'MyUnion'
```

在使用联合类型时，通常需要类型守卫来缩小类型范围：

```ts
function processValue(value: string | number) {
  if (typeof value === "string") {
    // 在这个分支中，value 的类型被缩小为 string
    console.log(value.toUpperCase());
  } else {
    // 在这个分支中，value 的类型被缩小为 number
    console.log(value.toFixed(2));
  }
}
```

字面量联合类型：

```ts
type Status = "pending" | "approved" | "rejected";

let currentStatus: Status = "pending";
// currentStatus = "invalid"; // Error: Type '"invalid"' is not assignable to type 'Status'
```

## 13. 递归类型

递归类型是指类型引用自身的一部分，用于定义复杂的嵌套结构，如树结构、链表、JSON 等。

树结构示例：

```ts
interface TreeNode {
  value: string;
  children?: TreeNode[];
}

const tree: TreeNode = {
  value: "root",
  children: [
    { value: "child1" },
    {
      value: "child2",
      children: [{ value: "grandchild1" }, { value: "grandchild2" }],
    },
  ],
};
```

JSON 类型示例：

```ts
type JSONValue =
  | string
  | number
  | boolean
  | null
  | JSONValue[]
  | { [key: string]: JSONValue };

const json: JSONValue = {
  name: "Alice",
  age: 30,
  active: true,
  tags: ["developer", "typescript"],
  metadata: {
    created: "2024-01-01",
    nested: {
      deep: "value",
    },
  },
};
```

## 14. 映射类型

映射类型（Mapped Types）是一种根据已存在的类型生成新类型的方法。它通过在类型中遍历已有的键来创建新的类型。

基础示例：

```ts
type MyReadonly<T> = {
  readonly [P in keyof T]: T[P];
};

type Person = {
  name: string;
  age: number;
};

type ReadonlyPerson = MyReadonly<Person>;
// 等价于：
// type ReadonlyPerson = {
//   readonly name: string;
//   readonly age: number;
// };

const person: ReadonlyPerson = {
  name: "Alice",
  age: 30,
};

// person.name = "Bob"; // Error: Cannot assign to 'name' because it is a read-only property
```

TypeScript 提供了一些内置的映射类型：

```ts
// Partial<T> - 所有属性变为可选
type PartialPerson = Partial<Person>;
// { name?: string; age?: number; }

// Required<T> - 所有属性变为必需
type RequiredPerson = Required<PartialPerson>;
// { name: string; age: number; }

// Pick<T, K> - 选择部分属性
type NameOnly = Pick<Person, "name">;
// { name: string; }

// Omit<T, K> - 排除部分属性
type WithoutAge = Omit<Person, "age">;
// { name: string; }
```

自定义映射类型示例：

```ts
// 将所有属性变为可选且可为 null
type Nullable<T> = {
  [P in keyof T]: T[P] | null;
};

// 添加前缀到所有属性名
type Prefixed<T, Prefix extends string> = {
  [P in keyof T as `${Prefix}${string & P}`]: T[P];
};
```

## 总结

通过学习这些基本的类型，我们可以：

1. 理解 TypeScript 的类型系统基础：从简单的原始类型到复杂的类型操作

2. 掌握类型组合技巧：交叉类型、联合类型、映射类型等高级特性

3. 为学习复杂类型做准备：这些基础知识是学习类型体操（Type Gymnastics）的基石

4. 提升代码质量：合理使用类型系统可以大大提升代码的可维护性和健壮性

这些基本类型是 TypeScript 类型系统的核心，熟练掌握它们将有助于我们编写更安全、更优雅的 TypeScript 代码。
