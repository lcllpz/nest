前面几章讲 Agent 的创建、中间件、工具、护栏，本质都在回答一个问题，怎么在一条链路上插入业务逻辑。真正落到执行时，你总会需要一份"当次调用相关的数据"，例如当前用户是谁、走哪套配置、是否允许读取长期记忆。若这些数据依赖全局变量或闭包传来传去，单测里很难稳定复现某个用户场景，多实例部署也容易出现状态污染。

LangChain 和 LangGraph 把这类"单次调用的环境信息"集中在 `RunnableConfig` 里。每次调用 `agent.invoke(...)` 你都可以传入一份配置对象，工具和 `BaseCallbackHandler` 也都能读到它。日常业务里最常用的三块是：

- `configurable`：当次调用的环境数据，由你在 `invoke` 时传入，例如 `userId`、`tenantId`、`role`、`thread_id`，在工具里通过 `config.configurable` 读取。
- `store`：`BaseStore` 实例，通过 `createReactAgent` 的 `store` 选项挂入，在工具里通过 `LangGraphRunnableConfig.store` 读写，用来跨会话持久化用户偏好、历史摘要等长期记忆。
- `signal`：`AbortSignal`，实现请求超时或用户主动取消。

工具和回调拿到的是同一份配置对象，同一次请求里读到的数据完全一致，不需要额外做同步。

![20260316101957](https://raw.githubusercontent.com/xun082/md/main/blogs.images20260316101957.png)

`invoke` 传入的 `configurable` 会随配置对象流经整条链路，工具执行时可以通过第二个参数 `config` 直接读取。`store` 则会随 `createReactAgent` 的选项一起挂入。

## 怎么通过 configurable 传入上下文

调用 `agent.invoke` 时，把当次请求的环境数据放进第二个参数的 `configurable` 字段，工具里用第二个参数 `config` 读取。

`configurable` 适合放：

- 用户标识：`userId`、`tenantId`、`role` 等，做鉴权或多租户隔离。
- 客户端实例：数据库连接、三方 API 客户端等，避免在工具里重新初始化。
- 功能开关：`enableDebugLog`、`allowHighRiskTools` 这类按请求维度开关。
- 请求级配置：回复语言、超时设置等临时参数。

不适合放进 `configurable` 的是需要跨会话持久化的数据，那类数据用 `store` 写入和读取。

下面这段定义了一个从 `config.configurable` 读用户信息的工具，在 `invoke` 时传入 `userId`、`userName`、`role`，当用户问"我叫什么名字？"时，模型先调工具拿到用户名再回答。

```typescript
import { z } from "zod";
import { tool } from "@langchain/core/tools";
import { createReactAgent } from "@langchain/langgraph/prebuilt";
import { ChatOpenAI } from "@langchain/openai";
import { HumanMessage } from "@langchain/core/messages";
import type { RunnableConfig } from "@langchain/core/runnables";

const model = new ChatOpenAI({
  model: "Pro/zai-org/GLM-5",
  apiKey: "sk-cjqjqkehyqpspbqcvnlkvifbvfjrmdusostyrvdbdaatrzyw",
  configuration: { baseURL: "https://api.siliconflow.cn/v1" },
  temperature: 0,
});

const getCurrentUserInfo = tool(
  async (_input: Record<string, never>, config?: RunnableConfig) => {
    const { userId, userName, role } = config?.configurable ?? {};
    return JSON.stringify({ userId, userName, role });
  },
  {
    name: "get_current_user_info",
    description: "获取当前登录用户的名字和角色。",
    schema: z.object({}),
  },
);

const agent = createReactAgent({
  llm: model,
  tools: [getCurrentUserInfo],
});

const result = await agent.invoke(
  { messages: [new HumanMessage("我叫什么名字？我的角色是什么？")] },
  {
    configurable: {
      thread_id: "1",
      userId: "u_001",
      userName: "张三",
      role: "admin",
    },
  },
);

const lastMsg = result.messages[result.messages.length - 1];
console.log(lastMsg.content);
```

## 在工具里读 configurable 和 store

工具函数的第二个参数是 `config`，可以读 `config.configurable` 拿当次调用的环境数据，也可以读 `config.store` 做长期记忆的读写。`store` 需要在创建 agent 时通过 `store` 选项传入 `InMemoryStore` 实例。

下面这个工具优先从 `store` 读用户的邮件偏好，没有时返回默认值，同时还演示了写入 `store` 的方式。

```typescript
import { z } from "zod";
import { tool } from "@langchain/core/tools";
import { createReactAgent } from "@langchain/langgraph/prebuilt";
import { InMemoryStore } from "@langchain/langgraph";
import { ChatOpenAI } from "@langchain/openai";
import { HumanMessage } from "@langchain/core/messages";
import type { LangGraphRunnableConfig } from "@langchain/langgraph";

interface EmailPreferences {
  style: string;
  language: string;
}

const model = new ChatOpenAI({
  model: "Pro/zai-org/GLM-5",
  apiKey: "sk-cjqjqkehyqpspbqcvnlkvifbvfjrmdusostyrvdbdaatrzyw",
  configuration: { baseURL: "https://api.siliconflow.cn/v1" },
  temperature: 0,
});

const store = new InMemoryStore();

// 预置一条偏好数据
await store.put(["users", "email_prefs"], "张三", {
  style: "正式",
  language: "英文",
} satisfies EmailPreferences);

const fetchUserEmailPreferences = tool(
  async (_input: Record<string, never>, config?: LangGraphRunnableConfig) => {
    const userName = config?.configurable?.userName as string | undefined;
    if (!userName) throw new Error("userName 未设置");

    const record = await config?.store?.get(["users", "email_prefs"], userName);
    if (record?.value) {
      const prefs = record.value as EmailPreferences;
      return `${userName} 的邮件偏好：风格"${prefs.style}"，语言"${prefs.language}"。`;
    }
    return `${userName} 的邮件偏好（默认）：风格"简短、礼貌"，语言"中文"。`;
  },
  {
    name: "fetch_user_email_preferences",
    description:
      "获取当前用户的邮件偏好，优先从长期记忆里读取，没有则返回默认值。",
    schema: z.object({}),
  },
);

const agent = createReactAgent({
  llm: model,
  tools: [fetchUserEmailPreferences],
  store,
});

const result = await agent.invoke(
  { messages: [new HumanMessage("当前用户的邮件偏好是什么？")] },
  { configurable: { thread_id: "1", userName: "张三" } },
);

const lastMsg = result.messages[result.messages.length - 1];
console.log(lastMsg.content);
```

![20260224100303](https://raw.githubusercontent.com/xun082/md/main/blogs.images20260224100303.png)

如果要在工具里更新偏好，调 `config.store.put` 写入即可：

```typescript
await config?.store?.put(["users", "email_prefs"], userName, {
  style: "简短",
  language: "中文",
} satisfies EmailPreferences);
```

`put` 写入的内容在同一 `store` 实例下跨会话可读，下次该用户进来时工具会读到更新后的偏好。

## 在回调里读取调用上下文

`BaseCallbackHandler` 的钩子可以通过 `metadata` 字段拿到调用时传入的额外信息，用于打日志或做请求级埋点。动态系统提示则可以用 `RunnableLambda` 作为 `createReactAgent` 的 `prompt` 选项，从 `config.configurable` 读取当次调用的用户信息注入提示词。

下面示例里，`dynamicPrompt` 根据 `configurable` 里的 `userName` 和 `role` 组装系统提示，`LoggingMiddleware` 在调模型前后打带用户信息的日志。

```typescript
import { RunnableLambda } from "@langchain/core/runnables";
import { createReactAgent } from "@langchain/langgraph/prebuilt";
import { BaseCallbackHandler } from "@langchain/core/callbacks/base";
import { SystemMessage, HumanMessage } from "@langchain/core/messages";
import { ChatOpenAI } from "@langchain/openai";
import type { BaseMessage } from "@langchain/core/messages";
import type { RunnableConfig } from "@langchain/core/runnables";
import type { Serialized } from "@langchain/core/load/serializable";

const model = new ChatOpenAI({
  model: "Pro/zai-org/GLM-5",
  apiKey: "sk-cjqjqkehyqpspbqcvnlkvifbvfjrmdusostyrvdbdaatrzyw",
  configuration: { baseURL: "https://api.siliconflow.cn/v1" },
  temperature: 0,
});

// 动态系统提示：从 configurable 读用户信息注入提示词
const dynamicPrompt = RunnableLambda.from(
  (state: { messages: BaseMessage[] }, config?: RunnableConfig) => {
    const { userName, role } = config?.configurable ?? {};
    const roleLabel = role === "admin" ? "管理员" : "用户";
    const systemMsg = `你是贴心助手。请用中文回复，称呼当前${roleLabel}为"${userName ?? "用户"}"。`;
    return [new SystemMessage(systemMsg), ...state.messages];
  },
);

// 日志中间件：从 metadata 读 userId，在调用前后打日志
class LoggingMiddleware extends BaseCallbackHandler {
  name = "logging-middleware";

  override async handleChatModelStart(
    _llm: Serialized,
    _messages: BaseMessage[][],
    _runId: string,
    _parentRunId?: string,
    _tags?: string[],
    metadata?: Record<string, unknown>,
  ): Promise<void> {
    console.log(
      `[${new Date().toISOString()}] 请求开始 userId=${metadata?.userId} userName=${metadata?.userName}`,
    );
  }

  override async handleLLMEnd(): Promise<void> {
    console.log(`[${new Date().toISOString()}] 请求结束`);
  }
}

const agent = createReactAgent({
  llm: model,
  tools: [],
  prompt: dynamicPrompt,
});

const result = await agent.invoke(
  { messages: [new HumanMessage("你好，我叫什么？")] },
  {
    configurable: { thread_id: "1", userName: "张三", role: "admin" },
    metadata: { userId: "u_001", userName: "张三" },
    callbacks: [new LoggingMiddleware()],
  },
);

const lastMsg = result.messages[result.messages.length - 1];
console.log(lastMsg.content);
```

需要在调用前读 `store`（例如拼历史摘要进系统提示）时，`RunnableLambda` 本身也支持接收 `LangGraphRunnableConfig`，从 `config.store` 读写即可。

## 多租户场景下的 configurable 隔离

`configurable` 天然支持多租户隔离。每次 `agent.invoke` 传入的 `configurable` 是当次请求的快照，并发执行的不同租户请求各有独立的数据，不会互相干扰。

常见做法是在 HTTP 请求进来时从鉴权层解析出租户信息，然后构造 `configurable` 传给 `agent.invoke`：

```typescript
import { createReactAgent } from "@langchain/langgraph/prebuilt";
import { ChatOpenAI } from "@langchain/openai";
import { HumanMessage } from "@langchain/core/messages";

const model = new ChatOpenAI({
  model: "Pro/zai-org/GLM-5",
  apiKey: "sk-cjqjqkehyqpspbqcvnlkvifbvfjrmdusostyrvdbdaatrzyw",
  configuration: { baseURL: "https://api.siliconflow.cn/v1" },
  temperature: 0,
});

const agent = createReactAgent({ llm: model, tools: [] });

// 模拟两个不同租户并发请求
async function handleChat(
  userId: string,
  tenantId: string,
  allowedTools: string[],
  userMessage: string,
): Promise<string> {
  const result = await agent.invoke(
    { messages: [new HumanMessage(userMessage)] },
    {
      configurable: {
        thread_id: `${tenantId}-${userId}`,
        userId,
        tenantId,
        allowedTools,
      },
    },
  );
  const lastMsg = result.messages[result.messages.length - 1];
  return lastMsg.content as string;
}

const [replyA, replyB] = await Promise.all([
  handleChat("u_001", "tenant_a", ["search"], "你好，介绍一下自己"),
  handleChat("u_002", "tenant_b", ["search", "send_email"], "你好，我是谁？"),
]);

console.log("租户 A 回复：", replyA);
console.log("租户 B 回复：", replyB);
```

两个租户并发执行，各自的回复互不干扰，`configurable` 里的 `tenantId`、`allowedTools` 可在工具里做数据隔离和权限校验。

## 小结

LangGraph 把请求级环境数据收拢在 `RunnableConfig` 里，三块各司其职：

- `configurable`：当次调用的静态环境（用户身份、租户配置、功能开关），在 `invoke` 时传入，工具里通过 `config.configurable` 读取。
- `store`：跨会话的动态状态（用户偏好、历史摘要），通过 `createReactAgent` 的 `store` 选项挂入，工具里通过 `LangGraphRunnableConfig.store` 读写。
- `signal`：`AbortSignal`，用于请求超时或用户主动取消。

把用户身份、租户配置等通过 `configurable` 传入而非塞进全局状态，Agent 就能保持无状态，便于单测、多实例部署和横向扩展。`store` 处理需要持久化的内容，`RunnableLambda` + `BaseCallbackHandler` 组合处理动态提示注入和请求级日志，三者搭配能支撑多用户、多租户、有长期记忆的复杂业务场景。
